Comment 6 Oleg Besedin 2007-05-08 16:47:40 EDT 
I think this bug needs to be re-considered. At the very least, we could add a "low memory" warning dialog for 1.5 and later VMs.

On 1.5 and later java.lang.management APIs could be used to track this information. In particular, MemoryMXBean allows trackers to be added. Such trackers could be set to trigger on a low memory conditions. Tracker could bring up dialog telling user the type of memory it is running out of and suggested action to take. (I really like the text suggested in the https://www.eclipse.org/bugs/show_bug.cgi?id=92250#c131,  +1 to using it :-)).

A new bundle (fragment?) with this code could be added and directed to require 1.5 VM. 

As a side benefit, such dialog could help with both help and non-heap memory (Code and PermGen). The current out-of-memory dialog is problematic as: 1) if you are really out of memory you can't create it; 2) it is not triggered in case VM hangs; 3) it happens too late to properly shutdown Eclipse. 

The reason I would like to see it done for 3.3 is that with each release amount of PermGen memory consumed by Eclipse gets bigger.

To put a number on in, in a very simple scenario Eclipse 3.0 used 32Mb; Eclipse 3.3M7 uses 47Mb.

The scenario consists of: starting Eclipse; creating a "Hello World" plug-in project from the wizard; opening and closing Help; opening one file in Java editor and one file in the manifest editor.

As you see, some very basic use case brings amount of PermGen memory consumed to 47Mb. 

At the same time, default limit on Sun VM is 64Mb. This limit can be easily reached just by using the functionality of the SDK itself.

Illustrative patch

Equinox team considered two approaches for PermGen: one is addition of the VM argument in the launcher; another is low memory warning dialog.

For 3.3 we will go with addition of the argument in the launcher (bug 186886); the low memory dialog will be considered post-3.3.

Attached is the draft/illustrative code that shows low memory warning dialog. As attached, dialog is generic and is not limited to just PermGen.

Open questions:
- At present it is implemented as a fragment on org.eclipse.ui.ide (this is there "out of memory" dialog is). Should it be available at the lower level?
- Preferences - having a new proceference page for this fragment caused a great deal of disagreement. General feeling was that putting preferences in preferences.ini file was enough. There are two preferences: "don't show this dialog again" and low memory threshold (as a % of maximum size). 
- Testing needs to be done on different VMs

Comment 19 Oleg Besedin 2011-06-07 10:55:54 EDT 
Created attachment 197505 [details]
Draft low memory monitor v2

Draft code for the low memory monitor.

This version offers to open [eclipse].ini file used by the running eclipse instance in the text editor.

Also archive includes a plugin to emulate heap and PermGen memory consumption.

Possible improvements:

- Add a Help page with detailed description of options; link it somewhere to the warning dialog.

This still needs testing on different OS & VMs.

====

There are 3 places where memory options can be specified. In the order of priority, from highest to lowest:

- runtime arguments 
- ini file
- the "--launcher" argument in the ini file

I briefly entertained an idea of automatically changing ini file but due to its free-form nature and ability to specify memory options in multiple places that seems potentially problematic. 

Instead, I decided to open ini file in a text editor and display (side-by-side?) a help entry describing memory options.
