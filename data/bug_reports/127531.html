Comment 14 Eric Moffatt 2007-01-16 10:38:29 EST 
One of the uses of the 'menus' extension is to allow the adding of trim. The story is that trim controls are simply toolbars containing a single 'widget' item and with a location URI indicating the desire to have a new Toolbar in a particular trim area. (exact definition TBD).

I'll switch the example over to using the menus EP (thus solving this defect...;-) as part of the port over to the new API; needed anyway to show the change in our 'best practices'.

A bit off topic but we'll also need a ToolBarManager whose TB honors the Trim API so it can be dragged (calling 'update(true)' after docking).
