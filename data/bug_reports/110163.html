public class MyEventQueue extends EventQueue{ 
    public void postEvent(AWTEvent event) {
        OS.gdk_error_trap_push();
        super.postEvent(event);
        OS.gdk_error_trap_pop();
    }

    protected void dispatchEvent(AWTEvent event){
        OS.gdk_error_trap_push();
        super.dispatchEvent(event);
        OS.gdk_error_trap_pop();
    }

    public AWTEvent getNextEvent() throws InterruptedException {
        OS.gdk_error_trap_push();
        AWTEvent event = super.getNextEvent();
        OS.gdk_error_trap_pop();
        return event;
    }

    public synchronized AWTEvent peekEvent() {
        OS.gdk_error_trap_push();
        AWTEvent event = super.peekEvent();
        OS.gdk_error_trap_pop();
        return event;
    }

    public synchronized AWTEvent peekEvent(int id) {
        OS.gdk_error_trap_push();
        AWTEvent event = super.peekEvent(id);
        OS.gdk_error_trap_pop();
        return event;
    }

    protected void pop() throws EmptyStackException {
        OS.gdk_error_trap_push();
        super.pop();
        OS.gdk_error_trap_pop();
    }

    public synchronized void push(EventQueue newEventQueue) {
        OS.gdk_error_trap_push();
        super.push(newEventQueue);
        OS.gdk_error_trap_pop();
        
    }
    
}

I'm working on the following AWT bug:

6678385: Random java.lang.StackOverflowError from various JDKs

http://bugs.sun.com/view_bug.do?bug_id=6678385

and recently noticed this problem in SWT_AWT bugs database. Although #110163 is marked as RESOLVED/FIXED, it seems to be some kind of workaround rather than a real fix - please, correct me if I'm wrong.

Briefly, what I'm going to implement. As Oleg said, AWT has never been a XError-free library, and it would take much time to refactor it. Instead, a synthetic XErrorHandler model is introduced: when AWT starts, it registers its XErrorHandler which is then used to dispatch all the XErrors for any real handlers and only uninstalled on AWT termination. XToolkit.WITH_XERROR_HANDLER and XToolkit.RESTORE_XERROR_HANDLER are rewritten accordingly, so no real XErrorHandler is affected.

I realize this is not a perfect solution and it may not help in some particular scenarios. Still, it makes AWT a bit more friendly to other libraries when used in the same process. In particular, issue #142644 which is closed as a duplicate of #110163 must gone away.
