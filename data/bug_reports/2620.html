Here is an initial list of faults I found with Japanese strings.  Some of these problems are
not because the strings are Japanese, but because the strings are generally longer than
english strings.

I will indicate a URL to an image that displays each fault (copy/paste it to explorer/netscape)

1) Edit Action Sets - the list box on the right does not grab excess space.
Q:\viper\team\jlebrun\NLS test\jp-nls01 (Edit Action Sets).jpg

2) Filter Tasks (Task View) - using the generated strings, this dialog requires a screen resolution
of 1280x1024.  Anything lower and it cannot be fully displayed.  The average user will be using a
lower resolution than this.
Q:\viper\team\jlebrun\NLS test\jp-nls02 (Filter Tasks).jpg

3) Find/Replace - the title has not been externalized.
Q:\viper\team\jlebrun\NLS test\jp-nls03 (Find-Replace).jpg

4) All wizards - the title of the message banner cannot display Japanese characters.  I
verified that the characters are in the messages.properties file but I assume this is a
limitation of the font used for the banner title.  example:
Q:\viper\team\jlebrun\NLS test\jp-nls04 (Save As...).jpg

5) Select Perspective - the list box does not grab excess space.
Q:\viper\team\jlebrun\NLS test\jp-nls05 (Select Perspective).jpg

6) Show View - the list box does not grab excess space.
Q:\viper\team\jlebrun\NLS test\jp-nls06 (Show View).jpg

** Wizards setting explicit window size **
7) Export Wizard - i. the 'Back' button is not visible, and the 'Next' button is cut off.
ii. the text of the export types in the list is cut off (in this case the 'File System' in the middle
of the list box (there is a lot of white space that could be used)
Q:\viper\team\jlebrun\NLS test\jp-nls07 (Export).jpg

8) Import Wizard - i. the 'Back' button is not visible, and the 'Next' button is cut off.
ii. the text of the import types in the list is cut off (in this case the 'File System' in the middle
of the list box (there is a lot of white space that could be used)
Q:\viper\team\jlebrun\NLS test\jp-nls08 (Import).jpg

9) New Resource Wizard - the 'Back' button is not visible, and the 'Next' button is cut off.
Q:\viper\team\jlebrun\NLS test\jp-nls09 (New Resource).jpg

10) New Project Wizard - the 'Back' button is not visible, and the 'Next' button is cut off.
Q:\viper\team\jlebrun\NLS test\jp-nls10 (New Project).jpg

The problem with 7-10 is that wizards are setting an exact size.  This should be set as a
minimum size.  The wizard is resized when next is pressed because the wizard auto-
adjusts its size.
On a related note, two difference sizes are used for wizards: 470x550 and 500x500.
(the first for import/export, the latter for new resources/projects)
Just a suggestion but maybe we should be more consistent by providing a standard
wizard size as Static ints in the WizardDialog class.  We could take this one step further
by adding a method setMinimumSize() that simply sets the dimensions as a minimum size.
This would give wizards a consistent look and feel but would only be optional.

JEL (25/07/2001 11:40:21 AM)
RE: 14) The problem here is that the button widths are calculated on the average character size
of the font used.  This is sufficient in most cases.  For the rare case where there are many
fat characters, the estimation will not be large enough.  We can add a safety measure that will
expand the individual button if the estimation is too small.

ex: 	buttonData.widthHint = Math.max(buttonWidth, deselectButton.computeSize(SWT.DEFAULT,SWT.DEFAULT, true).x);
This applies to classes: org.eclipse.ui.wizards.datatransfer::WizardFileSystemResourceImportPage,
and WizardExportResourcesPage
The button will be slightly larger than the other buttons, but at least the text will show.

RE: 7i, 8i, 9, 10) The size is being set, we should check to make sure it is not smaller than the calculated size.

ex: Point size = dialog.getShell().getSize();
	dialog.getShell().setSize( Math.max(SIZING_WIZARD_WIDTH, size.x), Math.max(SIZING_WIZARD_HEIGHT, size.y) );
This applies to classes: org.eclipse.ui.internal::NewProjectAction, ImportResourcesAction, and ExportResourcesAction,
and org.eclipse.ui.actions::NewWizardAction,
