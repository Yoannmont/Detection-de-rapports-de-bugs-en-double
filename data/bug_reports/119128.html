Comment 3 Harold Gartner 2005-12-12 14:29:43 EST 
This bug is not entirely fixed in RC2.  In the original text:

"One more thing which is not working (although it may once the above is
corrected), is that only j2ee.jar, and not all the jars in the 'lib' directory
of the WebSphere installation, is the only jar file added to the build path of
the server runtime container.  I would expect all the jar files to be there."

All of the jar file in the lib directory need to be set on the container's classpath.  Otherwise the code we generate won't compile as is references classes outside of j2ee.jar.  Creating a basic J2EE runtime library adds all of the jars in the specified directory to the container classpath.  I would assume that targetting a Webshpere server should be doing exactly the same thing.
