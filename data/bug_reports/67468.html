Comment 10 Martin Aeschlimann 2004-06-21 03:19:58 EDT 
There are two modes:
- pure Alphabetically (When you check the preference)
- by relevance and then alphabetically

The relevance is offered by jdt.core. Mostly type proposals have the same 
relevance, except if the full type name is there, then that proposal has a 
higher relevance.

We used a Collator to sort the proposal by their displayed label. The problem 
is that the collator sorts whitespaces behind letters. Try the 
example. 'JavaUI - test' is sorted last. The current fix uses 
String.compareToIgnoreCase again. This is a workaround. We should definitly 
use collators.


public class A {
    public static void main(String[] args) {
        String s1= "JavaUI - test";
        String s2= "JavaUI2 - test";
        String s3= "JavaUIBla - test";
        String s4= "JavaUIPla - test";
        String[] arr= { s1, s2, s3, s4 };
        
        Arrays.sort(arr, Collator.getInstance());

        for (int i = 0; i < arr.length; i++) {
            System.out.println(arr[i]);
        }
    }

}
