Description Jim 2004-06-15 12:10:47 EDT 
Using Eclipse 3.0 RC2.
With the following code:

public class Test {
/*
final String prefix = "BT_";
String sss = "BT_aaa"; 
*/
}

After Source | format I have:
public class Test {
/*
 * final String prefix = "BT_"; String sss = "BT_aaa";
 */
}
The comments are concatenated into a single line.


Now when I selected the commented block and choose "Source | Remove Block 
Comment"

I get:

public class Test {
 * final String prefix = "BT_"; String sss = "BT_aaa";
}

which is not compilable.

Somehow, eclipse is assuming that the comments are javadocs, which in this case 
is not true and it is unable to restore the commented block to compilable code 
(apart from performing an undo)
Comment 1 Dani Megert 2004-06-15 12:52:40 EDT 
Add Block Comment adds
/*
<selected block>
*/

and Remove Block Comment simply removes /* and */ for the block which causes
this error.

The formatter generates block comments as speced in the Java Code Conventions
document i.e.
/*
 * some comment
 */

We should
1) fix the Remove Block Comment to handle all kinds of block comments
2) fix Add Block Comment to generate same block comments as the formatter

Comment 5 Zach Toups 2006-07-21 09:59:46 EDT 
Just to explain where this is a problem...

I'll often test code by commenting out sections and writing new ones using a normal block comment...the trouble is that I tend to Format my code on a regular basis. If I end up formatting while some useful code is commented, I then have the joy of removing all the *'s and fixing the line wraps when I uncomment it.

Seems like there should be a Formatting preference to disable making pretty block comments (separate from pretty JavaDoc comments, which I'd use no matter what).
-Zach

You would need something similar to this in the config file for the repo:

[remote "origin"]
	url = ssh://dmegert@git.eclipse.org/gitroot/jdt/eclipse.jdt.ui.git
	fetch = +refs/heads/*:refs/remotes/origin/*
	fetch = refs/notes/*:refs/notes/*
[remote "gerrit"]
	url = ssh://dmegert@git.eclipse.org:29418/jdt/eclipse.jdt.ui.git
	fetch = +refs/heads/*:refs/remotes/origin/*
	fetch = refs/notes/*:refs/notes/*
	push = HEAD:refs/for/master

And of course you have to set up SSH as described in the article.

Another important thing is that you can only push a commit if it is signed off by you (e-mail address), and that address matches what you have in the Gerrit account.
