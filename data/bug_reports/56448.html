Description Benjamin Pasero 2004-03-27 06:24:39 EST 
Hi,

in my application I am making use of the ViewForm widget for the overall look 
and feel.
I noticed, that the thin, darkgray line that seperated top-controlls and 
contentcontrolls in the ViewForm, is not showing anymore using SWT 3.0 M8.

I made two screenshots of my application to show the difference:

ViewForm with SWT 3.0 M7: http://rssowl.sourceforge.net/download/rssowl_old.JPG

ViewForm with SWT 3.0 M8: http://rssowl.sourceforge.net/download/rssowl_new.JPG

Looking at it, the divider line seems to be there (or at least space for it), 
but not with the same darkgray color as in the older version.

Sample Snippet:

public class Main {
	public static void main(String[] args) {
		Display display = new Display();
		Shell shell = new Shell(display);
		shell.setLayout(new FillLayout());

		ViewForm viewForm = new ViewForm(shell, SWT.BORDER);

		Composite c = new Composite(viewForm, SWT.NONE);
		viewForm.setContent(c);

		Composite c2 = new Composite(viewForm, SWT.NONE);
		viewForm.setTopLeft(c2);

		shell.open();
		while (!shell.isDisposed()) {
			if (!display.readAndDispatch()) {
				display.sleep();
			}
		}
	}
}

Any suggestions or reasons why the dividerline is not showing anymore?
Ben

Comment 7 Veronika Irvine 2004-05-26 11:39:07 EDT 
The ViewForm reflects the desired look for Eclipse 3.0.

There is a separator line between the top area and the content area but no 
separator line between the individual sections of the top area.

This is intentional and will not change.  The 3.0 look tries to reduce the 
visual clutter of unneccessary lines.  Changes to this appearance must be made 
through the UI design team.

Comment 10 Veronika Irvine 2004-05-26 13:31:38 EDT 
The purpose of ViewForm is to provide the look of a View to all views in 
Eclipse.  As the Eclipse look changes so will the appearance of this widget.

If a new class were created, plugins would have to change to use the new class 
in order to be consistent with the new look.  That is not desirable.  We want 
the plugins to look right in Eclipse 3.0 without any change.

If you need to maintain a specific look, I recommend you define your own 
widget.  ViewForm is explicitely for views in Eclipse.

Comment 15 Mike Wilson 2004-05-26 16:21:19 EDT 
To clarify what Veronika said, the look of a widget is *not* API unless it is specified in the API 
contract. API == Application Programming Interface. It is the set of classes, methods and fields that are 
publicly available to application code. The javadoc for these API methods specify the contract for their 
use. As was previously mentioned, the classes in the org.eclipse.swt.custom package were specifically 
created to provide the custom branding for Eclipse. That is what they are for, and that is what they do. 
There are absolutely no guarantees that the appearance of these widgets will remain the same between 
releases. You can not claim that the fact that the appearance is different implies that there has been a 
breaking change in the API.

Having said that, I recognize that you are looking for a way to get the old appearance. Is it possible that 
you could leverage the "R2.1 presentation" work that has been done for R3.0 to get what you want?
