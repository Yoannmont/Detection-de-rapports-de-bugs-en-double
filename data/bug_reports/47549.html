Comment 9 Brian de Alwis 2004-03-08 15:04:29 EST 
NetBSD man pages for getrlimit/setrlimit say that SIGSEGV (11) is sent when the
process runs out of memory:

     The system refuses to extend the data or stack space when the limits
     would be exceeded in the normal way: a brk(2) call fails if the data
     space limit is reached.  When the stack limit is reached, the process
     receives a segmentation fault (SIGSEGV); if this signal is not caught by
     a handler using the signal stack, this signal will kill the process.

Extending my allowable data segment size seems to have put this off.  Looking at
some Linux man pages
<http://www.icewalkers.com/Linux/ManPages/setrlimit-2.html>, SIGSEGV seems to be
sent if there's a stack overflow.
