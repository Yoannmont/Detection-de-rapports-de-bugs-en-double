Description Luca Truffarelli 2004-11-17 10:36:08 EST 
Using:
- j2sdk1.4.2_05
- Eclipse 3.0.1
- EclipseMe 0.6.0
- Sun's WTK 2.2

If I create a new J2ME project its "J2ME" Project Properties is not available.

j2sdk1.4.2_05 was already installed, to reproduce the bug I:
- Installed Sun's WTK 2.2
- Installed (extracted) Eclipse 3.0.1
- Installed (Find and install) EclipseMe 0.6.0
- Configured EclipseMe adding the Wireless toolkit
- Replaced 
eclipse\plugins\eclipseme.toolkit.sun_0.6.0\definitions.xml
     with
http://cvs.sourceforge.net/viewcvs.py/*checkout*/eclipseme/eclipseme.toolkit.su
n/definitions.xml?rev=1.5.
to fix a bug in EclipseMe.
- Created a new J2ME project

Then, opening the project properties window, the "J2ME" tag is not present.

I already opened a bug to EclipseMe team, and we done a lot of tests, but it 
is not an EclipseMe bug.
The stack trace shows it is dying in base Eclipse code.  Looking through the 
code related to the stack trace, we can see absolutely nothing that EclipseME 
could be doing to cause this problem.

Comment 3 John Arthorne 2004-11-19 09:40:29 EST 
Moving to UI, since this is a problem with the properties dialog.  However, the
Eclipse team will likely close this bug unless the EclipseMe developers can show
why this is a bug in the Eclipse platform. Clearly other property pages work

Comment 5 Craig Setera 2004-11-19 14:06:40 EST 
Two quick follow-ups on this bug.

1) The plugins are being installed as an Eclipse Archived Update site.  I'm
assuming that the Update support is doing the correct thing so that the features
and plugins are being appropriately added to the registry.  I also believe that
-clean was attempted in this case and it did not help.

2) As the EclipseME developer, I've tried everything I can think of to help the
two users that have reported this problem.  The property category does not
display for two users out of a very large number of users that see the category
fine.  I'm not doing anything fancy with the category.  I am more than happy to
do whatever I can to help debug this situation, but I've run out of ideas.  From
an outsider looking in (plugin looking at the platform), it appears that the
platform is ignoring the EclipseME support in a very small set of circumstances.
 If someone can give me some tracing or debugging options, I will attempt to
further debug the situation, but the fact that it works for everyone else makes
me question whether there is an underlying Eclipse bug.

Comment 14 Luca Truffarelli 2004-11-24 10:42:43 EST 
We found a workaround to the problem.

The option isn't available in the "Resource perspective" of 
Eclipse but only the "Java Perspective". So just switch to the 
Java perspective and everything will work again.

As described in:
https://sourceforge.net/tracker/?
func=detail&atid=581073&aid=1021750&group_id=86829
