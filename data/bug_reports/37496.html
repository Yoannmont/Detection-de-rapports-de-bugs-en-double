Comment 20 Alex Blewitt 2006-11-01 18:39:47 EST 
Re comment #19, I agree. Sort members is missing from the clean up options, and could easily be put in the same place as the organise imports and code formatter. (Like the new sort members function, it should probably allow sorting of field members to be selectively enabled as well as other members, but it doesn't seem too hard to imagine the kind of checkboxes for other parts of the CleanUp interface). I'd be tempted to suggest that sort members should be default, but sorting fields shouldn't be (because the latter can cause problems). 

Once it's in the cleanup profile, it should be easy to hook up with the clean-on-save functionality listed in bug 6749.
Comment 21 Alex Blewitt 2007-01-02 18:30:27 EST 
For those that are interested, I've contributed an implementation of the Sort Members that can be used inside the Clean Up code, which includes the ability to hook this into the format-on-save functionality.

Code (based on Eclipse 3.3M4) is at bug 168954, though not as a patch at present.

If you want sort members functionality to be added to the clean-up wizard, please vote for bug 168954.
