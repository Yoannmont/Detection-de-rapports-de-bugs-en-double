Comment 3 Karen Butzke 2007-08-23 09:51:01 EDT 
The changes I have made in bug 199715 will prevent the dialog from appearing if you get exceptions during the resynch.  The resynching thread is a background thread and if more java source changes are made while it is running our model will not be up to date with the java model.  This can lead to many exceptions.  We are now catching and logging those exceptions (not swallowing them since there could certainly be programmer errors) instead of having the dialog appear.  The next time the resynching thread runs, the model should be up to date with the java model.  As more changes occur it will just run again.  This should handle the first 2 exceptions you mentioned.


In comment #2 you say:

And if during the refactoring you attempt to modify the persistent/orm xml
models in adaptor code, this exception occurs sometimes:

I am not sure what you mean by "modify the persistent/orm xml models in adaptor code".  Any more info?
