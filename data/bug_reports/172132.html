Description Vinay Pandit 2007-01-30 05:18:26 EST 
I have implemented support for persisting the format applied to a variable (local, global and register) during a debug session and across debug sessions. I am also persisting the cast applied to a variable in the same way. 

For implementing this support, I require to access the original expression of a variable, ignoring any currently applied cast. This original expression serves as the key to save the format of the variable. I also need to access the start index and offset specified by the user if he/she has cast the variable to an array. 

This information is currently available in the class VariableDescriptor and the class IInternalVariable. In order to access this information at the level of class CVariable, I have added / modified the following methods:

A list of all additions and modifications is as follows:

1. Interface ICastToArray
1.1 Add method int getCastingArrayStart()
1.2 Add method int getCastingArrayEnd()

2. Class AbstractCVariable
2.1 Add abstract method public abstract String getOriginalExpressionString() throws DebugException

3. Class CVariable
3.1 Add method public String getOriginalExpressionString() throws DebugException
Adding this method enables the client persistence code to retrieve the original expression of the variable, ignoring any currently applied cast. This original expression serves as the key in client code to save the format of the variable.
Thus for variables, the key is constructed as [functionname]:uncastedexpression, and for registers and global variables, the key is constructed as uncastedexpression.
3.2 Add method public int getCastingArrayStart()
3.3 Add method public int getCastingArrayEnd()
(methods 3.2 and 3.3 are added to conform to changed interface ICastToArray)
These methods enable the client persistence code to save the start index and offset if the variable is cast to an array. These methods delegate to the methods of the same name in ICDIVariableDescriptor (added in 6.1, 6.2). This delegation is done by getting the IInternalVariable object by calling getCurrentInternalVariable and calling the method added in 4.1.

4. Interface IInternalVariable
4.1 Add method ICDIVariableDescriptor getCDIVariableObject()

5. Class InternalVariable in classes CGlobalVariable, CLocalVariable, CRegister
5.1 changed access modifier of existing method ICDIVariableDescriptor getCDIVariableObject() from private to public (to conform to changed interface IInternalVariable)

6. Interface ICDIVariableDescriptor
6.1 Add method int getCastingArrayStart()
6.2 Add method int getCastingArrayEnd()
Adding these methods makes it possible for code in the methods 3.1 and 3.2 to access the existing methods of the same name in the class VariableDescriptor

I am attaching a patch which contains the methods I have added and changed in the interfaces and classes mentioned above. I request the team to add the methods into the CDT codebase.

Comment 7 Vinay Pandit 2007-02-09 06:08:23 EST 
The patch is ready with us. From the legal documents it looks like we can directly attach the patch to this bug. The reviewer/committer who will look into this patch will initiate the questionnaire and other things later. Am I right, or do I need to fill those before attaching patch?

Description/Working of the patch:

The patch will contribute two features,
  1. Changing format or type or applied cast to array will be persisted during a debug session and across debug sessions.
  2. Format applied to array variables will be propagated to its children.  

The patch will introduce 3 new classes, CVariableFormatPersistanceManager, VariableFormatRepresentation, VariableFormatPersistenceXMLTasks. CVariableFormatPersistanceManager saves, maintains and initializes the added variable formats information like type cast added, cast to array added and changes in display format to hex/decimal/binary. Each debug target (i.e. in class CDebugTarget) will have an instance of CVariableFormatPersistanceManager.
 
The class CVariableFormatPersistanceManager listens for IDebugEventSetListener. Upon changing format or type or applying cast to array, this class gets notification with events kind = DebugEvent.CHANGE, events detail = DebugEvent.STATE and event source instance of AbstractCVariable. For each such variable for which state changed, a VariableFormatRepresentation object will be created which will summarize all the changes made to the variable. And each such object will be stored in a map maintained in this class with key computed as follows,
 1. For local variables, key = function_name : original_variable_expression 
 2. For registers & global variables, key =  original_variable_expression
To confirm whether the type has changed the original type will be compared with current type of the variable. In case of format applied to an array variable, along with storing the format change information to the map the format will be propagated to all its children only if number of children is less than 100.

Whenever a new CLocalVariable/CGlobalVariable/CRegister object is created, these classes will request CVariableFormatPersistanceManager to apply stored format. CVariableFormatPersistanceManager will check if there exists an entry for the variable in the map and if so it will apply the stored format/type.

Upon terminating the target, the information stored in map will be stored to launch and during subsequent launches these will be read and map will be updated with information present in the launch. This task is performed by VariableFormatPersistenceXMLTasks class.

Multi-Target launch issues:
In case of launch having multiple targets, format information stored in launch could possible be overwritten by format information of last terminated target. To avoid this, a “persist-able name” will be given to each target and will be stored in launch along with format change information.  The client implementation need to override method getPersistableName() in CDebugTarget to give unique name for target. The default implementation returns empty String. 

Overriding this feature by client implementation:

To turn off or to have own implementation of this feature, the client implementation must override all of the  following methods of class CDebugTarget,

setCVariableFormatPersistanceManager()
initializeCVariableFormatPersistanceManager()
getCVariableFormatPersistanceManager() 
disposeCVariableFormatPersistanceManager()
saveVariableFormatChanges()
 
Known limitations: 
In case of two local variables with the same name defined in different blocks in the same function, format will be applied to both the variable during the next debug session.
