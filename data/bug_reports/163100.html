Description David Williams 2006-11-01 12:47:33 EST 
Now that osgi/equinox will be providing some "servers" (most likely via "orbit" project) Seems to me we in WTP could take advantage of them and "pre-req" or "reship" one for the ability for end users to be able to install and immeidately start using WTP. I'm not sure what version of J2EE they are using (I think an old one?) but would be worth exploring. And, BTW, just because they are shipping it, we'd still have to do some amount of IP work to make sure ok that we re-ship it. 

I'm just opening this here for comment, and to continue bug 153141.
Comment 1 Tim deBoer 2006-11-01 16:43:27 EST 
So you are requesting we create a default server adapter that uses osgi/orbit server capability?
Comment 2 David Williams 2006-11-01 17:05:27 EST 
Yes, let's consider it. I think they (per OSGI spec) will have a HTTPD server and a servlet 2.1, not sure about JSP ... some of the more "advanced" versions may be part of equanox incubator? 

And, I do mean an adapter for WTP and, then, "re-ship" the OSGI bundles that give that server support. That way, end-users who just wanted to try out WTP, or that was just getting started with web development, would not have to worry about which server to use right off the bat.
Comment 3 Jeff McAffer 2006-11-17 20:20:14 EST 
interesting.  What is *likely* to happen in the Eclipse SDK is that we will move from "standalone" Tomcat to embedded Jetty for our Help support.  Keep in mind that this is the only reason we include a server in the first place.

However, with that in place, you should be able to start launch Jetty for standard WTP test usecases.

This is different however from the server side Equinox usecases described on 
     http://www.eclipse.org/equinox/server/
Here the actual web content is in bundles.  Servlets and such are actually loaded by the OSGi classloaders etc.

I suspect that support for these structures really is a separate feature.

Comment 15 Tim deBoer 2007-05-29 13:18:42 EDT 
The original point of this bug was to provide a usable server adapter in WTP right 'out of the box'. That is now complete in WTP 2.0 - we're internally reusing the platform's Jetty bundle to provide both HTTP and J2EE 'preview' support. A user can easily run their apps directly from within WTP without downloading an external server or server adapter.

There are other out of the box enhancements tracked elsewhere (and please make sure that additional bugs are opened if you have other concerns), but since this one is complete I am resolving it as fixed.
