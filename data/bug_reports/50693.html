Description Jared Burns 2004-01-27 12:30:30 EST

a ${env_var:TEST_PATH} in the "Local Directory" text box, it allowed me to do 
so, and even verified that the path existed, and let me save it.  Wahoo!.  But,

${env_var:TEST_PATH}, it mysteriously moves the ${env_var:TEST_PATH} to the 
"Workspace" text box, and now, it tells me that "The Specified project or folder 
does not exist".

Comment 10 Darin Wright 2004-06-16 23:17:57 EDT 
The patch effects 3 plug-ins:

1) The jdt debug ui plug-in: WorkingDirectoryBlock is modified to contain one 
text field with three push buttons. It was neccessary to combine the text 
fields since it is no longer possible to tell if the directory is an external 
directory or a resource when a variable reference is present (which we cannot 
reliably infer/resolve when the dialog is open).
2) the jdt launching plug-in. Resolution of the working directory is (almost) 
the same. When a variable resolves to a "workspace relative path", it still 
contains a prefixed slash (which is interpretted as an absolute path in the 
platform). Thus, if the absolute path does not resolve to a file in the file 
system, we attempt to interpret it as a workspace relative path before giving 
up.
3) Ant ui - subclass of the working directory for the Ant JRE tab. Changes to 
reflect changes in superclass/widgets.

Darin S/Jared: Please take a look at the patch and see if you think this is 
safe enough to put into RC3. If we have agreement, then we will look for 
another component owner to approve our change.
