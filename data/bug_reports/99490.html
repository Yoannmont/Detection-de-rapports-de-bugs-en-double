Comment 2 Tom Hofmann 2005-06-13 03:35:02 EDT 
Strange. 

Inspecting the code in RangeDifferencer, this could happen if a IRangeComparator
with a negative-length region was used.

Inspecting DocumentLineDiffer, I believe we cannot compute a negative range,
although this behavior is never asserted (it probably should).

(In reply to comment #0)
> The text editor on a small text file seems to have problems. I typed a line at
> the end of the file, the text comes out mixed up. 

This indicates that the document structures were already corrupt after typing
the new line. Paul, can you confirm that the document was already corrupt (i.e.
not showing the expected contents) after entering the new line, but that the
exception only showed up when you pressed 'Delete'?

> Then I do a delete and I get: java.lang.NegativeArraySizeException

If line information on the document is not correct, it is of course possible
that we get bogus line ranges. In that case, the exception in comment 0 is
merely a symptom of another problem.
