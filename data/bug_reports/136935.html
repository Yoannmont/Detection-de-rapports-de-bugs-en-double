Comment 8 Nitin Dahyabhai 2006-08-02 08:48:20 EDT 
Created attachment 47215 [details]
followup patch moving input stream creation into the helper object

Jeffrey or Lawrence, could one of you revisit this one?  The attached patch delays the input stream creation until a validator actually requests it, but it's not the main reason for my request.

It looks strange to me that we're converting the contents of the structured document into a byte stream, where the current patch takes effect, only to change it *back* into text at some point in all of the delegated validators.  XMLValidator creates an InputSource and ultimately uses a Reader to feed the text (converted back from the InputStream) to Xerces.  In the WSDL ValidationController and its DefaultXMLValidator, the stream is converted into a string and back again more than once.  The XSDValidator is feeding the inputstream into its XMLInputSource, which could just as easily use a Reader, preferrably one directly backed by the Structured Document (it's an IDocument after all).  Switching from supplying an InputStream to a Reader that supports mark/reset could save a good amount of time and memory when validating.
