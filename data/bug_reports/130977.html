Description Missing name 2006-03-08 15:37:06 EST 
Bizarre error occurs when an XML document with a root element that has an attribute named "default"

Steps to reproduce:

1. Create a new XML file called "foo.xml" and enter the following content:
 
<?xml version="1.0" encoding="UTF-8"?>
<project default="1">
</project>

2. Save the file and close the XML editor.
3. Right click on the file in Package Explorer
4. Select "Open with" -> "XML Editor"
5. Error message dialog box shows up. See attached image.
6. File is opened but with the TextEditor instead of the XMLEditor

Additional Info:
- If you remove the "default" attribute, everything is ok
- If you move the "default" attribute to a non-root element, everything is ok.

Comment 4 Amy Wu 2006-03-08 17:22:05 EST 
Actually, the Ant Content Type (Ant Buildfile) is a subtype of the XML Content type and our XML editor opens all XML content type, whether it's really XML or a subtype.  So there's no need to specifically define that our editor can open ant content type.

I think the issue is probably that we're not using the right document provider.  Or at least we're not getting a StructuredDocument back from the document provider.

I was not able to recreate this problem on WTP 1.5M5.  Analogue, what do you have installed?  Do you have anything extra installed besides WTP?
