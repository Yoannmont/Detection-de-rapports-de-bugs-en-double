Description Andre Costa 2006-04-21 12:35:41 EDT 
If I try to debug the application I'm working on with 3.2RC1, Eclipse aborts with OutOfMemory (this also happened with 3.2M6). The same app, with the same vmargs parameters (-Xms128m -Xmx512m -Dide.gc=true) debugs just fine with 3.1.2. This is proving to be a major obstacle for 3.2 beta testing, since it's forcing me constantly to go back to 3.1.2...

(my machine has 512MB of RAM. Please don't tell me I will need to buy more RAM to run 3.2, I am starting to like it and would hate to get stuck with 3.1.2 ;-))
