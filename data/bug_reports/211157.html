Comment 8 Carolyn MacLeod 2007-11-28 15:28:00 EST 
The native "Edit" control does the same thing as StyledText, so I'm not sure what to say here...

import org.eclipse.swt.*;
import org.eclipse.swt.widgets.*;
import org.eclipse.swt.layout.*;
import org.eclipse.swt.custom.*;

public class TextVsStyledText {
	static Display display;
	static Shell shell;
	static Text text, text2;
	static StyledText styledText;
	static String string = "test\baaa";
		
	public static void main(String[] args) {
		display = new Display();
		shell = new Shell(display);
		shell.setLayout(new GridLayout(2, true));
		
		new Label(shell, SWT.NONE).setText("Text (SINGLE):");
		text = new Text(shell, SWT.SINGLE | SWT.BORDER);
		text.setText(string);
		
		new Label(shell, SWT.NONE).setText("Text (MULTI):");
		text2 = new Text(shell, SWT.MULTI | SWT.BORDER);
		text2.setText(string);
		
		new Label(shell, SWT.NONE).setText("StyledText:");
		styledText = new StyledText(shell, SWT.MULTI | SWT.BORDER);
		styledText.setText(string);
		
		shell.pack();
		shell.open();
		while (!shell.isDisposed()) {
			if (!display.readAndDispatch()) display.sleep();
		}
		display.dispose();
	}
}

What are you trying to do ?

If you want to display "tesaaa" want don't you set "tesaaa" in your text widget (instead of "test\baaa")?

If you are reading bytes from a stream you can handle \b yourself (use styledText.invokeAction(ST.DELETE_PREVIOUS)).

GUI Text processing does not handle \b, it works on the console (IMO) cause it is an old text UI.
