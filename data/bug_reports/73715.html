ActionSetMenuManager.dispose() does:
        final IContributionItem[] items = getItems();
        for (int i = 0; i < items.length; i++) {
            items[i].dispose();
        }
        removeAll();
which is reasonable, but getItems() is overridden here to:
	public IContributionItem[] getItems() {
		return getParentMenuManager().getItems();
	}

This can't be right.  The whole intent of a SubMenuManager is to "virtualize"
the "real" menu manager, so it should never directly provide items from its parent.

getItems() has been like this forever (since 2002), but the dispose() is new
(fix for bug 64024 in 3.0).
