Comment 4 Susan McCourt 2006-04-11 20:34:48 EDT 
In looking through the code history, it seems that in bug #61069, the error message text was changed from a label to a read-only text field to address accessibility concerns.  The requirement was that the error message should be able to take focus so that screen reader users could tab to it in order to read it.  

I suppose we could consider disabling the text field when it is empty.

Comment 11 Susan McCourt 2007-03-21 19:19:38 EDT 
Disabling the error text works as desired on Windows.  Still need to check other platforms before providing a fix.  

Note that we would disable the error message text in two cases:
1 ) When the error message is null, it means there is no error.  
  - The OK button is enabled.  
  - We propose to disable the error message text control so that you cannot tab to it, since there is no error.

2) When the error message is set to an empty string, it means that there is an error in the dialog, but no explanatory text provided.  
  - The OK button is disabled
  - We propose to disable the empty error message text control so that you cannot tab to it, since there is no text info provided about the error.

Is this acceptable?
Comment 12 Philipe Mulet 2007-03-22 05:17:21 EDT

Comment 13 Michael Burkhart 2007-03-22 10:10:02 EDT 
Yes, I think the solution proposed in comment 11 is acceptable and should solve the cases we have encountered. I just hope that people don't use empty strings as error messages, it seems confusing that the OK button would be disabled if there is no error message. But I guess it's the dev's responsibility to take care of that situation. Should this be noted this in the javadoc possibly?
Comment 14 Susan McCourt 2007-03-22 12:12:12 EDT 
Yes, this is the developer's decision.  The javadoc for IInputValidator.isValid documents this pretty clearly.  InputDialog.serErrorMessage also touches on this.  I've found cases in the SDK where an empty String or a single space are used this way, so the fix needs to strip whitespace to determine whether there is error text to show.

I think the fix is straightforward, but it needs more testing.  Should have something to report early next week (after our M6 deadline is passed.)
