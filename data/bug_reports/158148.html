Description Hari H Krishna 2006-09-21 10:00:07 EDT 
https://cs.opensource.ibm.com/bugzilla/show_bug.cgi?id=10737

Environment:XP

This is an excessive memory usage problem with the log parsers.  
The extent of the problem will vary from parser to parser based on the parsers 
specific characteristics.  
For example, Import a 50M native DB2 diag log into a 1.5 IBM JVM using LTA 
eclipse, getting an out-of – memory exception. 
This can be successful done using SUN JDK. This is not the JDK problem, but 
need to understand how those implementation is different in both SUN and IBM 
JDK.
Please look at the attached word documentation for detail description and the 
possible solution for this problem.

After fixing the memory usage problem, I can now import that same log into the 
default 500M JVM

TPTP adapters that have been modified 
1. AbstractAccessLogParser
2. AbstractErrorLogParser

We are in the process of measuring the memory utilization, the defect severity shall be modified based on the memory utilization difference between before fix and after fix readings.
