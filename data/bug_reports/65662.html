Description Thomas Whitmore 2004-06-03 22:27:00 EDT 
Using the Outline and Debug views for informational purposes (viewing, not 
navigating) frequently moves the cursor/ file when not wanted.

Moving the cursor/ file away loses the user's work position, breaks the thread 
of thought and is highly unsatisfactory.

(unreadable horizontal tabs, non-usable 'back' button, 10 - 14 files open). 
Finding work position again is excessively expensive and it is not satisfactory 
for this to be moved when not explicitly wanted.

I also frequently note the Debug view while Launching another process. Due to 
some old obsolete process not being 'selected', Eclipse has declined to kill it 
from my Terminate keyboard shortcut. Then I notice it while the other one is 
launching, carefully select the old process to kill it and right-click 
terminate. Just at this time the new process arrives at a breakpoint and 
Eclipse terminates the good one instead.

The Debug tree keeps getting under my feet in this way. If I'm not precise in 
mousing the head of it to terminate, it's opening files I don't want. If I 
didn't carefully select it, it doesn't terminate it. If the other one arrives 
at a BP while I'm terminating, invariably the wrong one gets killed. If I click 
the Debug window in general to see it or kill a process, Eclipse opens the file.

Better behaviour would be for Eclipse to Edit from the debug stack on double-
click or left-click, but not on right-click. And not to move my focus in the 
debug window while I'm terminating. And terminate a single running process if I 
shortcut for that, even if it's not selected. (Which other one did I want to 
terminate?).

Also, I don't need to see EventDispatchThread when an exception occurs. There's 
no useful info there, there's nothing I can do at that point, I would 
appreciate Eclipse not opening more clutter into my working view.

Presumably after I close EventDispatchThread, the 'back' btn will still insist 
on taking me there and re-opening this. This is an example of why the 'back' 
btn is almost completely useless and why I do not use it.
