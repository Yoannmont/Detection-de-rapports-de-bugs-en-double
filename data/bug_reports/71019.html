Description Wayne 2004-07-28 15:45:15 EDT 
In a ToolBar, setting a ToolItem's control to a Button doesn't work in Linux 
GTK.  I was under the impression that it should work, since it worked in 
Windows and adding other SWT controls such as Text worked.

A workaround was to add the button to a Composite first.
Comment 1 Steve Northover 2004-08-04 19:05:41 EDT

Comment 2 Carolyn MacLeod 2004-08-05 17:33:32 EDT 
Doesn't seem to be fixed - in fact, I can't get the suggested workaround to 
work either. Here is the snippet I am using. Before I go any further, Wayne, 
please let me know if this is what you mean. Note that both Button and 
Text "ToolItem-Separators" work on Windows, but as you say, only Text works on 
GTK. But the Button-in-a-Composite workaround does not work on GTK either. Out 
of curiosity, why do you want to use a Button-as-Separator in your ToolBar 
instead of just using a Button ToolItem?

Here's the snippet:

import org.eclipse.swt.SWT;
import org.eclipse.swt.widgets.*;

public class ToolItemControlTest {
	
public static void main(String arguments[]) {
	final Display display = new Display ();
	final Shell shell = new Shell(display);
	shell.setBounds(0, 0, 200, 200);
	ToolBar toolBar = new ToolBar(shell, SWT.BORDER);
	toolBar.setBounds(10, 10, 150, 50);

	ToolItem item1 = new ToolItem(toolBar, SWT.SEPARATOR);
	//Button button = new Button(toolBar, SWT.PUSH);
	//item1.setControl(button);
	Composite composite = new Composite(toolBar, SWT.NONE);
	Button button = new Button(composite, SWT.PUSH);
	item1.setControl(composite);
	button.pack();

	new ToolItem(toolBar, SWT.SEPARATOR); // separate the 2 items

	ToolItem item2 = new ToolItem(toolBar, SWT.SEPARATOR);
	Text text = new Text(toolBar, SWT.BORDER);
	item2.setControl(text);
	text.setText("Type here");
	text.pack();

	shell.open();	

	while (!shell.isDisposed ()) {
		if (!display.readAndDispatch ()) display.sleep ();
	}	
}
}
Comment 3 Wayne 2004-08-06 09:31:03 EDT 
Hi Carolyn, I don't have access to a Linux GTK machine right now, so I cannot 
verify this in the snippet, but I had this line of code in there and it fixed 
the problem for me

>> Composite composite = new Composite(toolBar, SWT.NONE);
composite.setLayout(new GridLayout(1,false)); //line to add
>> Button button = new Button(composite, SWT.PUSH);

Also, I didn't call

  button.pack();
