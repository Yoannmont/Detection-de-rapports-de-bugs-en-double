Description Jamie Liu 2007-05-23 10:37:59 EDT 
The About dialog allows there to be an image on the left and text on the right, given the image has a small enough width.  The height of the dialog is determined by whichever has the greater height.  We can coordinate the size of the image with the amount of text, but the font can change on different platforms or machines, and different languages may require the text to be even larger.  

Could we make the text scrollable, such that the height of the dialog is solely based on the height of the image?  That would give the dialog a more consistent look across platforms, instead of having the dialog size vary based on the font/text size.

The patch as provided is problematic not only because it forces a scrollbar, but also because it makes the dialog non-resizable.  We wouldn't want to change that behavior, as resizing can get the user out of situations like this bug and other about text layout problems.

I'm got several hours into a solution that recreates the styled text with or without the scrollbar based on the computed height.  (based on Boris' experiments in comment 16).  It's not quite working, I'm debugging the layout right now and trying to get rid of unneeded composite layers. Question for Jamie is:  how critical is this, it sounds like you have a workaround in the product right now.  Also, I suspect the behavior is a little different (but maybe not improved) due to the changes we made in 3.6 for bug 292868.  Have you tried it on the 3.6 stream?

Comment 20 Susan McCourt 2011-01-06 19:51:06 EST 
Created attachment 186237 [details]
patch which recreates the styled text as needed

I rather like this patch.
- it introduces a vertical scrollbar when the text won't fit (by creating the styled text on the fly)
- it retains the nice default sizing of the platform SDK about dialog (at least on win7, we would have to double check)
- the resize behavior in general is VASTLY improved.  In the current code, you get horizontal scrollbars rather than wrapping and the resize behavior is quite unpredictable.  It is very predictable now, except for the fact that the text WILL get truncated if the user resizes the dialog to be shorter than the about image (that behavior is true without the patch as well).

To release the patch we would have to verify that:
1 - it solves Jamie's original problem
2 - the scrollbar behavior is working nicely on all platforms.  (Boris had problems on the Mac in his original experiments.  Not sure what it was or if the different characteristics of this layout/dialog would have it.)
3 - the default SDK about dialog size on all platforms should remain appropriate (no scrollbar initially)

If the above could get verified, I would be in favor of releasing into 3.7 immediately and I might even push for 3.6.2 given how much better the resize behavior in general is.
