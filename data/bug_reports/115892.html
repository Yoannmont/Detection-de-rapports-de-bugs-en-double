Description John Pitman 2005-11-10 14:42:14 EST 
In the add/remove facets dialog, if I have a facet which depends on a second
facet, and I select the first facet, the second facet should automatically get
selected.  Right now the user has to read the error message, then manually
select the second facet.

Comment 11 Jim Zhang 2005-12-06 10:33:58 EST 
I think there's a lazy way on the framework's part to provide a minimum solution, which is basically returning a tree structure.  This would give a truthful representation of the full permutation of the possibilities.  Then it's up to the client to traverse it and resolve the most appropriate list.

I think in most cases the tree would actually be just a trunk without any branches because from what I've seen <AND> is used most of the time.  Note that to accommodate the multiple <AND> operands each tree node would be IProjectFacet[].

Other than providing traversing API on this tree data structure, two other APIs can help: 
- getFacetsAsSet(): this would return all the facets contained in the tree nodes as a Set.  In branching situations one branch can be picked arbitrarily.  Clients can use the next API to check whether it should call this method or traverse the tree themselves.
- hasBranches(): this tells the client whether it can simply call "getFacetsAsSet()", in the case that there's no branch on the tree, or it should traverse the tree, picking the most appropriate branch themselves.

If this is reasonable for the first step then it can be done without API breakage or UI changes.

How does that sound?
Comment 12 Konstantin Komissarchik 2005-12-06 10:46:49 EST 
Jim,

Unfortunately, it's way too late in this release to be talking about implementing enhancements like these. The best way for you to procede on this may be to try to implement this in your code based on the framework api that's currently available. In the 1.5 release we will have time to properly consider all the corner cases and implement a viable solution. Note that I believe this is outside the scope of the 1.0.1 release as well, based on the definition of that release.
