implements some plugins. Most of these widgets have an internal API so they are
not considered as reusable. Here is a list of widgets that we found useful.

Comment 3 Darin Swanson 2004-11-08 18:33:38 EST 
It would be great to revisit this bug report and push down some of the most

Comment 4 Dirk Baeumer 2004-11-09 08:38:19 EST 
The problem is that JDT/UI isn't the right layer to make things like

CC Martin and Nick for comments.
Comment 5 Nick Edgar 2004-11-09 11:22:39 EST 
While better support for databinding and forms support is an area that I think
is important (FieldEditors don't cut it), I don't see us simply pushing down the
DialogField framework.  I'd prefer for us to step back and do something more
along the lines of the databinding support provided in frameworks like XUL,
JDNC, WinForms and Essential Data.
Comment 6 Dirk Baeumer 2004-12-13 18:05:08 EST 
No action planned in JDT/UI.
Comment 7 Sébastien Gandon 2006-10-17 11:56:49 EDT 
I reopen this bug as these APIs are really handy to create wizard blocks and editors pages, these APIs should really be promoted to the platform project.

In my case I need to create a Java project wizard very close to the JDT java project wizard and I have to do it all myself not to be using internal packages.
This is just a pain.

Please could you reconsider, at least for the future developpers, I do not expect it for myself.

SeB.
Comment 8 Martin Aeschlimann 2006-10-17 12:29:00 EDT 
The list contains various things: dialogfields would be something that has to offered by platform.ui. Please file them a bug again (there are already several in this direction).

For all it can be said that they
- do not belong to jdt.ui to be made API
- are not API ready. API classes need to following name conventions, be minimal, maintainable ect.

I understand your frustration, but I want to keep this bug closed and suggest to file specific requests against platform.ui.
