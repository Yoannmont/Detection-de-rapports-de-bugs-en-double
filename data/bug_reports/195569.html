Comment 19 Chris Mead 2007-09-05 07:30:38 EDT 
I think I've found the cause of this (or certainly an irregularity):

org.eclipse.cdt.managedbuilder.internal.dataprovider.BuildEntryStorage line 360

This reads: entries[i] = (ICLanguageSettingEntry)CDataUtil.createEntry(kind, paths.toString(), null, null, flags);

Surely this should be paths[i].toString() ??

This would explain the Strings of a format equivalent to the output of an  Object.toString() invocation as found here.

I personally find this if I define <envVarBuildPath> entries that refer to environment variables for a specific tool.
