Description Rob Frost 2006-06-29 15:08:31 EDT 
Covers a rework of the WLS generic servers to use the wldeploy Ant task, fully exploded EARs and deployment only when changes have occurred in the relevant modules.

This will require an extension of the generic AntPublisher and customization of the default generic server assembly logic.
Comment 1 Rob Frost 2006-06-30 17:58:08 EDT 
Created attachment 45628 [details]
Patch for org.eclipse.jst.server.generic.weblogic plugin

This patch makes the following changes:

-Implements a custom extension of AntPublisher that checks module publish delta, updates child module publish state and overrides standard assembly logic to create fully exploded deployments.
-Changes Ant build script to call wldeploy rather than create/copy archives to autodeploy directory.
-Alters server instance properties: autodeploy dir is no longer needed, username and password are required for wldeploy
-Changes server classpath from api.jar to api.jar and wls-api.jar (available in WLS 9.2)

Comment 7 Rob Frost 2006-07-07 17:15:33 EDT 
Created attachment 45969 [details]
Patch for org.eclipse.jst.server.generic.weblogic plugin

The changes are now fully backward compatible with existing WLS server instances and deployments:

-instead of asking for username/password during server instance config (and saving these in plain text in the servers.xml file for later use with wldeploy), the publish logic now retrieves the username/password from an encrypted user configuration/key file pair (this file pair is dynamically created if necessary using a username/password supplied by the user via the Ant "input" task).

-since the module name for auto-deployed modules will differ from that used now with wldeploy, it is necessary to ensure that any previously auto-deployed archives for that module are deleted from the auto-deploy dir (and thus undeployed from the domain)
