Comment 31 Duong Nguyen 2008-07-17 16:28:38 EDT 
Another observation that I'm found that explains the retry code is that we're trying to get the clipboard (and obtain a lock for it) but the clipboard is currently being used (locked) by some clipboard manager (eg. the one used by Excel or Klipper). We have to retry (usually only once or twice) before we can lock the clipboard for our own use.

Comment 34 Duong Nguyen 2008-07-17 18:31:35 EDT

Clipboard set/get example

Comment 35 Duong Nguyen 2008-07-17 18:50:37 EDT 
Here's another interesting finding. If you have the Excel clipboard open and you copy only the RTF string into the clipboard, you will get an error message from the Excel clipboard manager -- "Format not supported by Office Clipboard" and you will not be able to paste anything after that. It appears that the Office Clipboard clears the contents clipboard if it sees a format that it does not support. Change the above attached example by renaming the TextTransfer to RTFTransfer and you should see this error message and you'll be unable to paste.

The ClipboardOperationAction will try to get the contents of Clipboard (TextTransfer and RTFTransfer) but if the textData failed to be retrieved, it will only add the RTFTransfer back to the clipboard.

	Object textData= clipboard.getContents(TextTransfer.getInstance());
	Object rtfData= clipboard.getContents(RTFTransfer.getInstance());
	........
	if (datas.isEmpty())
		return;

When this happens, you will see the error message from the Office clipboard and the copy/paste will fail. I suggest that the ClipboardOperationAction be changed so that it would bail if the textData is null.

As indicated in the previous example, it's possible that getContents return null even when you just put something in the clipboard due to the clipboard being locked. SWT will try to prevent the occurrence of the null/lock problem but ClipboardOperationAction should be fixed too.

Comment 70 c hoekstra 2012-08-14 11:51:12 EDT 
I was able to fix this in Websphere Rad with this process: 

Window > preferences > Keys

scroll down to Copy (with binding Ctrl+C).

Change the 'when' column from 'In Dialogs and Windows' to anything else. Apply and save. 

Go back into Keys. 

scroll down to Copy (with binding Ctrl+C).

Change the 'when' column back to 'In Dialogs and Windows'. Apply and save.

(I'm guessing this rebinds/resets the key mappings.)

Repeat for Paste, Cut, or any other key that is no longer functioning.
