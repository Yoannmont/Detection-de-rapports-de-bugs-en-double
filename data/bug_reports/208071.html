Comment 1 Alain Lee 2007-10-31 12:29:33 EDT 
Created attachment 81734 [details]
Changes in Rendering.java

I am proposing the following changes in Rendering.java:
1. If the request for the same block is being processed, do not add the request
to the queue.
2. We are only interested in the cache corresponding to the last request in the
queue even though the queue has multiple requests waiting. Therefore, skip all
requests that we are no longer interested in? This can save the unnecessary
data read from the target and the unnecessary screen redraws. 
3. Change populateCache() to run redrawPane() synchronously. This ensures that
redraw is completed when populateCache() is done. This also guarantees that
only one populateCache() runs at a time.
4. When a request is being processed, mark it as "In Progress". Remove the
request from the queue when populateCache() is done.
