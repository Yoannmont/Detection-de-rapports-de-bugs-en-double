Comment 12 Wassim Melhem 2005-02-17 10:06:19 EST 
You don't see the product information because you are not launching in product 
mode.
Choose your favorite product on the main page of the Eclipse app launch config 
and launch.

we discussed this with Nick (bug 85511), as I was seriously contemplating 
changing the default setting for the Eclipse app launcher because of this.
I'm sure there is new behavioral changes though on the runtime side as I'm 
certain I used to get intro up even when I launch with an application.
Comment 13 Tod Creasey 2005-02-17 10:10:16 EST 
I understand that but this is a significant change that no one knew about - 
self hosting looks wrong and we dug through preference code until we happened 
to find Nick who had some background.

You should at least announce this change on eclipse-dev. Better yet select the 
product in your launch config.
Comment 14 Wassim Melhem 2005-02-17 10:18:53 EST 
Tod, is comment 13 directed at me?  If so, then you misunderstood what I 
said.  
No changes in PDE have been made in this area whatsoever.
We have always launched in application mode, not in product mode as default.  

Not sure why we're seeing new behavior (perhaps someone from Runtime has 
insight), but we are seeing it.  As a result of this, I thought it would be a 
good idea to change the launch config defaults now to launch in product mode 
(bug 85511), so that users don't get startled.  But I haven't acted on it yet.
Comment 15 Tod Creasey 2005-02-17 10:23:51 EST 
Yes it was - if this is a bug in the runtime rather than a PDe change we

Comment 16 Pascal Rapicault 2005-02-17 11:18:38 EST 
This is not a bug, but rather an improvement.
Runtime was always defaulting eclipse.product to be org.eclipse.platform.ide,
which was causing problems for RCP app that were not defining a product.

The creation of new application launcher should by default pick the product.
Note as well that it should be possible to change the application even the
product has been picked since they are orthogonal concepts.
