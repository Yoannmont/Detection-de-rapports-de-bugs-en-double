Description Litrik De Roy 2006-04-20 07:45:02 EDT 
Originally reported as comment 1 of bug 135461:

When refreshing the Synchornize view using the icon in the view's toolbar (or using F5) I get a modal progress dialog with a height occupying the full screen.

In the old dialog (3.1) you could click "Run in background". This is no longer possible and impacts developer productivity.
Comment 1 Michael Valenta 2006-04-20 08:55:44 EDT 
It works fine for me. Perhaps you can tell use what build you are using. Also, could you attach a screenshot of the failure?

Screenshot showing the modal dialog

The attached screenshot shows the dialog that appears after pressing F5 in the Synchronize view. Note the height of the dialog.

This happens on a WinXP machine using Java 1.4.2 with Eclipse 3.2 RC1.

There are over 300 plugins in my workspace. Maybe this triggers the behavior.

PS: I can resize the dialog, but next time it reopens to the size shown in the screenshot.
Comment 3 Michael Valenta 2006-04-20 09:15:44 EDT

Comment 10 Tod Creasey 2006-04-20 20:54:37 EDT 
Generally the idea is good Mike but I am not sure why you have all of the disposed checks in the show callback - is it possible to get this callback on a disposed shell?

Even if it is I think setMessage has that covered. I think

public void handleEvent(Event event) {
				if (messageLabel!= null && !shell.isDisposed() && shell.isVisible()) {
					if (message != null)
						setMessage(message);
				}
			}

could be 

public void handleEvent(Event event) {
		setMessage(message);
}

The only other concern is that this gets set twice but I think it is fine if the show callback only happens on the initial open.
