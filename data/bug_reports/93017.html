Description Jeffrey Forbes 2005-04-27 17:14:42 EDT 
Bug found in 10.4.0 (Tiger), on a Powerbook G4 Titanium. Happens on all versions tried since 3.0.2 (didn't 
try < 3.0.2)

Use the key combination or click on the "Spotlight" icon in the upper right, and start Eclipse. 
This hangs the UIManager process, which provides access to all of the menu tools such as the Clock, 
Power Meter, Volume, and Airport status.

Exiting eclipse will un-hang the process. Couldn't find any other workaround.

Comment 17 Scott Kovatch 2005-10-12 22:03:55 EDT 
Our apologies on this one -- it was marked as a duplicate of a bug marked third party to fix, but no one 
thought to tell the reporter of the original bug what the problem was.

Spotlight is blocked because Eclipse is not responding to the open-doc (odoc) AppleEvent being sent to it. 
The native launcher of Eclipse needs an AppleEvent handler. I haven't looked at the launcher code in 
Eclipse to see if this is feasible.

Comment 24 Andre Weinand 2005-11-15 19:08:36 EST 
Thanks Scott for providing the details of the problem.

I still think this is an MacOS problem, because a user level application should not be able to stop a 
system level service (Spotlight) from working properly.

However, I have a simple workaround for the Eclipse launcher: running an application event loop for a 
short amount of time handles the open-doc (odoc) AppleEvent and prevents Spotlight from blocking.

Since the Eclipse launcher lives in SWT land, I'm moving this bug to SWT.

Silenio, could you please verify my patch. Thanks.
