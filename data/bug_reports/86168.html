Comment 20 Markus Keller 2010-03-29 09:56:47 EDT 
(In reply to comment #19)
> I can't believe this has been sitting here for 5 years with "no strong
> opinion".

That's not the only comment in this bug, right? E.g. comment 12 says:
> use the New File wizard to create a file called package-info.java

That's still our take. It's overkill to create and maintain another wizard just for this, and it would pollute the UI with a rarely-used command. We could add a checkbox "Create package documentation" in the New Package wizard to create the package-info.java or package.html.

If someone wants to write a patch, please make sure
- the checkbox is only enabled if no documentation exists for the current package
- the checkbox enablement is dynamically updated when the package name or the source folder is changed
- the Finish button is enabled in case the checkbox is enabled and checked
- the checkbox state is remembered (see NewClassWizardPage for how to do this)
- the generated package-info.java starts with the file comment template, followed by a Javadoc comment

Furthermore, we'd also need a code template for the generated package Javadoc. Caveats:
- API freeze is over, so this has to wait until 3.7
- When the code template is empty (really empty, not just an empty comment), we still have to generate an empty Javadoc comment if the file comment template would otherwise mutate into a Javadoc for the package declaration.

For source level < 1.5, the text from the new template should be extracted and put into the generated package.html.

Comment 26 Philippe Marschall 2012-04-13 11:50:37 EDT 
Created attachment 213986 [details]
patch against jdt.ui

This patch adds a "Create package documentation" checkbox to the "New Package" wizard.

If the source level is 1.4 or below a package.html is created, otherwise a package-info.java is created.

The package-info.java contains the file and type comments followed by the package declaration. The file is formatted using K_COMPILATION_UNIT code formatter kind.

Whether a package documentation should be created is saved using the same mechanism as the "New Class" wizard (the working one from M7). The default is false.

In the case of a package.html the type comment (minus // /* */) is used as a body of the HTML document. The file comment is added as an HTML comment. The package name is used as a document title.

Comment 34 Markus Keller 2012-04-16 11:20:37 EDT 
Thanks a lot, nice work. I've released the jdt.ui patch with a few tweaks:

- open the generated package-info.java file (I don't want to open the package.html, since will open in a browser, depending on the installed plug-ins and preferences)
- use CodeGeneration to create stubs. That also ensures that all variables are correctly replaced, including e.g. @tags.
- use JavaDoc2HTMLTextReader and JavaDocCommentReader to convert the Javadoc to HTML. I admit I had to fix a line delimiter bug first ;-). The advantage is that we don't reinvent the wheel and that tags in the templates are rendered nicer (e.g. @author).
