Description Stefan Knieling 2005-11-29 15:31:10 EST 
Overview Description:

Setting property IDeviceRenderer.CACHED_IMAGE does not store the image in the specified BufferedImage correctly. The image is incomplete, i.e. it is cut-off. If I leave out the line where I set CACHED_IMAGE the image is created correctly as I can see looking at the file stored on disk.

Steps to Reproduce:

1) Create a chart, here it was a bar chart with depth.
2) Generate an image from the chart with the following code, set 
   image width=300, height=300:

final PluginSettings ps = PluginSettings.instance();
  IDeviceRenderer idr = null;

  try {
   idr = ps.getDevice("dv.PNG");
  } catch (ChartException ex) {
   ex.printStackTrace();
  }
  BufferedImage bufferedImage = new BufferedImage(width, height, 
BufferedImage.TYPE_INT_ARGB);
  Graphics g = bufferedImage.getGraphics();
  Graphics2D g2d = (Graphics2D) g;
  idr.setProperty(IDeviceRenderer.GRAPHICS_CONTEXT, g2d);
  Bounds bo = BoundsImpl.create(0, 0, width, height); // width=300, 
height=300
  idr.setProperty(IDeviceRenderer.EXPECTED_BOUNDS, bo);
  idr.setProperty(IDeviceRenderer.FILE_IDENTIFIER, "E:\\1.png");
  idr.setProperty(IDeviceRenderer.CACHED_IMAGE, bufferedImage);

  Generator generator = Generator.instance();
  GeneratedChartState state = null;

  try {
   IDisplayServer server = idr.getDisplayServer();
   state =  generator.build(server, chart, null, bo, null);
   generator.render(idr, state);
  } catch (ChartException e) {
   e.printStackTrace();
  }

Expected Results: 

I actually want to have the generated image stored in the BufferedImage instance
so that I can further process it in my application, i.e. converting it to an SWT image and displaying it in an Eclipse View.
