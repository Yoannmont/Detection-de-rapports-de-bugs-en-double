Description Dejan Glozic 2005-01-31 22:15:18 EST 
This feature is opened to track the request to expose the newly created Help 
search facility in the UI:

1) A new menu item should be added - Help>Search. This item should call the 
new API in IHelpSystem - showSearch().

2) The action for the menu item should be made available as a factory so that 
RCP apps can add it to their Help menus if they want. It should be added to 
Eclipse IDE.

3) A combo box should be added to the cool bar by default. The combo box 
should be editable and have the initial text 'Search Help'. When text is 
entered and 'Enter' is typed, it should call 'IHelpSystem.search(String)'.

4) The last N search expressions should be remembered and shown in the combo 
box list. Selecting the item from the list should be equivalent to typing it 
and pressing 'Enter'.

5) The class 'AbstractHelpSystem' should be modified to include 'showSearch()' 
and 'search(String)' APIs. For backward compatibility, they should have no-op 
implementation (not be abstract), otherwise 3.0-based implementors will be 
broken when moving to 3.1.

Attached is the proposed initial implementation - feel free to make any 
modification necessary.

Comment 7 Nick Edgar 2005-02-11 22:03:39 EST 
Patch applied with the following changes:
- adapted to the workbench advisor refactoring
- changed '\r' to SWT.CR
- moved combo from ActionFactory to ContributionItemFactory
- added dispose() for combo in WorkenchActionBuilder.dispose()
- added "help" to combo tooltip: "Enter help search expression and press Enter"

Note that the tooltip only appears when over the drop-down or the border of the
combo, not the text field within it.  This may be an SWT or OS bug (I'm on Win2K).

Also, I suggest changing the search history to put new items at the top rather
than the bottom.  This makes more recent items closer to pointer, and is also
consistent with the way the Search view works.
If you like, I can make this change.

- added an elipsis to the action's text, since it takes input before the search
is complete
- added the "Help" suffix, since I feel it's needed to distinguish this action
from the other non-help actions in the Help menu
- changed the tooltip to Search Help instead of Help Search (verbifying it)
