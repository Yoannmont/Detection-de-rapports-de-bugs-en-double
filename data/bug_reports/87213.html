Comment 12 Billy Biggs 2006-01-15 04:11:12 EST 
SWT is implemented on Linux using GTK+ 2.  The GTK+ 2.0 series (June 2002)
was the first release series of GTK+ 2 and was rather buggy and did not appear
in many Linux distributions.  Version 2.2 (February 2003) was both more stable
and more prolific, and is used in Red Hat Enterprise Linux 3, their first enterprise
release that provides GTK+ 2.

Comment 14 Grant Gayed 2006-02-17 16:54:24 EST 
fixed > 0217; note that this is not in the pending 3.2M5 release

While I'm somewhat opposed to this change since the problem's root cause is an invalid user installation, it would be unfortunate for eclipse to not work solely because a user that does not require accessibility facilities does not have the right accessibility library.  atk_object_add_relationship is the only utilized atk function call that is not in atk 1.0 (outside of the accessibility package of course), so this should be the only function that needs a dynamic link.

Comment 17 Billy Biggs 2006-02-20 23:58:53 EST 
Note that Fedora Linux is not even supported by Red Hat, it's a community-supported
distribution that they basically use to test their enterprise releases.  Experimental
patches have been known to go into Fedora to see what applications break ;)

That said, I think it's reasonable to expect that the Fedora developers wouldn't
intentionally break Eclipse, and that we'll of course make sure we work on Fedora
Linux as much as possible.  However, given the nature of Fedora I do not think
it would be a good idea to "officially" support it.
