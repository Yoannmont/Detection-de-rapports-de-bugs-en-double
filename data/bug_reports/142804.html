Comment 10 Kahli Burke 2006-05-23 15:45:18 EDT 
I have a colleague who just purchased a 17" MacBook Pro with the 2.16 Core Duo CPU.  He is experiencing the same problems described below.  He is using 3.2RC5 though we've tried RC 3 and 4 as well.  He has installed the Apple Java 5.0 release 4 update.

It seems that actions that need to access the compiler or installed JREs will cause the system to lock up completely.  These actions would include creating a new Java project, importing an existing Java project, or trying to view or configure the list of installed JREs in the Eclipse preferences.

Since just clicking on the control for the installed JREs (Window/Preferences/Java/Installed JREs) will immediately lock up the entire OS, such that the computer must be forcefully powered off and restarted, I wonder if this in some way has to do with scanning for the available JVMs in the system.

Comment 11 Jeremy Adams 2006-05-23 16:01:43 EDT 
As a work around (note, this is not a fix... just a work-around) for those who absolutely MUST get eclipse running right away, following these steps:

1. Reinstall Mac OSX (yes, a complete reinstall). You'll want to choose the option to erase the disk when you do this.

2. Right after the installation is finished (and before you install ANY updates), install Eclipse.

3. Run Eclipse and create a new dummy java project.

4. Go ahead and install all the system updates.

I followed those steps and have been able to run Eclipse successfully on my 15" Macbook Pro. I know this is a terrible thing to have to go through to get Eclipse working, but if you rely on Eclipse as much as I do, it's worth it. I hope the Eclipse project group can come up with an actual fix though, as I'm guessing they're going to start seeing this issue more and more from here on out.
