Hi Bob,

The TypeComparator, which is the class that validates expected types vs. calculated types has six different diagnostics that it generates that are not currently tunable by severity:

Expected type mismatch on value binding (the one causing your problem)
Parameter count mismatch on method binding
Parameter type mismatch on method binding
Return type mismatch on method binding. 
Expected readable value binding (i.e. no bean getter)
Expected writable value binding (i.e. no bean setter)


We will provide the same workspace preference settings for these as for EL validation: they can be set to Ignore, Warning or Error.  Ignore causes them not be reported.  

Unfortunately we cannot default any of these to ignore (as was your desire) since we are not permitted to add UI in 2.0.1 that would permit the user to turn them on, however we will drop the one affecting you to Warning, which is where it probably should have been to start with.

As Raghu has stated, we will improve the mechanism using your suggestions in https://bugs.eclipse.org/bugs/show_bug.cgi?id=202697 as the base.
