Comment 8 Wassim Melhem 2004-05-26 10:03:54 EDT 
PDE is not a psychic, we cannot foretell which zips will compile and which 
won't.  But after complaints from users saying that their importing experience 
is not as pleasant as it ought to be because the code being imported does not 
compile, we (kicking and screaming) compiled a naughty list:
org.apache.ant
org.osgi.util 
org.osgi.services

We hate to do that, but that stopped the bug reports from coming in.

Another nasty case is the swt plug-in.  On platforms where swt fragments 
contain JARs (e.g. on gtk and mac, etc..), the code in the fragment references 
the code in the plug-in and vice versa, causing us to create circular 
dependencies among projects which causes headaches when building the workspace.
Therefore, swt is always imported as binary.  If you want to see it in source 
form, check it out of cvs, that project will contain all the code in one plug-
in, and will not be split across plug-in/fragments.
