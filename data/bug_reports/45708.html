Comment 8 Jörg von Frantzius 2005-07-22 06:17:46 EDT 
(In reply to comment #0)
Using Eclipse 3.1 GA, I'm still seeing this bug, I believe. I debugged into it a
bit, and it turns out that Table.releaseWidget() is called twice after a single
call to Workbench.close(). In Table.releaseWidget(), the items field is first
accessed and then nulled out, so on the second call an NPE will be thrown.

If you compare the two stack traces that I'll attach in a minute in Eclipse,
you'll see that WorkbenchWindow.hardClose() first closes all pages and then
closes the window. Both calls result in Table.releaseWidget() being called,
which seems not to be guarded from running twice. 

Maybe there is a more thorough solution to this problem, but my proposal would
be to simply guard Table.releaseWidget() from running twice.

Comment 19 Carolyn MacLeod 2005-08-03 13:30:16 EDT 
We have come up with a general solution, which we will add to the 3.2 release.

However, this still leaves Jorg with an NPE in Table in 3.1 because someone is 
calling releaseWidget twice. So we will put the null check back in Table 
(only) for 3.1.1.

For testing the general solution: The simplest test is to call dispose from 
inside a dispose listener. Here's a snippet for Table. To see the (3.1) 
failure described in comment 8, run it, and close the window. To see the 
failure described in comment 0, change MAX_COL to 1, run, close window. Our 
3.1.1 "null-check" fix will cover both of these cases.


import org.eclipse.swt.*;
import org.eclipse.swt.widgets.*;
import org.eclipse.swt.layout.*;
import org.eclipse.swt.events.*;

public class TableDisposeTest {
	static Display display;
	static Shell shell;
	static Table table;
	static int MAX_COL = 3;
	
	public static void main(String[] args) {
		display = new Display();
		shell = new Shell(display);
		shell.setLayout(new GridLayout());
		shell.setText("Table Dispose");
		
		table = new Table(shell, SWT.MULTI | SWT.FULL_SELECTION);
		table.setLayoutData(new GridData(GridData.FILL_BOTH));
		table.setHeaderVisible(true);
		for (int col = 0; col < MAX_COL; col++) {
			TableColumn column = new TableColumn(table, SWT.NONE);
			column.setText("Column " + col);
			column.setWidth(100);
		}
		for (int row = 0; row < 5; row++) {
			TableItem item = new TableItem(table, SWT.NONE);
			for (int col = 0; col < MAX_COL; col++) {
				item.setText(col, "item" + col + row);
			}
		}

		table.addDisposeListener(new DisposeListener() {
			public void widgetDisposed(DisposeEvent e) {
				table.dispose();
			}
		});

		shell.pack();
		shell.open();
		while (!shell.isDisposed()) {
			if (!display.readAndDispatch()) display.sleep();
		}
	}
}

Comment 30 Jörg von Frantzius 2005-08-10 13:11:19 EDT 
You guys (and girls) did a good job making people not able to override
releaseWidget() etc., it's all package-protected... 
I'm happy to have found a better workaround than System.exit() which only works
for the last window ;)
Actually I'm so happy about it that I'll post it here in case anyone needs it as
well. Setting all Table's itemCount to 0 does the trick. So in my
WorkbenchWindowAdvisor, I'm doing this:

private void setAnyTableItemCountToZero(Composite parent) {
	Control[] children = parent.getChildren();
	for (int i = 0; i < children.length; i++) {
		if (children[i] instanceof Table){
			((Table)children[i]).setItemCount(0);
		} else if (children[i] instanceof Composite) {
			setAnyTableItemCountToZero((Composite)children[i]);
		} 
	}
}

public boolean preWindowShellClose() {
	if (super.preWindowShellClose()) {
		IWorkbenchPage[] pages = window.getPages();
		for (int i = 0; i < pages.length; i++) {
			IViewReference[] viewRefs = pages[i].getViewReferences();
			for (int j = 0; j < viewRefs.length; j++) {
				IViewPart viewPart = viewRefs[j].getView(false);
				if (viewPart!=null && (viewPart instanceof EditableView)) {
					Composite composite = ((EditableView)viewPart).getParentComposite();
					setAnyTableItemCountToZero(composite);
				}
			}
		}
	}
	return true;
}

EditableView is my ViewPart subclass that keeps track of its parent Composite.
