Comment 5 Andrée Proulx 2004-05-04 10:30:46 EDT 
Thank you for your feedbacks. 

Be assured our intent is not to confuse people, nor to change metaphor for the 
sake of it. We are very attentive to your comments.
We all liked the running. We also understand the distinction between the run 
and debug mode.

In this release we are tackling metaphors which have been tracked as issues 
since the inception of eclipse icons.
When we did the V1.0 icons for eclipse we debated to remove him, because in ui 
design a general direction is to avoid as much as possible the representation 
of body parts. 

We also received valid comments from Information Developers reminding us that 
this illustration was used to represent exit. Exit in the IDE context is 
similar to close.

Also we noted all other competitive IDEs (Sun - NetBean, BEA - WebLogic, MS - 
VisualStudio) are using the green arrow. For globalization purposes we should 
consider the use of the green arrow.

We understand the distinction between the run and debug mode. We understand 
that resume and re-run are strong associated concepts to debug and run.

We are working at making a discernable and satisfactory visual distinction 
between these concepts. Based on the above criteria it is not possible to bring 
back the running man. We hope these explanations clarify the issue.

Comment 11 Mike Beltzner 2004-05-06 11:37:28 EDT 
Created attachment 10347 [details]
UK, European Exit Signs

I've attached an image of emergency exit signs that are commonly found in the
UK and the EU. These are the broad versions of the signs, which include extra
markup (usually an arrow and the type of exit, either door or stair) that add
information. Square versions of the signs are simply the "running man" on a
background that implies an open door.

By removing the long-serving "Running Man", I think the design team is trying
to be a good global citizen, realizing that for a large portion of the world, a
running man icon is taken to represent quick exit. While it's not an onerous
task to adjust one's expectations and realize that a running man might also
mean "Run", I can see why we wouldn't want to overload the iconography if it
can be easily avoided and replaced with a more universally understood symbol.

I do want to note, however, that the original bug poster's concerns are
warranted: if this new "Run" symbol causes collisions with existing icons for
"Resume", and especially if those icons are in close proximity on the GUI, then
some work should be done to sharpen the differences.

Comment 17 Jared Burns 2004-05-06 13:55:39 EDT 
*ahem* Asserting that there's emotions involved here is insulting. Please refrain from using non-technical 
arguments. The "few" in this case are those (you) who want the play icon. As I explained, we've received 
zero complaints about the running man looking like "exit" and multiple complaints about the play icon. 
That's a hard fact supported by evidence in the form of bug reports and comments. "If it ain't broke, 
don't fix it"

As a Platform-Debug committer, I know that one of our long-standing trouble-areas with users is making 
the distinction between Run mode and Debug mode. Our existing icon made that distinction. The new 
icon loses that important distinction. As an added bonus, it also overloads the meaning of the "play" 
arrow in the Eclipse debugger.

Running man pros:
1. Visual distinction from "debug"
2. Visual distinction from "resume"
3. Eclipse users know this icon and look for it to launch.

Running man cons:
1. Looks like "exit" (no known bug reports support this claim)
2. Has body parts? (like the debug icon)

Play pros:
1. Has no body parts?

Play cons:
1. Loses "run/debug' distinction
2. Too similar to "resume"
3. New icon that users have to learn
4. Has no body parts. This makes it inconsistent with the Debug icon, which does have them.

If there's something I'm missing here, please tell me.
