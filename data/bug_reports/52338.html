Description Uwe Kubosch 2004-02-18 02:37:55 EST 
When doing a local search-replace in one file with regular expressions turned on 
with

Search: "; "
Replace "\n"

Search: "; "

I would expect the first one to insert a newline, or the second if some Java 
Stringliteral logic has seeped into the dialoge.

Comment 12 Tom Hofmann 2006-03-10 02:31:34 EST 
Agree - this bites me once in a while, too.

The problem is that we have to provide a fix on top of the Java regex support - from bug 85952 comment 0:

>the java.util.regex package cannot handle special characters in the replace
>string. You can search and replace Tab by using the normal (non-regex)
>search and pasting the Tab into the find and replace entry fields. You can
>also file a bug report so that we can either implement this on top of
>java.util.regex or at least remove it from the "Replace With:" field's
>content assist list.

Those reporters that try to *replace* a line delimiter should also be aware of bug 130843, where the entire document structure gets corrupted.
