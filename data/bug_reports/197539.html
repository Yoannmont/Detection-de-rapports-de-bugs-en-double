Hi Robert,
we are near to the resolution.

Now, the wrapping is done, but, when there are long lines without spaces, Bugzilla does not "cut"/wrap them, while now Mylyn does. For instance, suppose you have written the following string in your comment:

com.cardinis.cardinis.app.bpc.gui.model.mixedtreetable.MultiprojectMixedTreeTableModel

This string does not have any spaces in it, so Bugzilla does not know how to wrap it: therefore, it doesn't wrap it at all. In other words, there can be some lines longer than the other lines, but only if they contain a string so long that it can't be wrapped.

With the latest weekly build, Mylyn is doing the following:

com.cardinis.cardinis.app.bpc.gui.model.mixedtreetable.MultiprojectMixedTreeTabl
Model

So, the long string is "truncated" and wrapped. Please also note that Mylyn is "eating" one character (the "e" of "Table" does not appear either at the end of the first line or at the beginning of the second line).
