Comment 5 Lynne Kues 2002-01-23 17:29:04 EST 
Tried to recreate with 20020123 build on Linux-Motif.  Could not.  The error is 
very strange.  Based on the walkback, word-wrap would have to be set for the

Comment 6 Knut Radloff 2002-01-23 19:56:33 EST 
SWT.WRAP is specified for StyledText in the following two places:
org.eclipse.help.internal.ui.ContextHelpDialog.createInfoArea
org.eclipse.ant.internal.ui.AntConsole.createPartControl

If you open context help (F1) anywhere you will see that the space where the 
StyledText widget should be is not drawn. There is no log message indicating a 
walkback. However, if you click in the StyledText widget you get an internal 
error with the attached stack trace in the log. ContextHelpDialog defines its 
own wrapping StyledTextContent. This is where the exception occurs. 

The Ant Console however does not do its own line wrapping.
James, do you remember having the Ant console or the context help open when you 
got the walkback?

Comment 18 Knut Radloff 2002-01-25 16:30:31 EST 
Checked write references to caretLine, which is used in 
StyledText.wordWrapResize to call WrappedContent.getLine. I don't see how it 
could get out of range (the exception indicates it was one line past the last 
line) other than WrappedContent having an inconsistent visualLineCount. The 
exception James got could happen if the visualLineCount is greater than the 
actual number of visual (wrapped) lines. Likewise I didn't find anything 
suspicious with regard to the visualLineCount.
