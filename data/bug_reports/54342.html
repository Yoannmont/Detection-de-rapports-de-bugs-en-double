Description Michael Van Meekeren 2004-03-10 12:58:41 EST 
build I20040309

- open an editor (such that there is no available drop down editor list)
- press Ctrl+E (or command+e on mac)
Note that the "Switch to Editor" dialog is opened however nothing should 
happen in this case
- open more editors so that the drop down editor list is available off the 
chevron image on the right of the editor area
- press Ctrl+E
NOW the list should drop down allowing you to type in a file name and hit 
enter to activate that editor
Comment 1 Chris McLaren 2004-03-10 13:36:32 EST 
Do we still want the 'switch to editor' dialog around? ActionFactory.SHOW_OPEN_EDITORS is public API 
which *currently* creates that dialog. As such, I'm not sure if its OK or not to change the 
'org.eclipse.ui.switchToEditors' command to open the drop down. I will create a new command 
'org.eclipse.ui.window.openEditorDropDown'. 

Another question: when a view is active in a stack where not all view tabs are visible, do we want this 
key to open the drop down of that view stack? 

If so, this new command should be called 'open drop down' (or 'show all items', etc.). 

If not, we could define a command 'org.eclipse.ui.windows.openViewDropDown' and perhaps take the 
variant keybinding Ctrl+Shift+E.
Comment 2 Michael Van Meekeren 2004-03-10 14:14:16 EST 
I suggest to start with adding the new action for open editor.  I think open 
view drop down has other issues, like which set of views, do I have to give 
focus to the view stack first and if so I am part way to clicking on the drop 
down list anyway.

Since the action for Switch to editor is public, would it be acceptable to 
change it's meaning in this case and remove the mostly redundant dialog.  
Especially if we add a close menu on the list

Comment 8 Chris McLaren 2004-03-21 18:57:09 EST 
- i have implemented a command 'Open Editor Drop Down' which opens the editor's
drop down list. 

- it is assigned a default keybinding of M1+E (Ctrl+E on most platforms,
Command+E on Mac).

- the drop down currently opens at 0,0 of the tabfolder's client area. see what
you think.. we could move it

- the switch to editor dialog is still around, and it has keybinding M1+M2+E
(Ctrl+Shift+E on most platforms, Command+Shift+E on the Mac).

- due to an annoying bug, currently the drop down will not open when more than
one workbench window is open. i will fix this in the morning in time for the M8
build submission. changing title to reflect this remaining problem

- i'm somewhat impartial regarding the ctrl+e/ctrl+d question. i'll leave it
ctrl+e for now until we come to a concensus.
