Comment 4 John Lanuti 2005-12-20 10:45:48 EST 
This could possibly hold up WTP 1.0 or it will definately have to go into 1.0.1.  There is a bug in the virtual container API which is now exposed because the deployable is correctly using the virtual container API to get the component members rather than the actual underlying resource members.  The bug is in how it was checking to see if a resource was already listed in the components container such that any resource existing with a subset of another will be ignored.  For example, jsp_admin is ignored in this case because it starts with jsp and jsp is already in the components members list.  This would be the case for any similar situations which is really quite ugly.  There is a simple fix which makes sure the paths for the files being matched end in "/" to show that is the end of the folder name.  If it is a file being matched to a file, this will have no effect on the result since they both get the "/" added to them.  I have attached the patch here.
Comment 5 John Lanuti 2005-12-20 11:34:47 EST 
Workaround for now is to rename folders so they dont contain subsets of other folders and will be top priority for WTP 1.0.1
Comment 6 John Lanuti 2005-12-20 11:36:07 EST 
We can also use path segment api to be cleaner, the patch was merely to prove a point of what the issue was.
