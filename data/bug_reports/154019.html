Comment 2 Matthew Phillips 2006-08-17 04:15:33 EDT 
Just some further information that might be useful: we're running the SWT app from a thread kicked off by an OSGi bundle. The error occurs after the bundle is closed and the thread is shutdown and then the bundle is started again, creating a new Display and event loop thread. I've been quite careful to ensure all UI operations are done only in the UI thread, and that the thread exits correctly and disposes its Display.

Since a lot of Win32 calls rely on resources being created/destroyed from the same thread, I wonder if it's possible that something from the previous SWT event thread is causing problems for the new one. I've since added code to ensure all cached images etc allocated by the app are disposed, but this hasn't fixed the problem.
