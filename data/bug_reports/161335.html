Comment 1 Eugene Chan 2006-10-17 23:34:48 EDT 
Ruth, You are partly right, 
                        <contexts>
                                <context refID="C/C++"/>
                        </contexts>
element in your new extension point should not be changed, only the id and class should be modified. eg.
	<extension
          point="org.eclipse.hyades.ui.contextProviders">
          <contextProvider 
 			id="org.eclipse.tptp.context.sample.clanguage.CContextProvider2"
 			class="org.eclipse.tptp.context.sample.clanguage.CContextProviderCopy">
 		 	<contexts>
	   			<context refID="C/C++"/>
	 		 </contexts>
		</contextProvider>
	</extension> 

And the new provider should be listed in the perference page and available to be associated with Application Context 'C language'. Applying the new context in preference page should reflect in the views under 'C language' context selection.

Comment 4 Pavel Dolgov 2006-10-31 11:43:03 EST 
Created attachment 53003 [details]
Modified trace events samlpe

It's not a bug, it's a misunderstanding with the input data (CTrace.trcxml).

The context combo box is filtered by actual data available in the project, to avoid overburdening. As long as the input data doesn't contain any items in "C/C++_copy" language this language is omitted in the list.

Here's a modified input data that shows two items in the context combo: both "C/C++" and "C/C++_copy".
