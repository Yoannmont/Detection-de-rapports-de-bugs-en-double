Description Oleksandr Alesinskyy 2006-10-20 05:13:06 EDT 
Starting from versoion 3.2 Eclipse IDE sporadically hangs in different places.
Processor usage goes 100% and program does not respond to attempts to kill it for several minutes (regardless is it done by Alt-F4 on its window or from TaskManager Applications tab). If kill javaw.exe process from TaskManager processes tab Eclipse dies momentanly, but it take a while to reach that tab due to processor usage. This is why this bug is higher then normal priority.

There is impression that bug is associated with network operations as it mostly occurs when some network resource is unreachable (some examples - hangs in update manager, hangs on XML validation, hangs on data platform operations, e.g. testing connections). But some cases seems to be not network-dependent, e.g. on invoking of pop-up menus in Project Explorer and Java Editor.

This behavior exhibits itself at least on 3 different computers with different hardware and slightly different Eclipse configuration (e.g. different plugin sets). None of them had such issues with Eclipse 3.1 (in all cases Eclipse 3.2 was installed in separate folder in strict accordance with installation recommendations).

JDK version Sun JDK 1.5 (1.5_06 and 1.5_08).

Regards,

Oleksandr Alesinskyy
