Comment 2 Sebastian Davids 2004-04-03 05:25:28 EST 
I have written a plug-in which displays rendered HTML and source side-by-side.

It works for regular HTTP Get but not for HTTP Post.

What I've done: 

I attached a LocationListener to the browser and load the source in a different
thread via URLConnection (w/ LocationEvent.location) in changed().

Changes through HTTP Post (e.g. submitting data entered in text fields) will not
change the "location" and therefore no event will be fired.

@@@@

The ability to see rendered HTML and source side-by-side is an essential feature
of any Eclipse web plug-in in my opinion.
Comment 3 Sebastian Davids 2004-04-03 05:27:14 EST 
To clarify:

The changes *might* change the location but one has no ability to get at the
post data to display the corresponding source.

Comment 9 Felix Schollmeier 2005-04-12 17:47:20 EDT 
I managed to get the HTML as String. The function is as specified String
getContents(). It returns either the HTML source as String or null.

I do this when the document has been loaded completely by getting first the
document, then the document element and finally the innerHTML of this element.
The innerHTML property gives me however, solely everything BETWEEN <html> and
</html>. I did not add this manually since there must be a better way. But I do
not feel like browsing the MSDN library anymore. Maybe someone here has more
experience than me with this.

I added a DocumentCompleteListener which fires a DocumentCompleteEvent when the
HTML source could be retrieved. When this listener is implemented, one can
easily retrieve the HTML from the Browser object by calling:
String html = browserObj.getContents();

I hope this might help. I use it for an HTML source view in combination with the
SWT browser and it works fine so far.

Hope it helps.
Best,
Felix
