Description Alex Blewitt 2007-02-10 19:12:04 EST 
The new-look for the UI Forms in the PDE Manifest editor is horrible. Not only is the text a vaguely lighter grey (making it more difficult to read on its own) the background has a gradient that's right in the text's base-line which makes it worse. And if you move your mouse over it, the text practically does appear invisible.

If these colours are meant to make the section headings stand out, they fail miserably. If their intent is to hide the text in the title, then they succeed remarkably well.

Comment 8 Curtis d'Entremont 2007-02-12 13:36:30 EST 
I'm able to reproduce something like this by changing around the highlight color in OS X system preferences > appearance. It only happens with some of the colors, like purple, orange. So if you don't mind different colors, this is one way to work around the problem until it's fixed.

FormColors.createTitleBarGradientColors() seems to try and incorporate LIST_BACKGROUND in the gradient in some cases, which is where it's going wrong. The header area gradient doesn't seem to be affected by this problem.
Comment 9 Kim Peter 2007-02-12 14:14:14 EST 
(In reply to comment #8)
> FormColors.createTitleBarGradientColors() seems to try and incorporate
> LIST_BACKGROUND in the gradient in some cases, which is where it's going wrong.
> The header area gradient doesn't seem to be affected by this problem.

Curtis, this is probably something you know, but in case useful: the colour spec for the header and section title graphic should be the same so if the header is working that's the thing to copy. Both use LIST_BACKGROUND to modify the foreground colour, e.g.,

Group 1
Rule: If at least 2 of the RGB values are equal to or between 180 and 255, then apply specified opacity for Group 1 (Examples: Vista, XP Silver, Wn High Con #2)

Gradient Bottom = TITLE_BACKGROUND + 70% LIST_BACKGROUND
Gradient Top = TITLE BACKGROUND + 0% LIST_BACKGROUND

Group 2
Rule: If at least 2 of the RGB values are equal to or between 121 and 179, then apply specified opacity for Group 2 (Examples: XP Olive, OSX Graphite, Linux GTK, Wn High Con Black)

Gradient Bottom = TITLE_BACKGROUND + 80% LIST_BACKGROUND
Gradient Top = TITLE BACKGROUND + 0% LIST_BACKGROUND

Group 3
Rule: If at least 2 of the RGB values are equal to or between 0 and 120, then apply specified opacity for Group 3(Examples: XP Default, Wn Classic Standard, Wn Marine, Wn Plum, OSX Aqua, Wn High Con White, Wn High Con #1)

Gradient Bottom = TITLE_BACKGROUND + 90% LIST_BACKGROUND
Gradient Top = TITLE BACKGROUND + 0% LIST_BACKGROUND
