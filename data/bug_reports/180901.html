Description Hristo Sabev 2007-04-04 05:15:54 EDT 
Hi Colleagues,

I'll start with a short description of how currently one can select the wsdl from which a client or TD web service is created. Then I will outline what we would like to have.

So, currently one starts the top-down/client wizard and the he's asked to enter the WSDL URI. Pressing the "browse" button opens the "Select Service Definition" dialog. This dialog has it's own browse button. Pressing this browse button opens a dialog which allows the WSDL to be browsed from the workspace.

The problem that we see with this approach is that one always need to know the WSDL URI before he starts the WS wizard, or the WSDL needs to be located in the workspace. Thus this leads to two-step client/TD ws generation. The first step is to retrieve the URL/save WSDL in the workspace. Second step - run the wizard.

What we need is to be able to plug in our own facility(code) that will help the user find the wsdl. This could then make the whole generation on-step i.e. only one wizard. For example the user could decide to browse the WSDL then he's asked too choose between the several registered "WSDL Locators". Let's assume that he chooses to locate the WSDL in UDDI. Then the UDDI browser UI opens up. The user enters classification and so on, finally locates the WSDL. Then he continues with the generation.

Regards,
Hristo

Comment 4 Chris Brealey 2007-04-04 12:59:26 EDT 
Hi Hristo,
no apologies necessary. I put just little enough information in bug 89103 to leave it open to plenty of interpretation - My bad.

There are two extension points to mention here:
1. org.eclipse.jst.ws.consumption.wsfinder - Obsolete.
2. org.eclipse.wst.ws.locator - Alive and well.

So far, the one place we managed to put WebServiceFinder (in package org.eclipse.wst.ws.internal.wsfinder) and the org.eclipse.wst.ws.locator extension point beneath it thru its paces is from the Web Service Explorer's WSDL Browse dialog (see wsdl_content.jsp).

If you define your own Web Service Locator extension and category to the org.eclipse.wst.ws.locator extension point following, if you like, the example in org.eclipse.jst.ws.axis.consumption.core for Axis 1.x, then the WSE's WSDL Browse dialog should honour it. That's the "available workflow" to try.

Next up (this RFE) should be the Web Service (skeleton) and Client wizards, and eventually the [J2EE] Project Explorer which I would like to wire up to the WebServiceFinder framework one of these days.

Hope that helps...
Cheers - CB.
Comment 5 Kathy Chan 2007-07-27 23:17:41 EDT
