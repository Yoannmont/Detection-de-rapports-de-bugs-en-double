Description mike andrews 2004-10-18 11:20:17 EDT 
when doing cvs, i.p. address was cached by eclipse in violation of dns
standards, beyond its advertised cache time. this caused eclipse cvs to fail.

circumstance: eclipse application was open for whole weekend, but i.p. address
of our cvs server was changed over weekend. hostname of cvs server was constant.
when cvs update was attempted on monday morning, eclipse cvs failed: it
apparently had cached the old i.p. address in violation of dns principles. see
rfc 1035 for dns specifications.

note: this is probably a pervasive bug in eclipse, not just limited to cvs
subsystem. it is a MISTAKE to cache i.p. addresses beyond their advertised dns
TTL (time-to-live) cache validity timeperiods. if an application (eclipse) does
not receive a dns TTL from the underlying operating system, it MUST NOT CACHE
the address at all: in this case it is the sole responsiblity of the o/s.

thanks!
mike
