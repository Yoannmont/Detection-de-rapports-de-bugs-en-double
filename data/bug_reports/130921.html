Description Natsuki Miyamoto 2006-03-08 10:54:12 EST 
Description:
 When creating a new element on XML Editor with characters defined 
 as Unicode compatibility characters,  then the element cannot be created  
 in the appropriate format. 
 
 For example, when typing "ABC" as the element name, then the element should 
 be put as
    <ABC></ABC>
 If typing DBCS "ABC", which are defined as Unicode compatibility characters, 
 then it is rejected 
 as the invalid input.
 If typing SBCS "ABC" and DBCS "ABC" (expressed with dAdBdC in one string, 
 the element has been put as
     <ABCdAdBdC dAdBdC></ABCdAdBdC> 
 And outline displays SBCS "ABC" only as the element name. 

 Problem: 
  1: When starting with the appropriate input, then the validator for the input field doesn't work.  
  2: Extra DBCS ABC as put as the invalid value
  3: Inconsistency between XML source and outline viewer

 Build date: org.eclipse.wst.xmleditor.doc.user  1.01v200602270930
             org.eclipse.wst.xml.ui 1.01.00v200602270930
                 wtp-sdk-S-1.5M5-200603022222.zip
                 xsd-SDK-2.2.0M5.zip

OS:   Windows XP Professional SP2 Japanese     

 Steps to recreate problem:
        1- Create a general project with "New Project" wizard with any name
        2- Create a XML file from Scratch
        3- On Outline view, select top element
        4- Right click and select "New Element"
        5- "New Element" dialog is shown up and input SBCS "fgh" + DBCS "123" as the test data
            (DBCS "123" are defined as Unicode compatibility characters as well as DBCS alphabets)
           Then, the invalid data has put into XML file.
              1) Invalid name can be used as the element name -> New Element dialog should claim
              2) Invalid value are added to the element header -> Invalid value should not be added
              3) Outline cannot display the invalid element name -> Outline should display the consistent value.
           See the attached XML.JPG for details 

 Expected output:
  1) New Element dialog should claim the invalid element name, even if the starting character is correct value.
  2) If creating the element with the users' input which may have the invalid characters,
     the element should be added with the correct format and invalid value should not be added. 
  3) If creating the element with the users' input which may have the invalid characters,
     Outline viewer should display the element name as users' input, not removing the characters from its name.

Comment 4 Natsuki Miyamoto 2006-03-13 03:34:28 EST 
Hello, I'm sorry for my insufficient explanation in the original script. 
Please let me try some more description. 

When creating XML scratch XML file on this Eclipse environment, XML version is 1.0, so please let me try to talk about XML 1.0 definition. 

According to XML 1.0 definition (http://www.w3.org/TR/REC-xml/#CharClasses), there is a description : 
  "Characters in the compatibility area (i.e. with character code greater 
  than #xF900 and less than #xFFFE) are not allowed in XML names" 
and this defect mentions the problem when working with characters in the compatibility area as mentioned in this description. 

For example, DBCS full-width alphabets (U+FF21 - U+FF5A) and numbers (U+FF10 -U+FF19) are defined in Shift-JIS, and Unicode (since Unicode 1.0 to the latest one regardless UTF-16 or UTF-8), and we can input them from keyboard and output any application as general DBCS. However, when considering about XML, they are defined in Unicode compatibility zone, and we cannot use these characters as XML names. 

To work with these characters in the current XML editor, I have three problems so far. I added 3 screen captures for each issues. Please review them. 

1) The validation seems not work for the element name, when using this problematic characters in some prat of element name. (Please refer to Problem1.jpg for details)
2) When adding the element name with the invalid character, then the inserted value is in the wrong format. When using the general DBCS (not comatibility characters), then the element can be added in the correct format. 
(Please refer to Problem2.jpg for details)
3) When adding the new element with the invalid character, then the invalid string can be used for XML source. However, on the outline view, these invalid characters cannot be displayed as the element name. I can see the inconsistency between the XML source and outline view. XML source and outline view should work consistency. (Please refer to Problem3.jpg)

My expectation is the validation should work for these compatibility characters, and they cannot be used in XML names as the invalid characters 
on XML editor. If the validation works fine, then the problem2 and problem3 can be avoided, but I need  your review. 

I hope this can be helpful to you. 
If they are still not clear to you, please let me know.
