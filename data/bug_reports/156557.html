Description Tim deBoer 2006-09-07 13:26:33 EDT 
I have only seen this once, but I swear I'm not hallucinating...

I brought up WTP for this week's smoke test, and my first action was to create a dynamic web project. I opened the wizard, created a new runtime (Tomcat v4.1) and hit Finish on the first page. The newly created project had an error - "The preferences for the xdoclet runtime does not point to a valid installation." Sure enough, I checked and the web xdoclet facet was enabled.

I haven't been able to reproduce with new projects or a new workspace, but Angel Vera was sitting beside me and he can verify that I didn't even go to the second (facets) page, so there is no way that I selected the xdoclet facet.

This may be a timing thing or unreproducable, but I thought it worth opening a bug in case anyone else sees it.

Comment 5 Naci Dai 2006-10-07 06:59:25 EDT 
- Notes -

-I have not been able to replicate the random enablement - XDoclet is a facet, and the only way for it to get added is that you add ejb/web doclet. It will be removed if the facet is removed.  Another way to add the facet is to use the servlet wizard with "annotations", which will add an xdoclet facet to the project.

If you can describe a scenario, we can try to fix it.


- Second topic where old wtp projects display an problem marker "Preferences for xdoclet .." indicates that you have a project that has an xdoclet builder.  When the xdoclet builder is triggered it validates xdoclet prefs and adds a marker to the projecft indicating the problem.  There are three ways to fix the problem:

a) You want to use xdoclet:  But you have an old project.  In this case add xdoclet facet to your project and configure your xdoclet preferences.

b) You do not want to use xdoclet:  Remove xdoclet facet and the xdoclet builder.  The easiest way to do this is first add the xdoclet facet, which will  fix the project to have proper facet and builder definitions.  Then remove the xdoclet facet, which will remove the facet, remove the builder, and clear the problem markers.


Try  these and  let us know.  I will keep the bug open for a while to hear from you guys, otherwise I will markit "worksforme" and close.
