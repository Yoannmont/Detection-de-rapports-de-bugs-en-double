Comment 7 Mik Kersten 2007-06-28 12:08:45 EDT 
This is a sideffect of moving our preference wizard from plug-in activation to first task activation (bug 194365).  There was considerable pressure and blogging to do that so I did it late in the game and should have realized that we would end up with this sideffect for users who installed Mylyn but did not use it.  Fyi the reason we need the wizard is JDT/Text bug 140886, but that's not an excuse, just a reason to consider that for 3.4.

At this point I think that I have to do is to move this code to happen automatically on activation since it is the only solution that will neither cause this bug nor bring up the wizard.  I'm investigating that fix now.  If it is not too late I will consider requesting a rebuild because this is just too severe a bug and users would either have to activate a task or to go to the content assist preferences to stop this from happening.

Comment 11 Mik Kersten 2007-06-28 12:57:03 EDT 
Paul, Dani: first of all, thanks 1M for spotting this.

The fix is as follows, with the additions/changes below "NOTE".

  public class JavaUiBridgePlugin extends AbstractUIPlugin {
  ...
	@Override
	public void start(BundleContext context) throws Exception {
		super.start(context);
		initDefaultPrefs();

		// NOTE: moved out of wizard and first task activation to avoid bug 194766
		if (getPreferenceStore().getBoolean(RecommendedPreferencesWizard.MYLAR_FIRST_RUN)) {
			getPreferenceStore().setValue(RecommendedPreferencesWizard.MYLAR_FIRST_RUN, false);
			JavaUiUtil.installContentAssist(JavaPlugin.getDefault().getPreferenceStore(), true);
		}
		...
	
Side effects of this fix:
* Major: we now automatically set a non-Mylyn preference without informing the user, whereas we have previously had a strict policy of never doing this.  This is more of a user perception thing than a functionality issue.  When Mylyn is not enabled its content assist proposals are the plain Java content assist proposals and do not run any extra code.
* Minor: the user has no way of turning off the Mylyn proposal with a click, but instead has to do the three clicks via Preferences -> Content Assist -> Advanced.

Paul, Dani: let me know if you have any additional thoughts or input.  Otherwise I will commit this fix and attach the build to this bug for verification on your end since the download site is frozen now.
