Description Sachin Patel 2006-01-23 14:22:08 EST 
When creating a new geronimo server, the generic server wizard fragment is used to prompt for server instance properties, (host, port, uid, password, etc...).  The hostname field defaults to localhost.  If modified when the wizard is finished, the value still remains localhost.  The modified value does not persist in servers.xml nor when the server editor is opened.

Only when the server editor is opened and hostname is modified does "hostname" save.

All other fields seem to save ok, the problem seems only to happen on hostname.

The serverdef defines the hostname property as the following...

<property id="serverAddress"
	label="%serverAddress"
	type="string"
	context="server"
	default="127.0.0.1" />
Comment 1 Tim deBoer 2006-01-23 14:54:11 EST 
The WST already defines a hostname on the first page of the new server wizard, and that is what is shown at the top of the server editor (and saved in server.xml). It sounds like a specific generic server has an additional hostname - is this persisted as an attribute or shown in the editor elsewhere?
Comment 2 Sachin Patel 2006-01-23 15:02:21 EST 
Ah you're right. both of them are there. in the servers.xml Didn't catch that...  Tim F. orginally put "serverAddress" there, and I think it may be becuse hostname doesn't appear in the wizard, since 127.0.0.2 is what I modified in the wizard page.  

<servers>
  <server server-type-id="org.apache.geronimo.generic.server.10" runtime-id="Apache Geronimo v1.0" hostname="localhost2" name="Apache Geronimo 1.0" id="1_23_06_2_16_PM2" server-type="org.apache.geronimo.generic.server.10">
    <map key="generic_server_instance_properties" rmiport="200" adminPassword="manager2" serverAddress="127.0.0.2" adminID="system2" port="8081"/>
  </server>
</servers>
Comment 3 Sachin Patel 2006-01-23 23:23:47 EST 
So looking at ServerTypeDefinitionDecorator, only the properties defined in the server def are displayed as fields in the wizard fragment.  So each server def must provide a serverAddress field.  In the server editor, the host property can be modfied, but that leaves serverAddress and host not being in sync.  Whats the best way to handle this?  Should host name be provided in the wizard fragment and all the server defs should remove the serverAddress property?
