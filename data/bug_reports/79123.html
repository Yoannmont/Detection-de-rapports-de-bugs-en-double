Comment 3 Douglas Pollock 2004-11-22 11:23:38 EST 
PAGE_DOWN and PAGE_UP.  In the future, please look at SpecialKey (and its 
friends) if you have any doubts as to the proper syntax for keys.  [I know this 
should probably be advertised better, but I'm not sure where....] 
 
Also, you cannot use deprecated attributes to access 3.0 features (namely, M1).  
Please turn up your PDE compiler settings to make usage of deprecated elements 
and attributes a warning or an error. 
 
So, the attribute should look like this: 
 
            keySequence="M1+PAGE_UP" 
 
 
As a side note, could you please chose a different key binding?  The 
"M1+PAGE_UP" and "M1+PAGE_DOWN" key bindings were added by SWT, at the request 
of Platform UI.  They mimick the tab switching behaviour of Firefox.

Comment 5 Steve Northover 2004-11-22 15:43:46 EST 
Seems to be working in SWT.  Here is some text code that has a StyledText 
inside a CTabFolder, a TabFolder and on its own.  You can see that the 
selection is not disturbed when Ctrl+PgUp and Ctrl+PgDn is pressed for the 
StyledText controls in the folders but it is change for the StyledText that is 
not within a folder.

import org.eclipse.swt.*;
import org.eclipse.swt.widgets.*;
import org.eclipse.swt.layout.*;
import org.eclipse.swt.custom.*;

public class PR_79123 {
public static void main (String [] args) {
	StyledText t [] = new StyledText [3];
	Display display = new Display ();
	Shell shell = new Shell (display);
	shell.setLayout (new FillLayout ());
	CTabFolder f1 = new CTabFolder (shell, SWT.NONE);
	for (int i=0; i<2; i++) {
		CTabItem item = new CTabItem (f1, SWT.NONE);
		item.setText ("Item " + i);
		StyledText text = t [0] = new StyledText (f1, SWT.V_SCROLL | 
SWT.H_SCROLL | SWT.BORDER);
		text.setText ("Text " + i + ".\nmore text.");
		text.setSelection (4, 16);
		item.setControl (text);
	}
	TabFolder f2 = new TabFolder (shell, SWT.NONE);
	for (int i=0; i<2; i++) {
		TabItem item = new TabItem (f2, SWT.NONE);
		item.setText ("Item " + i);
		StyledText text = t [1] = new StyledText (f2, SWT.V_SCROLL | 
SWT.H_SCROLL | SWT.BORDER);
		text.setText ("Text " + i + ".\nmore text.");
		text.setSelection (4, 16);
		item.setControl (text);
	}
	StyledText text = t [2] = new StyledText (shell, SWT.V_SCROLL | 
SWT.H_SCROLL | SWT.BORDER);
	text.setText ("Some text.\nmore text.");
	text.setSelection (4, 16);
	shell.pack ();
	for (int i=0; i<8; i++) {
		text.append (i + "-Line of text-" + i + "\n");
	}
	shell.open ();
	while (!shell.isDisposed ()) {
		if (!display.readAndDispatch ()) display.sleep ();
	}
	display.dispose ();
}
}

Comment 13 Kim Horne 2004-12-10 16:45:39 EST 
Okay, this is a Platform UI problem.  We're done bickering about that.  The
problem is probably as follows:

1.) Traverse event arrives at the StyledText widget, and the StyledText widget
does nothing.
2.) The presentations code intercepts the traverse event and forwards it to the
decorations.
3.) The CTabFolder gets the traverse event and switches tab.
4.) SOMETHING GOES WRONG, and SWT does not realize that work has been done with
this traverse event.
5.) SWT sends a key down event to the StyledText widget.  Cursor position changes.

Comment 17 Veronika Irvine 2005-04-05 09:13:23 EDT 
In PartPane, the code that forwards on the traversal to the parent is 
incorrect.  It should be changed as shown below:

public void keyTraversed(TraverseEvent e) {
            // Hack: Currently, SWT sets focus whenever we call 
Control.traverse. This doesn't
            // cause too much of a problem for ctrl-pgup and ctrl-pgdn, but it 
is seriously unexpected
            // for other traversal events. When (and if) it becomes possible 
to call traverse() without
            // forcing a focus change, this if statement should be removed and 
ALL events should be
            // forwarded to the container.
            if (e.detail == SWT.TRAVERSE_PAGE_NEXT
                    || e.detail == SWT.TRAVERSE_PAGE_PREVIOUS) {
                ILayoutContainer container = getContainer();
                if (container != null && container instanceof LayoutPart) {
                    LayoutPart parent = (LayoutPart) container;
                    Control parentControl = parent.getControl();
                    if (parentControl != null && !parentControl.isDisposed()) {
                        //parentControl.traverse(e.detail);
                        //e.doit = false;
                        e.doit = parentControl.traverse(e.detail);
			if (e.doit) e.detail = SWT.TRAVERSE_NONE;
                    }
                }
            }
        }
