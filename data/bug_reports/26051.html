Comment 18 Jared Burns 2003-04-09 15:51:19 EDT 
*motions to the band* One - more - time!

Here's another test case. This one's simple and it's worked every
time on the machines we tested it on.
1. Turn off autobuild.
2. Edit a file and save.
3. Turn on autobuild.
4. Before the build finishes, bring another window to the top.
5. When the build finishes, Eclipse pops to the top.

We tested this on Mandrake 9.0 and 9.1 in Gnome and WindowMaker.
While testing with WindowMaker, we noticed that if the desktop is
set to use sloppy focus, Eclipse won't come to the top in step 5,
it will just take the focus. Setting the desktop to "autoraise"
mode (which raises a window when it gets focus) causes the window
to come to the front when the build finishes.

So the underlying problem might be that Eclipse is stealing focus.
The coming to the front might just be a side effect from the window
manager.
