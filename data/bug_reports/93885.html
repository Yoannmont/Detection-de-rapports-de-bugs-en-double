Description Chris Nappin 2005-05-06 05:21:18 EDT 
on WTP 1.0 M4 in Eclipse 3.1 M6

The JSP editor reports variables as never read, even if they are used in a JSP
expression. For example:

<%
  String myVar = "/test.gif";
%>
<?xml-stylesheet type="text/xsl" href="<test:someTag><%= myVar %>" ?>

Comment 15 Amy Wu 2007-05-24 17:39:40 EDT 
While the title of this bug does report the problem everyone is seeing, there appears to be 3 very different use cases, so the issues should be tracked in 3 different bugs (rather than piling them into 1):
1) described in the original description, comment #0
<%
  String myVar = "/test.gif";
%>
<?xml-stylesheet type="text/xsl" href="<test:someTag><%= myVar %>" ?>
This has to do with jsp expressions not being parsed correctly in an xml-stylesheet processing instruction.  This bug should be used to track this issue since that was the original issue reported.

2) described in comment #5, comment #12, comment #13 - validator does not recognize variables are used in EL expressions.  There is another bug open on this issue: bug 129559.  People should use/cc that bug to track that issue (also move your bug votes there if that's the real fix you wanted)

3) described in comment #8 - I don't really understand the scenario, but if the problem is being reported in the Java editor, that's different than the JSP editor and a bug should be opened on Eclipse/JDT
