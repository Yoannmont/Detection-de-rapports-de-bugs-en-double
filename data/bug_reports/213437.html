Description Markus Knauer 2007-12-19 06:12:19 EST 
The features

  org.eclipse.platform_3.4.0.v20071210-_19wFA0F-t4fsN8Yw6jHdAaXSex9
  org.eclipse.rcp_3.4.0.v20071210-969CEcdEhBDtp-HRdw8M5HG

contain references to bundles with version "0.0.0" that are not available from the update sites:

  org.eclipse.core.filesystem.win32.ia64
  org.eclipse.core.resources.win32.ia64
  org.eclipse.swt.win32.win32.ia64
  org.eclipse.swt.gtk.linux.s390

  org.eclipse.swt.gtk.solaris.x86

These missing bundles result in a Ganymede build failure.

Thus the fragments don't exist as part of the update site.  Please change the Ganymede build to ignore these missing plugins.

Comment 7 John Arthorne 2008-07-17 15:48:45 EDT 
We should remove these extra entries from the org.eclipse.rcp and org.eclipse.platform features. They were added in anticipation of a community contribution of these fragments to the Eclipse build. Since this hasn't happened, we end up with invalid entries in the binary version of the org.eclipse.rcp feature.  If a third party wanted to added support for these platforms, they would have to alter the binary feature to update the plugin version number. The end result would be two *different* version of the org.eclipse.rcp feature with the same feature version number. This would cause problems for our install/update technology which uses id/version to identify unique content. 

If someone wants to add plugins for ia64 or s390, they can bundle them in a separate feature, or contribute the fragments back to Eclipse so we can include them in our platform build.
