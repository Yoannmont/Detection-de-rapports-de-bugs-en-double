Comment 10 Veronika Irvine 2005-06-17 14:34:58 EDT 
I don't think this code (focus event -> applyEditorValue operation) ever worked
for the DialogCellEditor.

In the TableEditorImpl, the control.addFocusListener(focusListener) is adding
the focus listener to a Composite which contains a Label and a Button.  The
Composite never gains or loses focus, it is the Button that has the focus.  The
focus listener needs to be applied to the Button, or you need to use a different
listener (e.g. a deactivate listener).

Moving to the UI team because the change needs to be made to JFace.
Comment 11 Tod Creasey 2005-06-21 10:05:10 EDT 
I have verified this is not a regression  as well using 20050621. Marking as
major instead of critical as this is not new behaviour and there is a workaround
(deactivate cell editors when focus gained on the other widget).

The fix is pretty straightforward - add 

control.addListener(SWT.Deactivate,new Listener(){
 public void handleEvent(Event event) {
    applyEditorValue();						
    }					
});

to TableViewerImpl#activateCellEditor.

My only concern is that this late in the game there may be other side effects we
are unsure of. As this is not a regression from 3.0 I would just as soon leave
it to 3.1.1 or 3.2.
