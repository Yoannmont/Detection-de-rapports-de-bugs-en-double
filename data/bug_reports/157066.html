Description Karl 2006-09-12 12:56:18 EDT 
This is an intermittent problem, happening with about 20% of copy/pastes.

If I copy some java code from one java file to another (using ^C, ^V), it sometimes doesn't paste the text in.  It does, however, update the imports at the top of the file if the code to be pasted contained classes not already imported to the new java file.

I have to hit ^C multiple times, or in extreme cases, copy something completely different first, before it will paste the actual code itself.

Comment 22 Dmitry 2008-02-01 11:29:10 EST 
Some more info about this problem. I installed RCP package of eclipse, downloaded version r331_v20070906 of org.eclipse.jdt.ui plugin and added breakpoint in ClipboardOperationAction.doCutCopyWithImportsOperation() at line 399 after this lines:
Object textData= clipboard.getContents(TextTransfer.getInstance());
Object rtfData= clipboard.getContents(RTFTransfer.getInstance());
Breakpoint has condition - textData == null.
After that run eclipse in debug and do some Ctrl+Inserts, i usualy get stop at breakpoint in first 10 copy actions. Variables view shows that textData is null, but if you add clipboard.getContents(TextTransfer.getInstance()) in Expressions view, it returns correct result, rtfData also has my selected text. It looks like some synchronization bug when you try to read clipboard before it got content in it or something like it. And of course Paste after such Copy doesn't paste anything, only imports :(
