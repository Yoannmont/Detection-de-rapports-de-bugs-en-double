should show up in the dialog, but the dialog contained no file name to be
selected.

Problem 3:
I then selected "Browse" button, in the dialog I selected debugC under Debug
directory. However the error message on the dialog was: "Selection must be a
binary file.", and OK button was disabled, so I cannot select file DebugC.

Problem 4 (the main problem):
I got around the above problem by directly input "Debug/debugC" into the
"C/C++ Application" box. Start to run, and got correct output. Then I invoke
menu Debug..., under GDB Debug node, I got a child node:

Comment 9 Robert Fu 2004-06-30 02:35:54 EDT 
Good news, debugging C and C++ program is working on my HP-UX 11.11 
environment now. Late last night, after too much flustruation with slowness of 
Eclipse on the old HP-UX machine with 2 CPUs of 450MHz, I took a bold action 
to reboot the machine (All other users went home). Then debugging C code 
worked. I guess reboot is needed after applying patch PHSS_288771. Patch 
PHSS_288771 solved link problem with +init, and HP swinstall did not inform me 
that I need reboot machine, so I didn't reboot it before.

Then I tried to debug C++ code, which is my main goal. I encoutered other 
problems, which I solved tonight. Here are problems and solutions/workarounds:

1. Perhaps because how things works on HP-UX with GCC/GDB, the executable need 
be linked with pthread (e.g., g++ -lpthread -o HelloWorld main.o) in order to 
avoid the following problem:

    /usr/lib/dld.sl: Unresolved symbol: pthread_once (code) 
from /home/rfu/gnu/usr/local/lib/libgcc_s.sl

2. Change attribute of the executable to enable SHLIB_PATH (e.g., chatr +s 
enable HelloWorld) in order to solve the following problem:

   	/usr/lib/dld.sl: Can't find path for shared library: libgcc_s.sl
	/usr/lib/dld.sl: No such file or directory 

If there are no better solutions to the above problems, may you please 
generate the makefile (for HP-UX only) to link with pthread (-lpthread) and 
run "chatr +s enable EXECUTABLE", where EXECUTABLE is the executable?

The other minor issues I found are related to stepping through input 
statements. After stepping to code like 'cout << "Enter your name:"; cin >> 
yourName;' in the HelloWorld tutorial, I didn't found prompt "Enter your 
name:" in any console, and did not know where to input. This is the first 
issue. The second issue is that after I input something in a console within 
IDE, debugging menu commands like 'step over' were not enabled, so I had no 
way to continue, and had to terminate it.

I'll attach files containing debug info when I followed steps in CDT Debug FAQ 
5, in case you're still interested in such info.

BTW, am I the first person in the whole world who used Eclipse/CDT to 
successfully debug C and C++ programs on HP-UX? I just wander it. It seems 
there was no HP-UX version of CDT 1.x.

Thanks a lot for your help! 

Robert Fu
