Comment 6 Konrad Kolosowski 2003-06-13 16:25:59 EDT 
JRE in preferences is only used by JDT, not Platform or Help.  You need to 
launch Eclipse with stable jre (pass -vm option).  Check which jre Eclipse is 
using by going to Help->About->Configuration Details.

If help preferences are set to Embedded Internet Explorer Adapter, than after 
launching help (you wrote you see the help browser), there must be 2 java 
proceses as help browser is launched as a separate java process from the 
Workbench.  It is important which one has problems (holding CPU).

Could you do the following debugging:
Copy .option file from eclipse\plugins\org.eclipse.help.ui_2.1.0 directory to 
eclipse dirctory.
Edit the copied file and change value of org.eclipse.help.ui/debug/ieadapter 
option to true.
Run Eclipse passing -debug option.
This will result in debug messages printed to console window.  Launch Javadoc 
to reproduce the problem by pressing F1 ...
One of the last lines in the console window will be "Sending the following 
command to the IE browser process: displayURL SOMEURL".
What is the SOMEURL?
Can you open this URL in the InternetExplorer when Eclipse is still running?
Can you open this URL in the InternetExplorer when Eclipse is not running?

Thanks.
