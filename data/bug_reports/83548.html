Description R. A. Rivas Diaz 2005-01-24 12:12:29 EST 
This RFE applies to UI *and* compiler.

Currently on Windows -> Preferences -> Java -> Compiler -> Error/Warnings you
can set the severity level to Error, Waring or Ignore.

Introducing also a "Inform" severity level with the asociated "i" as an icon in
the java editor will allow us to clasify some settings as "informe us about
this, but don't mark the code and the project as 'have warnings'".

I prefer to get all the information the compiler can give me, but some settings
are only considered for me as suggestions, not even warnings. I prefer that my
code don't have warnings.

Also the "Filters" dialog that can be open from the "Problems" view allows us to
view/ignore problems with severity level "Info" but it is not used in compiler
settings.

In WSAD 5.1 you can get some "infos" from the compiler about some EJB info, so
this functionality probably is already supported in the Java editor and/or Java
Compiler.

Hope this can be added.

Notes on the fix itself, which is still a draft:

1. This is why I call the patch a draft: To make the reviewing/testing easier, the patch simply converts all the ignored problems to 'info'. To test, simply pick an optional problem, mark it as Ignored and introduce that problem in a Java file and you will see the errors appearing as 'info' marker in both views.

2. No marker resolution (i.e. no quickfix) yet for Info markers.

3. Info problems only make it to the problems view and markers view and are not passed onto the command line compiler, APT, code generation etc.

4. Changes to ContributedProcessorDescriptor in JDT UI is for completion sake and doesn't seem to affect any functionality in my testing.

5. Tests yet to be added.
