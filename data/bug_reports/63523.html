Description Age Mooy 2004-05-22 09:04:03 EDT 
I upgraded from M8 to M9 today. Since then, every time I open my workspace, my
open editor tabs appear in random order. I have no been able to find much of a
pattern except that the editor that was active before the workspace was
previously closed will be the first tab after startup.

I also noticed that since the upgrade, newly opened editor tabs will appear
first in line instead of last.

I run eclipse on Sun jdk 1.4.2_04 with the following commandline:

E:\bin\ide\eclipse-3.0-m9\eclipse.exe -data E:\workspaces\serenity -vm
E:\bin\jdk\1.4.x\bin\javaw.exe -nosplash

I have reproduced this same problem on my laptop, which has the same OS/VM/workspace

Comment 8 Kim Horne 2004-06-10 20:10:59 EDT 
The core problem is that the workbench does not call the relevant serialization
methods on the editor presentation.
Comment 9 Debbie Wilson 2004-06-10 21:22:17 EDT 
In DefaultPartPresentation.restoreState, the serializer tries to organize the 
parts. But when it tries to retrieve a part (given an id), the part is always 
null as they have not been set up yet.  Not sure if serialization needs to 
happen later for editors than for views or if the part populating needs to 
happen earlier.
Comment 10 Debbie Wilson 2004-06-14 10:33:29 EDT 
Definitely not for 3.0RC2.  Likely we should defer this to 3.1 as the fix 
(which Kim has had more success with) touches several areas.
