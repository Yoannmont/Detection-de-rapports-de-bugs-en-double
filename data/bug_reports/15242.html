Comment 4 Lynne Kues 2002-05-08 14:34:46 EDT 
In the "more test cases" attachment.  Two test classes exist.

1. Run CoolBarLayoutTest1.  Notice that the "script" coolitem is placed all the 
way over to the right.  This is unexpected behavior.  If you change the 
addItems0() method to be as follows:

	// add search at end ==> position 0
	ToolBar toolBar = createToolBar(toolNames[0]);
	createCoolItem(toolBar, -1);
	// add script at end ==> position 1
	toolBar = createToolBar(toolNames[1]);
	createCoolItem(toolBar, -1);
	// add debug at end ==> position 2
	toolBar = createToolBar(toolNames[2]);
	createCoolItem(toolBar, -1);

the desired behavior occurs (i.e., last tool is not slid all the way over to 
the right).  The problem appears to be related to creating items and specifying 
an index.  There are instances when Eclipse needs to create an item at a 
specific index, so I do not think I can work around this.

2. Run CoolBarLayoutTest2.  Press the "Delete" button, which will delete the 
last cool item.  Notice that the "script" cool item slides all the way over to 
the right.  It should stay where it was.

Comment 27 Lynne Kues 2002-06-03 12:44:45 EDT 
To summarize, there are three cases where layout to the right occurs:

1. when items are created/added
2. when items are deleted (e.g., you close an editor which causes editor action 
bar to be removed)
3. when items are moved (via drag when you rearrange a coolbar)

Case 1 has been fixed.  I have not seen any cool items slide to the right when 
they are created/added, regardless of how many rows you have.

Case 2 (one scenario is presented in 18517) and Case 3 have not been fixed.
