Description Pat McCarthy 2003-09-14 09:56:42 EDT 
The problem was reported to my by an ISV using eclipse - this is what I sent 
them:
I think the failure of the build.xml generation is a bug. 

I could not fail using the code you sent, but only by replicating the scenario 
based on your description.
As soon as you have a required plugin where the version referenced is not in 
the target platform, it fails.  

The same logic that would let the runtime platform use any acceptable plugin 
version (equivalent or compatible) does not work for the logic used to create 
the build.xml file. It specifically says the same version must be available and 
this is wrong (In my opinion).  I'm going to open a bug on eclipse.

You can change the pde create build.xml logic so that it does not care - seems 
to work for my simple test.

Try it on your side and let me know how it goes.

In this 
plugin: 		org.eclipse.pde.build
package: 	org.eclipse.pde.internal.build;
class:		AbstractBuildScriptGenerator
method:	addPrerequisites

Use this logic modification - it is near the end of the method:

		// add libraries from pre-requisite plug-ins.  Don't worry 
about the export flag
		// as all required plugins may be required for compilation.
		PluginPrerequisiteModel[] requires = target.getRequires();
		if (requires != null) {
			pluginChain.add(target);
			for (int i = 0; i < requires.length; i++) {

				// TODO Changed here instead - don't use 
version when calling method
				PluginModel plugin = getPlugin(requires
[i].getPlugin(), null);
				//getPlugin(requires[i].getPlugin(), requires
[i].getVersion());

The only change is the removal of the version when calling getPlugin();

I tested this on my replication of the problem and the build.xml is now created.
Comment 1 Pat McCarthy 2003-10-27 12:08:53 EST 
This has hit a few people.... What I end doing is sending them an alternate JAR 
with this description of what to do with it:
-Stop Eclipse/WSxx product. 
-Find the org.eclipse.pde.build plugin in your system.  Rename it to 
pdebuild.jar.org.
-Copy the alternate JAR into the dir.
- Try build process again.

I'll attach the JAR to this bug report - it works for Eclipse 2.1.1 and 2.1.1 
based products.
