Description Lorenzo Bettini 2007-10-01 09:35:12 EDT 
Build ID: M20070921-1145

Steps To Reproduce:
1. download eclipse 3.3
2. search for updates
3. you'll get an error about http://download.eclipse.org/technology/emft/updates/


More information:
If you download an eclipse 3.3. version with EMF (e.g., RCP/Plugins development) and try to search for updates, you'll get this error:

Unable to access site:
"http://download.eclipse.org/technology/emft/updates/" [Server returned HTTP
response code: "403 Forbidden" for URL:
http://download.eclipse.org/technology/emft/updates/.] 

since it still refers to the old update site, instead of the new one:

http://www.eclipse.org/modeling/emft/updates/

FWIW, the other way to circumnavigate hardcoded feature update URLs is to use policy files:

http://wiki.eclipse.org/Using_Policy_Files
Comment 4 Lorenzo Bettini 2007-10-29 13:15:20 EDT 
I tried to use this policy file


<?xml version="1.0" encoding="UTF-8"?>
<update-policy>
  <url-map pattern="org.eclipse.emf.*" url="http://download.eclipse.org/modeling/emft/updates/"/>
</update-policy>

but it does not work... probably it's not clear to me how to solve this problem with policy files...
Comment 5 Nick Boldt 2007-10-29 14:36:21 EDT 
(In reply to comment #4)
> <?xml version="1.0" encoding="UTF-8"?>
> <update-policy>
>   <url-map pattern="org.eclipse.emf.*"
> url="http://download.eclipse.org/modeling/emft/updates/"/>
> </update-policy>

Try using pattern="*" or url="http://download.eclipse.org/modeling/emft/updates/site.xml"
(or url="http://download.eclipse.org/modeling/emft/updates/site-interim.xml")
Comment 6 Lorenzo Bettini 2007-10-30 04:09:37 EDT 
This one seems to work:

<?xml version="1.0" encoding="UTF-8"?>
<update-policy>
  <url-map pattern="org.eclipse.emf.*"
url="http://download.eclipse.org/modeling/emft/updates/site.xml"/>
</update-policy>

If I used * then only that site would have been checked for updates, and I'd have missed updates for all the other features.

However, now I was wondering if, with this solution, I only checked for EMF updates from the EMFT site... what about the updates from the EMF site?
Comment 7 Nick Boldt 2007-10-30 10:51:38 EDT 
Sorry, I should have been more clear:

1. enable policy file to override /technology/emft/ paths in features
2. run update on any features that can be updated from that site
3. disable policy file, now that emf* features have the correct URL
4. run update on all features; should no longer get error about /technology/emft/
