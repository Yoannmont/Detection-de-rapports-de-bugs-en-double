Description Wassim Melhem 2006-06-14 00:32:11 EDT 
User Assistance provides a rich set of features (eg tables of content, context-sensitive help, cheatsheets, ...), but plugging into this UA support is currently tedious, cumbersome and error-prone.  

PDE should provide tools to help author and validate User Assistance-related files.

Here are some ideas:

Cheat sheet authoring
Cheat sheets should be promoted as a handy general purpose tool for expressing and exchanging sequential instructions. A strong and easy to use authoring tool is a vital element of this strategy. Authoring would allow people to create a cheat sheet in minutes, hook it up with help topics and available commands. Previewing a cheat sheet should also be easy.

Context help authoring
this is a sore point for many people because context help authoring requires adding an extension, editing an XML file, listing 'see also' links for each context id and associating the id with a widget in Java code. 
A tool similar to 'Externalize Strings' that introspects Java code and locates constructors where 'setHelpId' can be inserted would be heaven, but certainly hard.
A context help file editor may be more realistic. A benefit here would be seeing contexts in a form with a text area for adding description and a lookup to choose topics for the 'See Also'. 
A great tool would also be a validator that would check broken references (topics that do not exist) and references in Java code that point at the missing context Ids.

TOC management
building up a TOC, hooking topics, previewing the TOC based on the toc.xml file etc. Again, a toc.xml editor with value add (validation, topic lookup, preview).

Comment 4 Wassim Melhem 2006-06-15 09:46:37 EDT 
re comment 3:

I suppose the answer in the SDK proper has been rather simplistic:

Platform is a platform and does not provide tooling
JDT provides Java tooling
PDE provides tools to develop Eclipse products

So if were were to have UA tooling in the SDK, PDE would be the place to host it since it's very much tied to Eclipse products. 

If tooling is to be done outside the SDK, they would need no hooks from PDE since all we are dealing with here are XML (and some Java) files.
