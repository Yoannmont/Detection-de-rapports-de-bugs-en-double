Description werner 2007-11-23 04:28:36 EST 
to reproduce:
ATTENTION: reproducing this bug successfully means killing your X session, including all running programs!

debug code A shown below. Set a break point at the line indicated in the code. Open the expressions view and add the expression "thing.toString()". Adding i as an expression makes sense to keep track of the iteration. Continue running the code and watch how i grows and how the big expression builds up (it doubles every iteration). What happens on my system is the following:

Around iteration 16 I get a black screen, then coloured dots scattered on the screen, then the X server dies and I'm returned to th login screen, i.e. eclipse and all other running programs have been killed!

Sometimes, shortly before the crash, I can observer that the label part of the expressions view, where "thing.toString()" is displayed, screws up: the x's from the result of the expression start being written on top of the correct label.

The same thing happens with code B when using eclipse 3.3.1.1. Eclipse 3.3.0 had no problem with code B. It displayed "deallocated" in the output pane of the expressions view instead.

I classified this as critical because it kills other applications which might cause data loss. For the same reason please don't just put this down as an nvidia bug or the like. Add at least some safe guard against expressions running wild. thanks

        String s = "x";

            s = s + s;
        }
        return s;

}
