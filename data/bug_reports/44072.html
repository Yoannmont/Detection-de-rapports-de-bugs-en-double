Description Dani Megert 2003-10-02 08:12:51 EDT 
I20030930

I use a Combo with SWT.BORDER | SWT.DROP down style. Since we want to provide
content assist we need to compute the caret (pixel) position. I can compute the
width of the text but there's a little offset (border width, client area) which
makes the content assist appear a bit to far on the left. I expected that using
getBorderWidth() and getClientArea().x would help but those return 0.
Comment 1 Steve Northover 2003-11-21 16:56:24 EST 
There may be a bug in these methods but what you really need is 
Text.getCaretLocation() but implemented for Combo.  Measuring it yourself is 
error prone and might fail on some platforms.
Comment 2 Dani Megert 2003-11-24 04:06:06 EST 
I agree that having such a method would even be more appreciated.
Comment 3 Markus Keller 2004-02-19 11:04:37 EST 
The same problem exists on MacOS X: getBorderWidth() and getClientArea().x are
0. Since the border is significantly larger than on Windows, it's even more
apparent.

I agree though that Combo#getCaretLocation() would be the perfect solution.
Comment 4 Steve Northover 2005-10-31 15:36:55 EST 
The border in the Combo on Windows is not a real Windows border.  It is drawn 
by the control.  Therefore, it doesn't get included in getBorderWidth() and is 
part of the client area.  For example, on Windows, you could create a GC and 
draw on top of it.

Can you give me a small snippet the shows the problem?
Comment 5 Markus Keller 2007-08-10 06:44:10 EDT 
(In reply to comment #4)
I agree that trying to find the caret location by using getBorderWidth(), getClientArea(), and measuring the text width is doomed to fail (we already fail when the text in the Combo is scrolled).

So let's take this as a request to implement Combo#getCaretLocation(), analogous to Text#getCaretLocation().

This API will be used in ComboContentAdapter#getInsertionBounds(Control).

Comment 15 Felipe Heidrich 2011-10-03 16:43:00 EDT 
My testcase:
package misc.prs;

import org.eclipse.swt.SWT;
import org.eclipse.swt.graphics.Point;
import org.eclipse.swt.graphics.Rectangle;
import org.eclipse.swt.layout.GridData;
import org.eclipse.swt.layout.GridLayout;
import org.eclipse.swt.widgets.Combo;
import org.eclipse.swt.widgets.Display;
import org.eclipse.swt.widgets.Event;
import org.eclipse.swt.widgets.Listener;
import org.eclipse.swt.widgets.Shell;

public class PR44072 {
	public static void main(String[] args) {
		final Display display = new Display();
		Shell shell = new Shell(display);
		final Combo combo = new Combo(shell, SWT.DROP_DOWN);
		//final Text combo = new Text(shell, SWT.RIGHT_TO_LEFT);
		final Shell hint = new Shell(shell, SWT.NO_TRIM);
		hint.addListener(SWT.Paint, new Listener() {
			public void handleEvent(Event e) {
				e.gc.drawString(combo.getCaretPosition()+"", 2, 2);
			}
		});
		combo.addListener(SWT.KeyDown, new Listener() {
			public void handleEvent(Event e) {
				display.asyncExec(new Runnable() {
					@Override
					public void run() {
						Point pt = combo.getCaretLocation();
						Point size = combo.getSize();
						Rectangle rect  = display.map(combo, null, pt.x, pt.y + size.y, size.x, size.y);
						hint.setLocation(rect.x, rect.y);
						hint.moveAbove(null);
						hint.redraw();
					}
				});
			}
		});
		hint.setSize(40, 40);
		hint.open();
		hint.setBackground(display.getSystemColor(SWT.COLOR_INFO_BACKGROUND));
		hint.setForeground(display.getSystemColor(SWT.COLOR_INFO_FOREGROUND));
		shell.setLayout(new GridLayout());
		combo.setLayoutData(new GridData(GridData.FILL_HORIZONTAL));
		shell.setSize(400, 400);
		shell.open();
		while (!shell.isDisposed()) {
			if (!display.readAndDispatch())
				display.sleep();
		}
		display.dispose();
	}
}
