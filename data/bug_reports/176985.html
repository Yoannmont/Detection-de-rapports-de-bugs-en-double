Steps To Reproduce:
1. install Eclipse SDK I20070306-1200 build
2. install the latest NL packs (you may have to use 3.2 NL packs and change the tolerance range in manufest.mf in org.eclipse.platform.doc.user to "[3.3.0,3.4.0)")
3. start Eclipse in a different language (e.g. -nl zh_TW)
4. open the on-line doc
5. select "Workbench User Guide>Legal" on the left pane
6. navigation link at the top of notices.html does not display NLS chars properly (see attachment 1 [details] below)
7. note that the navigation link at the top of other pages, e.g. the "What's new" page, are displaying okay (see attachment 2 [details] below)
8. I think the problem is related to the fact that the notices.html page is not translated. The English notices.html page is actually loaded.

Problem is the doc we're injecting the links into (notices.html) has an encoding that doesn't support what we're injecting. The only real fix would be to re-encode the html, but I think this will be far more problematic than it's worth and I would want to do this only as a last resort.

Instead, I've adjusted the encoding to utf-8. Any docs like this that aren't translated will need to do the same (be on the lookout).

I've covered the platform docs, will CC folks from PDE and JDT do cover their doc plug-ins.

Comment 11 Chris Goldthorpe 2007-09-14 14:41:28 EDT 
Fixing this in the Eclipse Help system would be quite risky. This is not something that could be considered for Eclipse 3.3.x.

It can also be fixed in the affected documents, which only seem to be notices.html. Unfortunately there seem to be a lot of projects which have an affected document, the JDT, PDE and Platform are now all clean.

There are two ways to fix the documents, the best way is to change the charset from to "utf-8" to "iso-8859-1", alternatively a version of the document with charset "iso-8859-1" can be added to the languag pack for any language which is affected.

Comment 16 Kit Lo 2007-09-17 15:43:13 EDT 
It's depends on the writer's preference, English pages usually have the following charsets: Windows-1251, ISO-8859-1, us-ascii, UTF-8.

Translators have to convert the first 3 to the charset for their country. We don't have to change for UTF-8 charset.
