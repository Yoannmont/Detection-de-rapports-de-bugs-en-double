Comment 5 Simon Kaegi 2007-05-31 14:05:22 EDT 
Thanks that helps.
--
osgi.compatibility.bootdelegation=false
--

When you run Equinox standalone you're running in OSGi "strict" mode. What this means is that it is "mandatory" to import all non java.* resources as this is what the OSGi spec stipulates.
See bug 178477

Try adding imports for javax.management to your bundle.

Comment 7 Simon Kaegi 2007-05-31 14:24:00 EDT 
Yes... exactly between M6 and M7.
Equinox is the reference implementation of OSGi R4 and R4.1 and the issue was brought to light partly in the context of the work done for JSR291.

We need to do a better job explaining the more strict behaviour spec'ed by the er... spec however I'm going to mark this bug as INVALID since we should always have Import-Package statements for non java.* packages.
