Comment 11 Markus Keller 2009-03-30 13:36:36 EDT 
Thanks for looking into this!

I don't think you should move the description into tooltips. Tooltips are annoying to use if you're looking for something and have to hover over every item to read them all.

The current tooltips are also getting into your way if you want to read the label of an item, since they overlap the target (normal tooltips don't do that). Tooltips have only been added because it was too late for a real preview back then, see bug 62789 comment 29. You should keep the Description field but make it fixed-height (3 lines).

In the color preview, I would add a "Sample Text" label in default color (black) to one of the fat horizontal rectangles to give an impression for background colors.
The RGB values from the tooltip are not visible any more. They are not too important though, but you could add them somewhere in the preview as well, e.g. in black after the left "Sample Text" label (this would also show a close comparison of a text color with black).

In the font preview, I would display the font name and style in the default font.

See also bug 60717.
Comment 12 Oleg Besedin 2009-03-30 14:47:14 EDT 
Created attachment 130287 [details]
Proposed patch

The patch adjusts layout problems and adds default color and font preview (to be used if no theme preview is available for the category). (See bug 60717 for more details on the preview requirements.)

The patch changes buttons: there are always 3 buttons: "Change" / "Use system" / "Reset" enabled and disabled as it fits the current selection.

To cleanup the dialog and remove non-essential information, the former "details" text is displayed in the tooltips.

There is a resizable sash that allows to change sizes of table <-> preview.

Usually I try to avoid changes to the format of the existing code, but in this case some parts were really hard to read. I adjusted formatting in several places in the patch taking into account that we are not limited to 80 character wide displays anymore.
