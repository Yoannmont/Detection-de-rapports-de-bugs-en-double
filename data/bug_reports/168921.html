Comment 9 Gang Liu 2007-02-01 00:00:20 EST 
By default Birt only supports Latin and CJK characters in PDF report, because only 14 type 1 fonts and CJK fonts are always available for Adobe reader. Birt also support TrueType fonts in PDF report, but these fonts are different from marchine to marchine. So if user wants to display others characters in PDF, user need to choose one of the following solutions:
A. Specify a TrueType font family for the text in Report design.
B. Change the default font setting. Birt get all font evironment by the configuration file org.eclipse.birt.report.engine.fonts/fontsConfig.xml. About detail configuration information, Please refer to the comments in fontsConfig.xml.
In the case, user can select one of the following solution if user can not accept that birt use HeiseiKakuGo-W5 to display Cyrillic characters :
1. Specify a TrueType font which can display Cyrillic for the text in Report design.
2. In the design, User do not specify font for the text. By default, Birt try to use serif(generic family, default font) to display the text. So user can change the font mapping in font-mappings section:
<mapping name="serif" font-family="Arial Unicode MS"/>
3. Because by default, Birt regards Times-Roman as concrete font of serif. Times-Roman can not display Cyrillic characters, so Birt will choose a font by unicode blocks of the character. In all-fonts section, you will find the line:

<block name="Cyrillic" range-start="400" range-end="4ff" index="8">
<mapping name="all-fonts" font-family="HeiseiKakuGo-W5"/>
</block>
If you want use Arial Unicode MS to display Cyrillic, change this block to:
<block name="Cyrillic" range-start="400" range-end="4ff" index="8">
<mapping name="all-fonts" font-family="Arial Unicode MS"/>
</block>
4. Birt also provides a simple way to define default font in M5 build. Add the default block to all-fonts section:
<block name="default">
<mapping name="all-fonts" font-family="Arial Unicode MS"/>
</block>
Then Birt will use 'Arial Unicode MS' to display all undefined blocks.

So user can make the report displayed correctly by configuration.

Comment 22 Gang Liu 2007-02-04 23:04:21 EST 
CJK font can display Russian symbols, perhaps these fonts are not acceptable for you.
I don't think it's hack information. It's by design that BIRT currently only supports Latin and CJK characters when user do not explicitly specify any font and do not change default configuration file. BIRT does not guess or scan user's font environment due to performance reason, user need to config font environment for BIRT sometime. I think it's acceptable. Maybe we will enhance it later.

In this case, Most of your reports include Russian symbols, so you should modify org.eclipse.birt.report.engine.fonts/fontsConfig.xml:
<block name="Cyrillic" range-start="400" range-end="4ff" index="8">
<mapping name="all-fonts" font-family="Times New Roman"/>
</block>
Please make sure to restart BIRT after you change the font configuration.

Actuatlly Time new Roman is a true type font which does not exist in all platform, so we can not make this change in default configuration.
In M5 build, we will also provide a simple way to define all default fonts. Detail Information please refer to the comments in config file.

Thanks for your cooperation and patient
