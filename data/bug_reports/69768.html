Comment 20 Tanya Wolff 2004-07-16 10:39:42 EDT 
No, CDT doesn't use templates.properties, only default-templates.xml with the 
translatable strings extracted to the top of the file in ENTITY fields.
Eg: 
<!-- NOTE TO TRANSLATOR:
* Only these ENTITY strings listed in the DOCTYPE definition
* are to be translated. The rest of this file should remain
* English as it produces compilable code. 
-->

<!DOCTYPE document [
	<!ENTITY withtempvar "with temporary variable">
]>

<template name="for" description="for loop &withtempvar;" context="C Function" 
enabled="true">
for (int ${var} = 0; ${var} &lt; ${max}; ++${var}) {
	${cursor}
}
</template>

This may not work so well if the translated text should come before the keyword 
as in "else &block" -> "else bloco" instead of "bloco else". Do the Ant and 
Java handle these positioning details in translation nicely? I will follow suit 
if that's the proper way to go.
Comment 21 Kevin Barnes 2004-07-16 11:56:19 EDT 
In ant.xml the entire decription is externalized. This allows the translator to replace the entire text. 
Where appropriate, the template name is externalized the same way.
