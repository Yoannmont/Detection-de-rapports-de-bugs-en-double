Comment 9 Markus Schorn 2006-07-10 03:06:35 EDT 
I think the goal is to create the public API for determining callers/callees for a function/method. See bug 149565.

There are already various kinds of objects involved (do we really want to
create yet another):

Finding Callers: Given a function (IASTName) you need to obtain its binding (IBinding) to find the references (IASTNames) for each of which you have to find the enclosing function/method (IASTName).
 
Even more complicated, finding callees: Given a function (IASTName) find all calls within its body (IASTNames), obtain their binding (IBinding) and find an appropriate definition for the binding (IASTName).

That's the way it works in the DOM world, I am not 100% sure whether PDom works the same way. Anyway it'd be my goal to cover that by an API, so that clients don't have to worry about it.

Does that relate to your ideas?
