Description Alex Blewitt 2003-09-03 18:04:12 EDT 
I created a simple 'Hello World' plugin using the Wizzard in Eclipse (3.0M3/MacOSX.2.6), and it ran 
fine. However, when I ran the build.xml file (for generating the JAR to fit into a feature/site), the 
compilation failed.

It transpired that the Ant property 'ws' was not being set, with the result that the SWT was not 
being included on the (build.xml) classpath, and causing a compile time error in the generation of 
the JAR file.

I manually put ws=carbon in the build, and it worked fine.

Have I invoked it wrong, or should the Ant plug-in contribute the 'ws' property so that this build 
works? I suspect that similar follow-on issues may apply for other properties if they are required ...

It may be me configuring/building/defining the features wrong, of course :-)

Comment 12 Wassim Melhem 2003-09-29 09:05:55 EDT 
The values ws,os,arch,nl are set on the ant launch configuration when you 
export or when you create the build.xml manually (via the 'create ant build 
file' context menu) by PDE.
To set these variables, PDE uses the values specified on the plug-in 
development>target environment preference page.
Look at the Properties tab of the ant launch configuration, do the values 
match the ones set on the target environment page?
