Description evansac 2003-02-20 03:06:30 EST 
I'm trying to detect mouse clicks on a TableColumn.

I know you can add a SelectionListener to the column, which is 
activated via a single mouse click, but I'm trying to detect a
double mouse click. I think it's quite common for tables to respond
to a double mouse click by sorting the table by the items in the column
(single clicks are reserved for selecting all items in the column).

If I add a Listener to a TableColumn via:

tableColumn.addListener(Swt.MouseDown, new Listener() ...);

it never receives any events.

See also Bug # 17871 - TableColumn does not send MouseHover and MouseMove events

Comment 7 Andriy Rysin 2005-07-07 10:47:20 EDT 
I was able to get mouse events for column headers with child class for Table
I also was able to get ContextMenu event working and right now finishing the
tooltips for column headers to work.
The Table2 class is attached. This implementation is only for Win32 platform. I
am planning to take a look at possible Linux implementation a bit later.
The class has to be in org.eclipse.swt.widgets package to work.

One can get mouse events for column headers by registering listener with
TableColumn, e.g.:

column.addListener(SWT.MouseHover, new Listener() {
	public void handleEvent(Event event) {
		System.out.println("column Hover event: " + event);
	} 
});


I hope to hear comments from eclipse developers and would ge glad to work on
this to incorporate it to standard SWT package.

I'll post tooltips implementation for context menu and win32 shortly.

The only problem I have right now is it works perfectly for SWT Applications but
when I try to use this code in Eclipse plugin I get this errors like this for
every call to methods from org.eclipse.swt* packages:

Unhandled event loop exception
Reason:
tried to access method
org.eclipse.swt.widgets.Widget.wmMouseMove(III)Lorg/eclipse/swt/internal/win32/LRESULT;
from class org.eclipse.swt.widgets.Table2

Comment 17 Steve Northover 2005-08-10 14:26:52 EDT 
This bug report needs to be broken up into a bunch of separate reports so that 
we can make some progress on the various parts.  Here goes:

Double Click:
The original problem (as stated by evansac@hotmail.com) was that "double mouse 
click" could not be detected in a column header.  However, 
SWT.DefaultSelection can be detected so the original problem no longer 
exists.  Therefore, I'd like to change the title of this report to reflect 
this and mark the bug as fixed.

Mouse Events:
I have updated the title of bug #17871 to reflect the fact that all mouse 
events are not seen in a table header.  The patch that Andriy provided belongs 
that that bug report.  Andriy, can you attacht the latest versions of your 
code there?  Thanks.

Context Menu:
The lack of a context menu in a table header is captured by bug #23103 so 
further work can happen there and this bug report can be closed.

ToolTips:
The lack of tool tips in a table header is captured by bug #4572 so we can 
close this bug report.

Once Andriy has moved his patches to bug #17871, I will retitle this bug 
report and close it.

Comment 26 Andriy Rysin 2006-03-14 11:31:08 EST 
Neil, there were some resource (win32 handles) leaks which I have fix but did not have a chance to post here. Just in case you did not face those yet here's the main part:
			public void widgetDisposed(DisposeEvent e) {
				Menu menu = getMenu();
				if( menu != null && menu.isDisposed() == false ) {
					menu.dispose();
				}
				if (lpstrTip != 0) {
					int hHeap = OS.GetProcessHeap ();
					OS.HeapFree (hHeap, 0, lpstrTip);
				}
				if( messageCallback != null ) {
					OS.SetWindowLong (hwndHeader, OS.GWL_WNDPROC, oldMessageProcForHeader);
					messageCallback.dispose();
					messageCallback = null;
				}
			}
		});
