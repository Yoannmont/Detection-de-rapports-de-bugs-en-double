Comment 1 Chris Brealey 2005-04-05 16:01:27 EDT 
Short answer: Internet Proxy Preferences should migrate to Eclipse 3.2.

Long answer:
Kevin Haaland and I had a brief chat today about the origin, purpose and 
destiny of the Internet Proxy Preferences page currently in WTP.

As far as we know, the nearest and only equivalent in base Eclipse today is 
the "Proxy settings" group of controls found in Preferences -> Install/Update. 
As designed into the JRE and as demonstrated by WTP Web services, the ability 
to configure firewall proxy settings is a general purpose feature and should 
not be tied specifically to Install/Update.

Kevin and I agreed that WTP's Internet Proxy preference page should be migrated 
to the Eclipse base, and that Install/Update's option should either be removed 
or be replaced by a handy link to the common Internet Proxy preference page.

The two preference pages share the JRE's properties and play fairly nicely 
together today, so the purpose of the migration effort is, to paraphrase Kevin, 
one of hygene vs. a fix to broken behaviour.

As Eclipse 3.1 is winding down, Kevin suggested Eclipse 3.2 (and WTP 1.x?) as 
the appropriate time to migrate the page.

Comment 11 Richard Birenheide 2006-07-27 01:52:16 EDT 
When considering resolving this issue, my opinion is, that a hook for temporarily changing the Authenticator should be added, as described in my blog on https://www.sdn.sap.com/irj/sdn/weblogs?blog=/pub/wlg/3937. Unlike the proxy, the Authenticator cannot be set individually per HttpURLConnection. Thus, a hook for temporary changes should be provided as well as a strong recommendation at prominent location _not_ to use java.net.Authenticator.setDefault(java.net.Authenticator).
