Description Wassim Melhem 2005-02-15 05:41:42 EST 
Build: Valentine's Day I-build.

1. Fresh workspace.
2. Create a plugin project with the RCP mail template.  Plugin editor will 
open.
3. Launch it through the link on the overview page of the plugin editor.
4. The app will come up.  Go to About > About RCP Product.  You will see a lot 
of text in the About dialog.  All good.  Close it.

5. Go to the plugin.properties of the plugin.  Change the value of aboutText 
to 'wassim'.
6. Relaunch the app and go to the about dialog.  It will not show 'wassim'.  
It still shows the old text.
Comment 1 Pascal Rapicault 2005-02-15 10:04:29 EST 
This is normal: the value is cached by the registry which tracks what's going on
at the fwk level. Because the fwk never checks the timestamp of this file, the
change goes unnoticed.

In this case I suggest that PDE apply the "manifest" deletion trick, or a touch
the manifest file if it exits.
Comment 2 Wassim Melhem 2005-02-15 16:54:48 EST 
Sorry, this behavior should not be considered "normal" if you launch with -dev.

The problem even occurs when the plugin has a manifest.mf, in which case the 
deletion trick of course doesn't work.

PDE is not in the business of touching manifest.mf files.  This will outrage 
the Eclipse community as it unncessarily shows outgoing changes on manifest.mf 
files.
Comment 3 Jeff McAffer 2005-02-15 23:29:15 EST 
Agreed.  

Would it be reasonable to have PDE delete the .registry file in the 
configuration when it sees a change to plugin.properties?  The 
plugin.properties are cached in there.  Restarting will be slightly slower as 
the registry will have to be rebuilt but its not that bad.

As for -dev, I agree but, the cost for us to maintain plugin.property 
timestamps and just so we could check them if we are sometime run with -dev is 
quite severe.  Currently I believe we do not do anything in this area.

Not punting back to you but seeing if the tooling fix is easier/better than the 
runtime checking.
Comment 4 Wassim Melhem 2005-02-15 23:33:03 EST 
We could certainly do that.  Moving back to PDE.
