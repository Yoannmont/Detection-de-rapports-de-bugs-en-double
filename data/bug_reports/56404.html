Description Robert Danek 2004-03-26 14:14:02 EST 
On Solaris, the Sun 1.4.x jdks have a serious bug in them that prevents us from
running the hyades java profiler. The bug basically arises whenever we request a
JVMPI_EVENT_OBJ_ALLOC during a method entry using the object id  in
event->u.method_entry2.obj_id. (Specifically, it's in the JvmpiWriter.c in the
method processMethodEntryEvent().) There is a bug on sun's site that describes
this problem:
http://developer.java.sun.com/developer/bugParade/bugs/4614956.html. The bug is
closed and labelled as fixed in tiger (java 1.5), which doesn't help us to work
with the 1.4 jvm. 

NOTE: This bug is a different issue from the problem described in bug 56189. The
problem there was also due to a sun bug, but was arising from a different cause
(mainly when performing a heap dump). 

I'm attaching a simple test case profiler that replicates the bug outside our
profiler, which confirms the crashing of our profiler is due to the bug in the
sun jvm. The test case is based on the simple jvmpi_example profiler available
from the JVMPI spec webpage. 


Tested using the following sun jdks on solaris:

java version "1.4.2_04"
Java(TM) 2 Runtime Environment, Standard Edition (build 1.4.2_04-b05)
Java HotSpot(TM) Client VM (build 1.4.2_04-b05, mixed mode)

and

java version "1.4.1_07"
Java(TM) 2 Runtime Environment, Standard Edition (build 1.4.1_07-b02)
Java HotSpot(TM) Client VM (build 1.4.1_07-b02, mixed mode)

Bug 63069 was the readme defect opened for this defect:

Basically, because of a sun jvm bug, when profiling, the jvm may crash. This can
be avoided if the *only* the Execution Analysis mode is used (WITH 'Show
Instance level' info turned OFF), or *only* the Memory Analysis mode is used. If
they are used in combination then the sun jvm bug can arise, or if Execution
Analysis is used with 'Show instance level' info turned on it can also arise.

selected.

> Select Code Coverage, Memory Analysis, Time Analysis > (DEfault filter set)
> Next > Finish
Note new Profiling agent with 3 types in Profiling Monitor (Basic Memory, 
Execution Time, Method Code Coverage)
