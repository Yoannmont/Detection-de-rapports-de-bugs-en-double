Description Eric Kolotyluk 2006-08-28 11:49:52 EDT 
If you go to the Common tab in the Run Dialog three is a section on Standard Input and Output. It is possible to assign stdout to a file, but there is no way to assign stdin from a file. For consistency and completeness there should be a way to define a file you want to assign to stdin, otherwise you have to manually enter a java command in the command shell to test your program.

Comment 26 Markus Keller 2014-12-16 06:51:19 EST 
There were quite a few UI strings problems that I've fixed for M5 with http://git.eclipse.org/c/platform/eclipse.platform.debug.git/commit/?id=ebe8f588bb7735d6f4b51aab477460d1ce3026f8 :

- "Input File" label was missing a colon

- Mnemonics conflicts are acceptable in menus (where the user can just press the letter again to jump to the next matching item). In dialogs, the resolution strategy is to just jump to the first item -> the second occurrence of the same mnemonic is not reachable, so it's better to not show any mnemonic.
Note that there are additional conflicts on PDE launch configurations, since they have more mnemonics on tabs.

- The redundant "Input xxx..." and "Output xxx..." labels in the browse buttons make the buttons look jagged. Either align the buttons or just omit the qualifier as we do it in other places (e.g. "New Class" dialog).
