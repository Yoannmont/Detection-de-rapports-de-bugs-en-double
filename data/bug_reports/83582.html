Comment 1 Leo Treggiari 2005-01-24 21:59:19 EST 
Is the space required with g++ on all versions of Solaris?  The way to add a 
space is to create a new projectType.  When defining the compiler tool, add the 
commandLinePattern attribute and set it to:

"${COMMAND} ${FLAGS} ${OUTPUT_FLAG} ${OUTPUT_PREFIX}${OUTPUT} ${INPUTS}"

The default is:

"${COMMAND} ${FLAGS} ${OUTPUT_FLAG}${OUTPUT_PREFIX}${OUTPUT} ${INPUTS}"

Comment 4 Leo Treggiari 2005-12-14 11:43:49 EST 
> I'm new to eclipse.  So if you could tell me how to create a new project type
or point me where to look at, I would appreciate that very much

The Gnu project types are defined in the file org.eclipse.cdt.managedbuilder.gnu.ui/plugin.xml.  Currently, there are 3 generic GNU projectTypes used for "solaris,linux,hpux,aix,qnx".  You would need to:

1.  Copy those 3 project types and paste them to create another copy.
2.  Remove "Solaris" from the generic Gnu project types.
3.  In the pasted copy, perform the rest of the steps...
4.  Set the osList attributes to only "Solaris".
5.  Change all of the ids - e.g. "cdt.managedbuild.target.gnu.exe" to "cdt.managedbuild.target.gnu.solaris.exe"
6.  For each c.compiler and cpp.compiler "tool" element, add the attribute:
commandLinePattern="${COMMAND} ${FLAGS} ${OUTPUT_FLAG} ${OUTPUT_PREFIX}${OUTPUT} ${INPUTS}"

Leo

Comment 11 Leo Treggiari 2005-12-23 14:09:54 EST 
(In reply to comment #10)
> 1)  On the New Project Wizard, on the type of project selection, the project
> type names for solaris and other unix like OS's are the same.  It looks like
> these names are picked up from plugin.properties file.  I'm hoping that if I
> make the appropriate changes in plugin.properties and plugin.xml file I can
> change the name.  Is there a better way?

The best way to do this is to change the "name" attribute of the projectType.  You can put the name directly in the plugin.xml or use a reference to the plugin.properties.  The CDT code uses the plugin.properties to allow localization of the strings.

> 2)  Is there a better way of adding new project types?  Any documentation will
> help.  Our group uses different versions of compiler, debugger, and OS
> depending on the hardware.  The only commanality is that all are Solaris.  So I
> was hoping that creating a new project type for each might be a solution.

Adding new definitions to the plugin.xml file will allow you to define additional "tool-chains" (compilers, OSs, etc.).  There are different mechanisms for defining a new debugger that I'm not familiar with.  

See the CDT help, CDT Plugin Developer Guide, Programmer's Guide, Managed Build System Extensibility Document for a description of how to define new projectTypes, tools, etc.
