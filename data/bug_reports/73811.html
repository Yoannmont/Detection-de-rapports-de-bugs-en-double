Description Rutger Ovidius 2004-09-13 17:44:26 EDT 
swt 3

In this example scrolling the column with the tableTreeEdtiors will overwrite 
the horizontal scrollbar, and the table column headers.   This does not happen 
on win32 or motif.

In another app they also overwrite the vertical srollbar, but I can't make it 
do so in this example.  PR67236 overwrites the vertical scrollbar.

public class TTE {
  
  public static void main(String[] a) {
    Display display = new Display();
    Shell shell = new Shell(display);
    shell.setLayout(new FillLayout());
    
    TTE tte = new TTE();
    
    final TableTree tt = new TableTree(shell, SWT.FULL_SELECTION | SWT.MULTI | 
SWT.VIRTUAL);
    Table t = tt.getTable(); 
    t.setLinesVisible(true);
    t.setHeaderVisible(true);
    

    for (int i = 0; i < 10; i++) {
      TableColumn tc = new TableColumn(t, SWT.NONE);
      tc.setWidth(100);
      if (i == 8)
          tc.setWidth(500);
      tc.setText("Column " + i);
    } 
    
    for (int i = 0; i < 100; i++) {
      TableTreeItem tti = new TableTreeItem(tt, SWT.NONE);
      addTTE(tte, tt, tti);
      
    }
    
    shell.setSize(300,500);
    shell.open();
    
    while (!shell.isDisposed()) {
      if (!display.readAndDispatch())
        display.sleep();
    }   
    display.dispose();
  } 
  
  public static void addTTE(TTE tte, TableTree tt, TableTreeItem tti) {
    TableTreeEditor editor = new TableTreeEditor(tt);
    tti.setData("editor", editor);
    editor.horizontalAlignment = SWT.LEFT;
    editor.grabHorizontal = true;
    editor.setEditor(tte.new myCanvas(tt.getTable(), SWT.NONE), tti ,8);
  } 
  
  class myCanvas extends Canvas  implements PaintListener {
      
      Image i;
      
    public myCanvas(Composite parent, int style) {
      super(parent, SWT.NO_BACKGROUND);
      addPaintListener(this);
      i = new Image(getDisplay(), 16, 16);
      GC gc = new GC(i);
      gc.setBackground(getDisplay().getSystemColor(SWT.COLOR_BLUE));
      gc.fillRectangle(0,0,16,16);
      gc.dispose();
      

    }
     
    public void paintControl(PaintEvent e) {
      e.gc.drawImage(i, 0, 0, 16, 16, e.x, e.y, e.width, e.height);
    } 
  } 
  
}

Comment 6 Billy Biggs 2005-05-07 17:04:29 EDT 
TableEditor#computeBounds() is not as robust as it could be.  It clips to the
client area by width, but not by height, which is why the horizontal scrollbar
is overwritten.  As well, the width clipping is not very robust, as you can make
the editors overwrite the vertical scrollbar simply by resizing the window
smaller horizontally when the editors are visible.

Fixing computeBounds() will stop the editors from drawing above the scrollbars,
but won't fix the problem of drawing on top of the headers.

Comment 12 Billy Biggs 2006-02-01 01:37:35 EST 
The scrollbars, the headers, and the bin window are all children
of the tree's window.  The scroll bars scroll the bin window.

For SWT's placement and movement semantics, we want to parent
children to the tree's window.  That way they don't scroll "for free"
when you move the scrollbars.  However, you need to fix the Z order
to ensure that children are above the bin window, but below the
scrollbars and the header.

Ideally, children would be under the bin window, but then you need
to dork with their position so that the scrollbars don't move them
around on you.
