Description Mike Boyersmith 2005-05-10 12:42:16 EDT 
As of M3 the Birt tutorial
(http://www.eclipse.org/birt/index.php?page=tutorial/script14.html) is no longer
valid in these areas. 

Below I listed a few areas that need to be rewritten, hope this helps out. 

- The Template Wizard section is out of sync and mentions a wizard page that
isn't there. 

- The Data Set Selection and usage sections are  no longer valid.

- the "create City Group" section is confusing
 the comment "Choose Insert Group from the context menu for the row heading for
the state group header row. The Group Properties dialog again appears. Isn't
clear, and the menuing seems to have changed. 

- The section on Formatting data (phone number) is wrong.
  in particular "Switch to the Format group in the Property Editor." is 
  misleadig.

- Setting Border is wrong, in particular "Select the Borders page in the     
   Property Editor." there is no borders page.

- the section on "Setting Report-Wide Properties throws exception 
org.eclipse.birt.report.model.api.command.NameException occurred
Error Code:Error.NameException.DUPLICATE
  Error Code:Error.NameException.DUPLICATE
  Error Message:The name "report" duplicates an existing name. Please choose a
different name.

the problem being that report already exists and if you try to add it you get
the nasty message above. The tutorial tells you to "Use the context menu and
select New Style" which is what causes the problem. 

- the Create a grid, says "Delete the last row by displaying the grid tab,
clicking on the tab, then on the shaded header to the left of the row. Press
Delete." don't need to do this just specify 1 row and 3 columns.  Then the
tutorial goes on to say "We want the contents of the header to be top-aligned.
And we'll use the second row as a spacer between the header proper and the
report body:" This is after you just told us to only have 1 row. It sounds like
your saying we should have started with a 3x3, and deleted the 3rd row, so we
have a 3x2. These sections are really messed up. 

- Insert the Run Data says 'Set the date format. Select the Format page within
the Property Editor' there is no format page. It also says 'Choose "Medium Date"
for the date format.' hmmm.. don't see that, I see 'format date' though.
Comment 1 Paul Rogers 2005-05-13 15:31:10 EDT 
Tutorial is revised to M3. Now uses the Classic Cars database.
