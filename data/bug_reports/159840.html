Comment 11 Hernan P Dacharry 2006-12-12 11:10:55 EST 
Inpecting the logs I found a workaround that doesn't require to turn off anti-alias, I tryed it in the eclipse/callisto distribution as well as in the latest eclipse / GMF stream stable build.

It seems to be due to a conflict with some of the installed system libraries, the solution was to preload the ones suplied in the eclipse package.

In Debian GNU/Linux testing (etch), using java-1.5.0-sun I created a link in my local eclipse directory to the xawt java library:

ln -s /usr/lib/jvm/java-1.5.0-sun/jre/lib/i386/xawt eclipse/configuration/org.eclipse.osgi/bundles/107/1/.cp/xawt

(the path to the java library depends on the particular version/distribution used... and the bundles subdirectory depends on the eclipse version installed, it should be the one containing the files libswt-* )

and then to execute eclipse I use the command:

LD_PRELOAD="./libcairo-swt.so ./configuration/org.eclipse.osgi/bundles/107/1/.cp/libswt-pi-gtk-3314.so" ./eclipse

(the libswt version number depends on the eclipse version installed)
