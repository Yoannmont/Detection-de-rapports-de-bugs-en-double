Description Igor Fedorenko 2005-11-16 23:26:26 EST 
Steps to reproduce

* start new workspace
* define tomcat 5.5 runtime and server
* create dynamic webapp project "war" and add it to the server
* add JUNIT_HOME/junit.jar to war's Web Libraries
* create server T that classes from junit.jar (see servlet source below)
* start tomcat server and try to access localhost:8080/war/T

Problem: there is no junit.jar in .metadata/.../tmp0/webapps/war/WEB-INF/lib and
requests to the servlet fail with NoClassDefFoundError

Servlet source

package t;
import java.io.IOException;
import javax.servlet.ServletException;
import javax.servlet.http.HttpServletRequest;
import javax.servlet.http.HttpServletResponse;
import junit.framework.TestSuite;
 public class T extends javax.servlet.http.HttpServlet implements
javax.servlet.Servlet {
	protected void doGet(HttpServletRequest request, HttpServletResponse response)
throws ServletException, IOException {
		TestSuite suite = new TestSuite();
		response.getWriter().println(suite.getName());
	}  	  	  	    
}

Comment 16 Tim deBoer 2006-01-26 14:23:18 EST 
Tested on this week's M-build with both jst.j2ee and my latest changes. Fixed a bug in the publish behaviour (released) and found a couple issues remaining in jst.j2ee:
 * There is no way to remove an external jar from the properties page. Unchecking it doesn't apply.
 * The external jar shows up in the root of a returned Web module although the WEB-INF/lib is correct in the path. It should really be contained within the correct ModuleFolder. Where it is causes problems for servers that create folders as it goes instead of checking on each file.
