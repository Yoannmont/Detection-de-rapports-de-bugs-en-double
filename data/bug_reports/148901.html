Comment 2 Chris Rose 2006-06-27 18:23:04 EDT 
No, the web.xml file exists in the war file, but the zip file entry for it is "/WEB-INF/web.xml", not "WEB-INF/web.xml".

It's reasonable to believe that WAS should be smarter about finding this, but the unix unzip utility considers absolute pathnames in a zip file to be worthy of a warning, so it would be best if the WTP didn't build war files with absolute paths.

Comment 24 Bob Hubbard 2006-09-20 18:42:52 EDT 
(In reply to comment #23)

Found my problem. Not sure if this is a new issue for jst or if its an issue with WAS itself.

Your workspace path cannot contain a space. For instance, publishing from an Eclipse workspace located at c:\javatools\workspace will work just fine. However, if your workspace is located at c:\Documents and Settings\bh6507\eclipse3\workspace, you will not be able to publish to WAS.

Verified.
