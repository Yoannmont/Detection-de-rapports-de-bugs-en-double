Description Moses Lecce 2007-08-17 12:29:51 EDT 
When entering an XML definition or any other instance of '<?' in your php code, it will interpret this to be a short tag.  I cannot find a way to disable this behavior to only allow the full php open tag '<?php'.
Comment 1 Roy Ganor 2007-08-19 01:47:29 EDT 
currently we don't support non-short tag documents.

this is a feature request.
Comment 2 Jerico Dev 2007-10-08 13:09:06 EDT 
Hi Roy,

I have got the same problem. I get Errors in my problem list for .tpl-files with XML/XHTML source code that is syntactically correct (...<?xml version="1.0"?>...). 

Well, I know this is a known problem with short tags. Compare http://docs.php.net/manual/en/language.basic-syntax.php#language.basic-syntax.phpmode. 

Don't see this to be a feature request however. Syntactically correct code must not get a failure annotation by default. I think that <? short tag parsing should rather be switched off by default as it is in PHP default config. And the feature request would then be to make short tags (optionally) configurable like ASP-tags.

Florian
Comment 3 Karl Debisschop 2008-06-27 21:55:01 EDT 
Aside from the arguments made already, there is the issue of interoperability with other programs. We have content authors that use an XHTML editor that insists on inserting the <?xml ?> declaration. The developers of that editor are adamant that they will not change. Meanwhile, the PDT developers keeps insisting that they don't want to support documents that contain the <?xml ?>. I'm just trying to get my team to adopt eclipse, and do a decent day's work. It sure doesn't help when PDT throws errors on perfectly valid XHTML files.

Call if a bug, call it a feature, call it an enhancement -- I don't care. Whatever you call it, I'd be awfully appreciative if this could be addressed.
Comment 4 Karl Debisschop 2008-06-28 07:18:29 EDT 
I just noticed that this interoperability issue exists within eclipse itself. That is, from the within the PHP perspective, if I open the PHP Explorer view, and select "New > HTML", eclipse will create an XTHML 1.0 strict file that PDT immediately flags as an error.
Comment 5 Roy Ganor 2008-06-28 14:06:21 EDT 
aaauch, sounds annoying.

I would try to solve this issue by testing the letters after the "<?" in the php lexer. and if it is an "<?xml" token it should reolved as an html tag.
Comment 6 Karl Debisschop 2008-06-30 13:38:01 EDT 
Roy, I think your approach would be OK, except if possible the test should be more like "if there are non-whitespace characters after <? and those characters are not 'php' (case insensitive), then it should be resolved as an HTML tag".

I'm personally affected only by the XML issue, but there were comments on some of the related threads about using other engines for processing (I think the reference was to TEX).

While probably not as widespread, those suggestions would also seem to have a degree of merit. If a case can be made that this sort of code change in PDT can be done and is worth doing, then it would be best to do it with enough generality that it can be changed once, and be done with.

Thanks for your consideration.

Comment 9 Sylvia Tancheva 2010-08-31 10:30:14 EDT 
Short tags can be disabled now:
Go to Project -> Properties -> PHP Interpreter
Enable Project Specific Settings -> disable Use Short tags and rebuild
