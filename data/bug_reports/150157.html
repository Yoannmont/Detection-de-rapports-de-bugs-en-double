Description Jim Graf 2006-07-10 11:20:58 EDT 
For the past six months I have been trying to debug a JVM crash in my RCP application without any success.

We are currently using an AWT-based terminal emulation program, incorporated into our RCP-based program using the SWT_AWT bridge class, in order to allow access my company's existing green-screen system. Our users have complained about unpredictable JVM crashes that occur while they are navigating through the green-screen system. We know for a fact that this issue is not caused by the terminal emulator because for over a year-and-a-half, our users used a stand-alone version of this exact same product without any problems. We are currently talking to the terminal emulator vendors about the issue, but they are starting to doubt that there's anything they can do about it.

The following three sample stack traces were generated by three different users doing three different things in the terminal emulator. The only consistent fact about this bug is that it occurs seemingly at random while they are switching from screen to screen, but that is the only commonality between them. All but one of our users are using JVM 1.5.0_06, one of these logs comes from the user using JVM 1.5.0_03.  Any help or suggestions would be greatly appreciated.
