Description Kathy Chan 2007-06-13 15:45:52 EDT 
Driver:  WTP 2.0 06/13 driver

Set "generate test case" in the Axis2 preference page.  Go through a Web Service Client scenario.  If you click Next, then "generate test case" is properly respected in the wizard.  However, if you just click "Finish" on the first page, then the preference is not respected and test cases are not generated.  Other preferences are also not respected when clicking Finish on first page.

This typically means that logics are put into the UI widgets to read from the preference rather than doing it from the commands (which gets run regardless of whether the UI page is brought up).  Actually, when bringing up the Web Services popup, the user could select "Do not show me this dialog box again" and disable even the first page of the wizard.  This is equivalent to not showing the first page and clicking Finish.  Please test this with the 3 Axis2 scenarios.
