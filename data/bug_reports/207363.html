Comment 16 Konstantin Komissarchik 2008-01-14 12:20:57 EST 
Just dropped a change into the 3.0 M5 code stream that takes advantage of the JFace non-native cell navigation facilities for trees. It is now possible to use keyboard to navigate and fully interact with the tree in the facets selection panel. The space bar activates the checkbox if selection is on the first column and activates the version field editor if selection is on the second column.

The behavior seen when using the mouse has been changed somewhat as the result of this. Since single-click is now cell selection, a double-click is necessary to activate the version field editor.

This change also includes several improvements to the behavior of the version field editor that are designed to reduce the number of clicks necessary to perform various actions. Double-clicking on the version field activates the editor with the drop-down list already open. Selecting an item in the drop-down list using the mouse closes the drop-down and de-activates the editor. Clicking elsewhere while the drop-down is open will also de-activate the editor. 

I have tested all of these changes on WinXP. Since a lot of this is non-native behavior, please report if you see glitches on other platforms. Make sure to include screenshots and detailed descriptions of behavior witnessed vs. behavior expected. 

Finally, many thanks to Tom Schindl for providing the JFace facility that made this possible and for fixing the bugs that I uncovered. See Bug 215069 for details. Note that you may see some visual glitches in this UI until WTP picks up a platform build that includes the fixes tracked by that bug.

Comment 18 Konstantin Komissarchik 2008-03-24 17:11:38 EDT 
I ended up reverting the prior fix (cell navigation to the version panel). While JFace cell navigation support has definite promise, there were too many remaining non-native behavior artifacts to make the change be worthwhile in this particular application. 

So, back to native behavior (no keyboard navigation between columns). To support accessibility, I added a popup menu that lets the user modify facet version via a separate dialog. Since it is possible to bring up the popup-menu using keyboard (Shift-F10 on windows) and the resulting dialog is keyboard-accessible, I believe that the base accessibility requirement is satisfied.
