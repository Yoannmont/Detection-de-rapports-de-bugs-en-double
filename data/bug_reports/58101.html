Comment 24 Kim Moir 2006-07-12 14:37:48 EDT 
Sonia and I looked at the code you submitted and have some suggestions for improvement

Instead of having a template file for each operating system with generated html from java could, would it be possible to have a php script that you pass a value to generate display the appropriate page?

Also, it's not very scaleable to have specific platforms referenced in the java code.  It would be better if they could be passed as properties from the ant script.  That way, we don't have to recompile code to generate additional content, just update the ant script. Or instead of properties in an ant script you could load a properties file.

Also, one thing that we forgot to mention is that some teams use our scripts for 
running their builds.  So it would be great if this could be kept in mind, for instance, if you need to change or add additional functionality, you could use an overloaded method or another approach to keep the existing functionality.

Although it's nice to have a download page in the Phoenix look, it's very confusing from an UI point of view if you "abuse" common navigation elements of that design. AFAIK, the top "tab" bar is reserved for Eclipse.org navigation.
