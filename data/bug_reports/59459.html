Comment 5 Tom Hofmann 2004-06-16 05:04:09 EDT 
Unable to reproduce with RC2, on RedHat 9.0 / GTK, however. 

I suspect that the clipboard was in a bad state when you reproduced this - for
example, an other application could have locked the clipboard or such.

One fix would be to guard against clipboard failure instead of falling all the
way out - however, then the clipboard problem would never be communicated to the
user.
Comment 6 Tom Hofmann 2004-06-16 05:20:06 EDT 
We should really guard against DND.ERROR_CANNOT_SET_CLIPBOARD everywhere we use
Clipboard.setContents as this exception can always occur like a checked
excpetion. A fix will be to catch these and handle appropriately:

- ask the user what to do
- log
- propagate where we cannot do either of the above
