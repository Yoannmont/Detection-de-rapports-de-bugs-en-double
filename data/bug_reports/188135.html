Description Martin Oberhuber 2007-05-21 13:11:07 EDT 
Eclipse 3.3RC1 on Linux-GTK-x86_64, Sun 1.5.0_10 JVM, TWM window manager
shown through a VNC virtual server.

Launch Eclipse SDK, open several editors. When the tab sheet cannot hold all of them a little ">>" indicator on the right can be clicked on to get a tooltip showing the list of those editors that are not part of the tabs any more.

When I click that indicator with the mouse, and then try to move the mouse down into the tooltip, it disappears as soon as I hit the boarder of the yellow tooltip window. This makes this feature close to unusable for me.
Workaround: use the new keyboard navigation by typing in first few characters of the editor to be opened.

I checked, and found the problem the same in Eclipse 3.2.2 already (did not check earlier releases though).

Comment 4 Martin Oberhuber 2007-05-22 05:44:07 EDT 
More news: I tested on the console of the affected machine and found that it works properly with KDE and fvwm, but fails with twm. I then tested on another 32-bit RHEL4 Linux machine and found that it works with mwm but fails with twm. So it's quite clear to me that the problem is due to the twm window manager. I'm changing the summary accordingly and reassigning to SWT (again).

My twm is configured with the defaults (focus-follows-mouse) while all other window managers do focus-on-click only. I can work around the issue by specifying the following in $HOME/.twmrc:

  NoTitleFocus

But this makes twm look really ugly. The twm manpage says for NoTitleFocus:

  This  variable indicates that twm should not set keyboard input
  focus to each window as it is entered.  Normally, twm sets  the
  focus  so  that focus and key events from the titlebar and icon
  managers are delivered to the application.  If the  pointer  is
  moved quickly and twm is slow to respond, input can be directed
  to the old window instead of the new.  This option is typically
  used  to  prevent this ``input lag'' and to work around bugs in
  older applications that have problems with focus events.

So, valid workarounds for the issue are setting NoTitleFocus, or using another window manager like fvwm, twm or KDE.
