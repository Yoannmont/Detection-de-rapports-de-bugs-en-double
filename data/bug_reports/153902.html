Comment 33 Anthony Hunter 2006-10-06 08:50:52 EDT 
Hi Team, good news that I have been able to resolve this issue.

The solution was two step. 
First was to back out the bundle loading implemented to resolve Bug 152159 and Bug 135995 . This means both are reopened and printing was broken.
The second was to rewrite the DLL loading code in the PrintHelper and perform some movement of the DLLs in the fragment.

After completing the code changes, I built the plug-in as a jar and deployed the jar in 1.0.1 on RedHat Linux and Windows XP. The jar file had to be carefully constructed for the test to match the version number of the plug-in in 1.0.1.
On RedHat Linux, observed that the error message no longer appears and Configuration Manager shows no errors.
On Windows XP, tested the printing and printing DLL on the ten languages supported by Eclipse by running eclipse -nl ja (repeated for en, de, es, fr, it, ja, ko, pt_BR, zh_CN, zh_TW).
