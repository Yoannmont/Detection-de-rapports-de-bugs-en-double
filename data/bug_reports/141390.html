Description Robert Elves 2006-05-11 15:24:01 EDT 
In the conversion to xml/rdf we lost the real names associated with the user email addresses. We need some mechanism for harvesting the user names and email addresses from bugzilla.
Comment 1 Mik Kersten 2006-05-30 17:54:21 EDT 
Jeff, are you interested in taking this on?
Comment 2 Jeff Pound 2006-05-30 18:05:36 EDT 
Sure, I'll have a look at it.
Comment 3 Mik Kersten 2006-05-30 20:11:09 EDT 
We considered running a dummy query which specifies that names should be returned along with email addresses (we have support for running custom queries like that, it's how we figure out what bugs have been modified and should be synched).  Then we would keep a cache of names that was lazily updated.  But it would be much better if there were a simpler solution.
Comment 4 Jeff Pound 2006-06-01 16:03:58 EDT 
The ideal solution here would be to have the real names in the xml stream. I've filed https://bugzilla.mozilla.org/show_bug.cgi?id=340001 as a request for that.

In the mean time we could use some type of screen scraping/custom query/local cache technique if it's important to have this feature. The only thing I don't like about that is it's 2 trips to the server per bug.
Comment 5 Mik Kersten 2006-06-01 16:09:11 EDT 
Let's hold off on this until we see what the Mozilla folks say.  

I'm marking the priority down to indicate that.  In general don't bother looking at P3 bugs unless P2s and P1s are done.
Comment 6 Mik Kersten 2006-06-05 15:52:09 EDT 
Jeff, could you summarize what it would take to do this, given the info we got back from the Bugzilla folks?
Comment 7 Jeff Pound 2006-06-05 17:04:58 EDT 
Well, it doesn't look like the real names can be retrieved along with the query XML, so we'd need a second request to the server. The easist way I can think of would be to grab the HTML page and just regex match the mailto email addresses to get the real names. We could cache the names per session or even persist them.

If custom installs obfuscate email address we might not be able to match the real names though, we could just default to email addresses in that case.
Comment 8 Mik Kersten 2006-07-05 22:24:17 EDT 
Could you check the status and vote on the Mozilla bug to encourage them to address this?  It is a very straightforward fix on their end and my last comment still hasn't had a reply.
Comment 9 Robert Elves 2006-12-08 21:42:28 EST 
The report (https://bugzilla.mozilla.org/show_bug.cgi?id=340001) appears to be resolved with a target milestone of 3.0 (their next major release).
