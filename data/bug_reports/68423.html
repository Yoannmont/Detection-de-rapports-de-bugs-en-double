Comment 17 Himanshu Mishra 2015-01-22 04:01:33 EST 
I have analyzed the bug and found out that the when the focus is on the Text Widget and we press Ctrl + Z then the latest property that is ID or Name has not triggered the ModelChangeEvent so it does not go to the Undo/Redo Stack. But if we perform a Focus Lost event or press enter, then you can undo/redo becuase the change has been committed using the commit() method on both the events. If we try committing on ModifyEvent then that happens with every key press, so every key press will go to undo/redo stack. I also tried to handle the keypress event of Ctrl + Z to commit as soon as we press Ctrl+Z in FormEntry Text class. But it behaves inconsistently. So can someone tell that how should we handle such a situation??
