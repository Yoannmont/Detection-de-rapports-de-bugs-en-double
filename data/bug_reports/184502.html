Comment 6 Markus Keller 2007-05-11 09:15:14 EDT 
Paul, I don't think it's a good idea to define a new context and keybinding in JDT/UI.

First, Ctrl+F10 should open a dialog's "view menu" everywhere, so the support should be in the workbench. Unfortunately, PopupDialog cannot directly support it, since that class is in JFace, so I guess the workbench would have to offer a keybindings-aware version of PopupDialog. And there are also other dialogs that should use the feature (e.g. FilteredItemsSelectionDialog, bug 96943).

Furthermore, the fix would add a second key binding for the "org.eclipse.ui.window.showViewMenu" command, so the user would have to change two key bindings if he wants to open the menu with another shortcut.
Comment 7 Paul Webster 2007-05-11 09:55:03 EDT 
(In reply to comment #6)
> Paul, I don't think it's a good idea to define a new context and keybinding in
> JDT/UI.
>

Another option would be to just move CTRL+F10 down to org.eclipse.ui.contexts.dialogAndWindows in platform.  Then both the keybinding and the context get handlered automatically, and the popup menu only has to submit a handlers.

Let me spin that patch.
