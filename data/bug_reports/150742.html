Comment 7 Thomas Watson 2006-07-18 12:06:45 EDT 
Looks like having a '!' char in the install path of eclipse is causing lots of things to go wrong.  The work around should be to move the eclipse installation from D:\!projects\ebXML\eclips to D:\projects\ebXML\eclipse without the '!' char.

What VM are you using?  It seems like the URLClassLoader is giving us some problems with the classpath URLs with '!'.
Comment 8 Thomas Watson 2006-07-26 14:15:37 EDT 
The original NPE posted in commont 0 has been addressed in bug 151413.

Don, can you confirm that extracting eclipse into a path without a '!' fixes your startup problem?

Comment 10 Lev Olkhovich 2006-07-26 15:12:07 EDT 
As for 151888, removing an exclamation mark fixes the bug there (reproduced on 2 different machines with Java 1.4.something and 1.5.0_02, both WinXP).

I think I traced the problem back to java (I'm using the 1.5.0_2 version):
Class.getProtectionDomain().getCodeSource().getLocation() (invoked from org.eclipse.core.runtime.adaptorEclipseStarter, line 131) returns an URL with an exclamation mark unescaped.

However, when replaced with "%21", URL.openConnection() seems to work, so there might be a workaround.
Comment 11 Lev Olkhovich 2006-07-26 15:23:08 EDT 
ops... a typo. I meant URL.openStream().

Also, I forgot to write: an exclamation mark causes problems because it's considered to be a separator in case of a "jar:..." URL. So it's not the classloader that fails, it's the URL.openStream() that fails when '!'s are unescaped.
Comment 12 Thomas Watson 2006-08-30 09:14:04 EDT 
This is a known issue and is covered in the README file shipped with Eclipse.

Also see bug 3109 and bug 17281.
