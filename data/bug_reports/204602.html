Comment 16 Kathy Chan 2008-05-13 14:53:18 EDT 
Summary:

After the Web Services Explorer (WSE) was switched to use the internal Jetty engine rather than the internal Tomcat engine early in WTP 3.0, the WSE plugin accounts for 552 files and 69 folders (it was packaged as a WAR to Tomcat in WTP 2.0.x and therefore the problem does not exist in earlier releases).  This adds to WTP download footprint unnecessarily.  We have attempted to switch the WSE from a directory to a JAR plugin earlier in WTP 3.0 but ran into problem because of how the file favorite_defaults.wsil was handled.  Now with the fix for bug 231351, we can finally JAR up the WSE plugin. 

The fix is straight forward involving changing the feature.xml, manifest.mf and build.properties to produce a plugin JAR rather than a plugin directory.  Gil has tested this with a test build.

PMC, please approve for WTP 3.0 RC1.
