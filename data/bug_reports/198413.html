Comment 1 Oliver Schoett 2007-08-01 13:19:52 EDT 
Additional information: it is not the JBoss server that crashes but the ACServer process.

I find that I can tunnel properly when I map port 10012 on the Eclipse machine to port 10002 on the ACServer machine.  However, that is the old agent protocol which needs to establish a reverse connection and hence cannot profile through a tunnel.

Mapping port 10006 on the Eclipse machine to port 10006 on the ACServer machine does not transfer any data to the remote machine.  I am under the impression that all connections to the local machine talk to the local IAC.

It seems that the default agent protocol on a nonstandard port number is the old agent protocol, and that I have no way to speak the new protocol over a port number different that 10006.  Since that port number is absorbed locally by the IAC, I cannot tunnel the new agent protocol through a firewall.

Maybe there should be an option to select the protocol (old vs. new) independently of the port number?

Comment 14 Igor Alelekov 2007-08-15 09:24:46 EDT 
Hi Oliver,
don't you mess network ports a bit?

It seems that you have configured new host (Preferences...) in your client eclipse on Windows with port value 10006. RAC port should be provided here.
10006 is the default port for new AC services and is not ready to process RAC requests yet.

Try to substitute it by 10002 :)

I'll submit a patch shortly to prevent such problems in future.
