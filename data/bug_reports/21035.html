Description Barry Lulas 2002-06-26 16:51:30 EDT 
Currently, there doesn't seem to be any way to distingwish key events coming 
from the keypad or the normal keys.  For example, I would like to map an action 
to the plus key on the numeric keypad and still be able to type the + key on 
the normal keyboard.  I should be able to do the following:

<accelerator 
  key="KEYPAD + '+'" ="org.eclipse.ui.edit.copy">         
</accelerator>

I am currently writing the Brief accelerator table and would be willing to 
provide it to the product as soon as this functionality is implemented.

Comment 6 Hans Deragon 2003-07-14 18:32:07 EDT 
What I am asking for is for have an official solution (hopefully an easy one for
the developer) to solve this problem.

It is important that when someone enters a number using the numeric pad in a
text field, that when the numeric pad "." key is pressed, the decimal separator
as defined in the locale shows up.

Ideally, the widget would take care of this.  The developer would only need to
call a function to enable this behavior.  If this is not possible for
"philosophical reasons", then some means must be provided to the developer to
achieve this (and please, not too complicated).

If normal Window behavior is crooked, then we must find a solution to go around
it, like the Excel spreadsheet does.  Ideally, Windows would be fixed, but good
luck influencing Microsoft...

BTW, OpenOffice has the same problem: 
http://www.openoffice.org/issues/show_bug.cgi?id=1820

Happy Birthday bug.  You are now 1 year old. :)
