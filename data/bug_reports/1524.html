Comment 14 Joe Szurszewski 2002-10-30 15:17:24 EST 
Fixed.  Annotations are created and applied to the appropriate editor when a stack frame is selected in the LaunchView.  This annotations exist until the owning thread or debug target is terminated or resumed, at which time they are removed.  This approach is very fast.  Once consequence is that if an editor is opened in some way OTHER than by selecting a stack frame, no instruction pointers will be seen in that editor (until the corresponding stack frame is selected).  Note that top stack frames are always programatically selected on a suspend event, such as the end of a step.  Another consequence is that if multiple stack frames are associated with one editor, the user only sees instruction pointers for those stack frames that have been selected since the last resume event.  

At present, there are 'temporary' icons used for the top stack frame instruction pointer and the non-top stack frame instruction pointer, pending arrival of 'official' icons from the artist.
