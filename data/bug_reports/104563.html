Comment 6 Tim deBoer 2005-07-24 15:07:42 EDT 
Hi guys,

Behaviour delegate instances should never be disposed/recreated while a server
is running. They should be reused as much as possible, even when changes are
made using a working copy (this is the one exception of course - the other
delegates [not the behaviour] are created and disposed for each working copy).
Since this is targetted for M6 please fix and try to get PMC approval or defer
as soon as you can.

Comment 11 Tim deBoer 2005-07-26 08:35:59 EDT 
Hi guys,

Ok, I think I see the problem now. The working copy should not have it's 'own'
behaviour delegate. In fact, the IServerWorkingCopy specifically doesn't include
methods that should access the working copy - since you can't edit the behaviour
delegate the same way that you can with the regular delegate, we don't allow you
to create a working copy of it.

However, it makes sense that the working copy will sometimes require a behaviour
delegate or will be adapted to one. The correct behaviour for a newly created
server working copy would be to create one, and then pass it down to the actual
server that is created. If the user then creates a working copy from that
server, it should just reuse the existing server behaviour.

I agree that the correct change is too late in 0.7 to be safe, so let's go with
the proposed fix here and do the full change after 0.7.
