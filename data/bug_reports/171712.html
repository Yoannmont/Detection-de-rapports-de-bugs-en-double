Description Sarah Gerweck 2007-01-25 12:39:25 EST 
Build ID: 3.2.1 (M20060921-0945)

This is a JVM crash, which occurs very frequently, making Eclipse unusable. Other Vista users seem to be having the same experience. My Vista build (6000) is the final release.

I have not observed a pattern that makes this bug reproducible, but clicking buttons in the UI in Vista, especially rapidly, seems to trigger it. If you use Eclipse in Vista, you will encounter this bug within an hour.

If developers don't have Vista yet, I'm more than happy to test any patch. If Eclipse is usable, it will be quite clear that the bug is fixed.

I read somewhere that this bug doesn't affect 3.2.0, only 3.2.1, but I haven't verified this myself.

More information:
This happens with a fresh download of Eclipse 3.2.1. I do not have any plugins not included with the standard download. Using auto-update to get the latest patches did not help.

Here is the process dump. I have several, and they all indicate COMCTL32.dll+0x5c1f4 as the offending access.
