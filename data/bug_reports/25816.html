Comment 26 Johan Walles 2002-12-07 16:39:31 EST 
I know what's wrong (Woho!).  We are using gcc 2.95.3.  Its c++filt (attachment
2684 [details]) has problems outputting through a pipe.  It outputs all the characters
correctly, but the trailing newline is buffered, and is not output until it gets
another piece of input.  It can be reproduced like this:

c++filt | cat

If you type first "one" and then "two", you get:

one
onetwo

two

... instead of the expected...

one
one
two
two

If it outputs directly to a terminal it works fine.  This has been fixed in
c++filt from newer versions of gcc.

The question is whether you should work around this, or just ignore it (since it
isn't your fault).  From our point of view, we can work around it ourselves now
that we know what's happening, so if you decide on not working around this
that's fine by us.  What talks *for* you working around this is that for those
who (like us) use older gccs, the CDT appears to be entirely broken (and you'll
probably get the blame).

If you are interested in working around this it could probably be done by
tricking c++filt into thinking it is outputting to a terminal.  That could be
done by doing tcsetattr(fd, TCSANOW, &termios_s) on the fd to which c++filt
outputs it data.  fd is (obviously) the file descriptor in question and
termios_s is a termios struct (from termios.h).  Unforturnately I have no idea
about what the termios struct's fields should be set to, but somebody at your
place may.

Comment 31 Brent Nicolle 2003-09-29 14:30:42 EDT 
I'm adding three notes here so I can close the duplicate defect:

1) How To Diagnose This Problem.

If you get the following response, you're fine:

% c++filt --version
GNU c++filt (C++ demangler), version 2.12

If you get the following kind of response, you need to put Gnu's c++filt before 
Sun's c++filt:

% c++filt --version
can't read file "--version"
% c++filt -V
c++filt: Forte Developer 7 C++ 5.4 2002/03/09
^D


2) Severity
Sun C++ Developers may very easily have Sun's Forte Developer IDE installed.
CDT requires Gnu to appear on the path before Forte, otherwise it hangs.


3) The Future Fix.  Alain wrote:
>The new implementation will let you change the path of c++filt and addr2line
>and disable them if the toolchain are buggy with a minimal loss of
>functionality.

Please bear in mind that it took Johan & Alain a month to identify the
original problem in this defect.  Having some advanced option to fix or
sidestep the problem (equivalent to changing the user's path) does not
make it easier for a user to *diagnose* their problem.  Very few users will 
deduce that the reason why CDT is hanging is because of the available
version of c++filt.

I strongly urge CDT performs something like the kind of version checking 
done above in "How To Diagnose This Problem".  

Agreed, this is not the kind of change you want to introduce in the end-stage
of 1.2.
