Description Dan Rubel 2003-02-27 11:37:28 EST 
When running Eclipse Ant in headless mode, you cannot use a workspace that has 
been touched in any way at any time by the Eclipse UI.  For example, if you 
shutdown the Eclipse UI and then run a simple Ant script using the workspace 
that you were using with the Eclipse UI, you will get an exception and the build 
will fail.  This works in WSAD v5 just fine.  Using the following steps to 
recreate the problem:




1) Place the attached ant_test.bat, ant_test.xml, and start_ui.bat files into a 
new directory




2) Modify the attached ant_test.bat script for your environment


3) Execute the ant_test.bat script


4) Verify that the files ant_test_out.txt, ant_test_err.txt, and 
ant_test_log.txt are generated as a result of executing the script and that the 
ant_test_log.txt file contains the text "It works!" verifying that the script 
successfully executed in a headless environment.




5) Modify attached the start_ui.bat script for your environment


6) Startup the eclipse UI with the same headless workspace that was used in 
steps #1 through #4 above using the attached start_ui.bat script or something 
like it


7) Create a new java project (any name will do)


8) When it asks if you want to open the java perspective, click yes.


9) Shutdown the Eclipse UI, leaving the java perspective and the welcome file 
open




10) Execute the ant_test.bat script again


11) Verify that this time the ant_test_log.txt file contains "BUILD FAILED" 
indicating (obviously) that the simple Ant script did *not* execute correctly.  
The file should contain something like this...




BUILD FAILED


org.eclipse.core.internal.boot.DelegatingLoaderException: 
org.eclipse.core.runtime.CoreException: Problems encountered starting up 
plug-in: "org.eclipse.core.resources".


	at 
org.eclipse.core.internal.plugins.PluginDescriptor.internalDoPluginActivation(Pl
uginDescriptor.java:746)


... etc ...




I'm guessing that some UI state has been captured in the workspace metadata and 
then reloaded during the headless Ant execution, causing UI plugins to be 
inadvertently loaded.
