Description Kresten Krab Thorup 2006-09-06 07:30:53 EDT 
I would very much like to be able to work on a project which is targeted at a "foreign" JRE on my Mac.  I want to do this, so that I can access JRE-specific classes in a given project.

For example, I have tried to simply copy all files in a JRockit installation to a Mac OS X box, and then add this as a JRE.  The dialog says the text "Standard VM not supported on MacOS".

I know that I will not be able to execute code with this JRE, I just want to be able to compile against these files in a convenient way.

Comment 3 Alex Blewitt 2007-02-08 16:31:19 EST 
FYI you can still use the Mac VM to achieve this, but then add the JRockit VM's jars as libraries. The Mac's classes will take precedence (since they're on the boot classpath) but you'll also be able to see any specifics in the Jar of the attached bundle.

To keep JDT sane, you should probably ensure that the Mac's VM comes first in the .classpath though ...

Comment 23 Darin Wright 2009-08-20 10:41:50 EDT 
This looks correct - as JDK 1.7 is a super set of SE1.6. I guess the real test would be to add a Standrd 1.5 JRE and see if it shows as an exact match for SE1.5 and does not show up as compatible for 1.6/1.7.

The caveat here is that the code that determines the "execution environment level" for a StandardVM has to launch the JRE to introspect it (list of some system properties). If the JRE is not executable, this will not work (however, it looks like the JREs are actually executable on the Mac OS).
