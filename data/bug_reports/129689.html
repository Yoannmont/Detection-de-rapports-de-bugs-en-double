Comment 3 Thomas Watson 2006-03-01 09:24:29 EST 
This is a PDE-UI issue.  When the RCP launcher is first created it includes the set of bundles which the rcp application currently requires.  When you add a new requirement the launcher is not updated to include that new required bundle.

To workaround the issue you should:
- Open the launcher for the RCP application.
- Goto the Plug-ins tab
- Click on "Add Required Plug-ins"
- Save and run the launcher again.

Comment 8 Brian Bauman 2007-08-15 18:10:24 EDT 
I will summarize Wassim's proposed fix.

Many people take great time crafting their launch configurations.  For these users, we don't want to start messing with their configuration if they have customized it.

So, if the user initiates the action by clicking the launch shortcut on the Manifest Editor and the launch configuration has not been previously edited, then we can assume the user would like PDE's help in launching the plug-in and its dependencies.  For this case we can run the "Add Required Plug-ins" functionality and add any new dependencies to the launch configuration.
