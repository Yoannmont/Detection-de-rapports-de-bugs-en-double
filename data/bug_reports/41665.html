Comment 3 John Arthorne 2003-08-19 15:46:37 EDT 
It looks like the property store data is corrupt.  (Possibly due to concurrent
access from that PerforceSynchronizer thread).  As Rafael suggested, we need to
improve the loop termination condition in StoreKey.readNullTerminated to check
for end of stream.
Comment 4 John Arthorne 2003-08-19 17:47:19 EDT 
I have released a fix to StoreKey.readNullTerminated to avoid the infinite loop.
 You will now get an exception 

However, this unfortunately means your property store is corrupt.  If this
happened several weeks ago I'm guessing it will be very difficult to diagnose
the problem.  Essentially there is a file in your .metadata directory called
.properties that somehow became corrupt.  Deleting the file will fix the
problem, but you will lose persistent properties such as classpath variables. 
This might be better for you that starting from scratch.
