Comment 1 Andrew Niefer 2006-06-29 11:07:57 EDT 
When attempting to download the pack.gz file retrieveLocalJar catches IOException and then just downloads the normal jar.

During a normal update, when a progress monitor is being used, a FatalIOException is thrown from FeaturePackagedContentProvider.asLocalReference line 228 when trying to get the size to display in the monitor.  This is caught properly and we proceed with the normal jar.

When there is no progress monitor, (ie standaloneUpdate), FatalIOException is thrown by checkConnectionResult, but it is caught by asLocalReference and wrapped in a CoreException, which is not caught by the retrieveLocalJar.
Comment 2 Andrew Niefer 2006-06-29 11:19:05 EDT 
Note, a workaround is to set the system property to turn off pack200:
-Dorg.eclipse.update.jarprocessor.pack200=@none
