Description Duong Nguyen 2002-01-16 10:35:34 EST 
When a WSWB session is open with some opened, dirty .txt file, shutting down 
Windows will cause the loss of the changes in the .txt file.

Steps To Reproduce: 
start WSWB and create a new project
create a text file in the project and add a line of text to it.  Notice that 
the  text file is marked as a dirty file
Left click on the Windows Start menu, select “Shut Down…” command to shut 
down/reboot the computer

Results:

There is no warning or question message box to give users a chance to save 
their dirty model(s), the computer is shut down/rebooted immediately.  Other 
Windows programs (like Word, Outlook etc. provide a “Do you want to save your 
file”, Yes, No, Cancel (which halts the shutdown) dialog box.

Expected Behaviour:
Should provide a “Do you want to save your file”, Yes, No, Cancel (which halts 
the shutdown) dialog box.

One other nice feature might be to allow the applications to control the 
shutdown/save also or to pop up there own windows that would also (if cancel is 
selected) allow the user to stop the shutdown.

Comment 4 Steve Northover 2002-04-11 11:58:51 EDT 
I have released the code.  When the window system is shut down, on Windows 
only, Display.dispose() is called.  Applications can use Display.disposeExec() 
run a runnable that can prompt the user.  Note that the runnable runs after all 
shells in the display have been asked to close().

If you still believe that you need API to cancel window system shutdown, please 
enter a separate DCR - "Need to be able to cancel window system shutdown".  
Thanks.

Comment 11 Knut Radloff 2002-05-15 11:08:25 EDT 
Assuming that the disposeExec mechanism will be changed slightly we can save 
the workbench properly. Currently the disposeExec runnable is called after the 
shells are disposed which prevents us from running the regular 
recordWorkbenchState code. Asked SN if this can be changed.

As for the editors. I will present a list of dirty editors. The user can select 
which ones to save but cancel won't be an option. There is no SWT support to 
cancel the shutdown.
Comment 12 Knut Radloff 2002-05-16 10:50:45 EDT 
See bug 13622. Apparently we want the shells disposed before the display 
dispose runnable is executed. 
This means we can't completely save the workbench state. Namely the window 
sizes/positions and maximize state would be missing.
We should probably still save all the other workbench state info. The windows 
would just revert to the default when you restart the workbench after rebooting.

Windows or logging off.  The stub is there but it has not been implemented on 
any other platforms.
