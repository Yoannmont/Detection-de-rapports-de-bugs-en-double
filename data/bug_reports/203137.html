Description Raishad 2007-09-12 15:57:09 EDT 
Created attachment 78228 [details]
Test Report

Build ID: I20070625-1500

Steps To Reproduce:
1. Create a library
2. Create a data source in the library with report parameters in the property binding section, ex. JNDI Name
3. Create a report that links to the data source
4. Create a dataset that uses the data source linked from the library
5. Create a report parameter that uses the data set as well
6. Run the report with the correct URL, passing in the Data Source Parameters

BIRT will not be able to connect to the data source when generating the report, but will be able to connect when populating the list box parameter.

If you then move the data source to the report design file and re-reun the report, BIRT will be able to run the report successfully.


More information:
Here is my setup
I'm using property binding to create the JNDI name for a data source at runtime. The data source is defined in a library. Here is the statement,
"java:jdbc/"+params["DATABASE"]+"_"+params["SCHEMA"]
DATABASE and SCHEMA are parameters that are passed to the report. It seems that when it comes time to generate the report, the property binding is not happening - I get no data back.

But, the property binding is happening when BIRT is generating the parameter page. The list boxes are being filled with correct values.

When I copy the Data Source from the library to the report design file, the parameter binding is happening, and I do get correct data back.

Attached is a sample design and library file. I'm deploying the report on JBoss, and for my ds xml file, I have

<?xml version="1.0" encoding="UTF-8"?>
<datasources>
    <local-tx-datasource>
        <jndi-name>jdbc/S1_F1</jndi-name>
<connection-url>jdbc:jtds:sqlserver://server:1433/Northwind;useLOBs=false</connection-url>
        <driver-class>net.sourceforge.jtds.jdbc.Driver</driver-class>
        <user-name>user</user-name>
        <password>password</password>
        <min-pool-size>25</min-pool-size>
        <max-pool-size>100</max-pool-size>
    </local-tx-datasource>
</datasources> 

I added some System.out's on the Data Sources BeforeOpen event
Like
java.lang.System.out.println("beforeOpen LibDS");
java.lang.System.out.println(this.getAllExtensionProperties());

As you can see, in the Parameter Page section, both Data Sources are the same. They have the same odaJndiName. But when it comes time to run the report, the Library data source (LibDS), does not have a odaJndiName, but the Locally defined data source (LocalDS) does.
