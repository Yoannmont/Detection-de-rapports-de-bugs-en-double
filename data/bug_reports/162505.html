Description Randy Hudson 2006-10-26 21:28:44 EDT 
I don't want to create yet another instance of a markerview, but I don't see an easy alternative. I need to display feedback/comments about a resource. Each comment is a marker, yet it doesn't fall under the category of bookmark, todo, or problem. So, our current approach is to display a new view called feedback. This view needs to support all of the common function of sorting, grouping, filtering, etc. that is free in MarkerView, hence the request to make it API.

I would prefer that there be a single generic marker view with multiple modes, as discussed in bug 37974. Then we could just define a new mode, give it a name, and define the marker types that belong in it. But the above request seems more likely to happen.
