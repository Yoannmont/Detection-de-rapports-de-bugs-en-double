Description Douglas Pollock 2003-09-12 09:24:34 EDT 
Pressing a key in a dialog freezes keyboard input in Eclipse.  Keyboard input
continues to work in other applications.  Keyboard input does not reach the
global key filter used by Workbench to process keyboard shortcuts, nor does it
reach the widgets themselves.

ENVIRONMENT:
locale="zh_CN.GB18030", Eclipse-GTK, GNOME/Metacity, RedHat 9.0, Sun JRE 1.4.2


STEPS TO REPRODUCE:
1.) Open a source file using the mouse only.
2.) Type random letters in the source file.
3.) Press "Ctrl+F" to open the Find/Replace dialog.
4.) Press "Esc" to close the dialog.
5.) Type more random letters into the source file.


ACTUAL RESULTS:
In step #2, keys are processed by the Workbench's global key filter and the
letters appear in the source file.  At step #4, no events are recived by the
global key filter and no letters appear in the source file.

EXPECTED RESULTS:
The event processing pipe should continue to flow.
