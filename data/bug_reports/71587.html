Description Nick Edgar 2004-08-06 12:56:00 EDT 
R3.0

- select a type name in a Javadoc comment
- Ctrl+Space
- it shows various completions, including <code> - </code>
- type 'c' to select this one (intuitively, this is what I thought would narrow
it down)
- it ends code assist and replaces the selection with 'c' instead
- '<' has a similar effect

Suggestion:
If the text selection is non-empty, then typing should just narrow down the list
of code assist possibilities, rather than affecting the document.

It would also be nice if for tags like <code>, the '<' could be omitted from the
prefix.
