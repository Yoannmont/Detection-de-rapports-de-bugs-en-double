Description Chris Reynolds 2007-07-12 07:39:05 EDT 
Is there any way to increase the timeout setting for a JIRA repository? Or failing that any other suggestions to resolve my problem.

Set up Mylyn last night (at about 6pm so unlikely anyone was using our Jira at the time) and managed to add task list using queries built using the form and from saved filters and it seemed to be working fine. Today every time I try and synch or an auto synch happens I get 'Query Synchronization Failed. java.net.SocketTimeoutException: Read timed out'. There would be approx 30 people using it periodically during the day but doubt we have much concurrent use apart form maybe a couple of people who assign incoming bugs and run occasional queries.

Have tried the following:
Updating attributes for the repository - this didn't give any errors.
New query using other saved filters - same error.
Creating new queries using the form - same error.
Connecting to your test jira server (http://mylyn.eclipse.org/jiratest), set up a query to return ALL issues using the form – it started to download tasks but I got bored before it finished.
Set up another query for your test server that returned less issues (54) and it sync'd ok.
Set up JiraDashboard plugin and used it to run several queries including the same saved filter that is failing above and it worked ok. The saved filter took about 12 seconds to return the results.

To me all of this points to a slow server apart from the dashboard part. I don’t get why that works when the same filter through Mylyn times out.

Any info or suggestions would be much appreciated.
Comment 1 Steffen Pingel 2007-07-12 10:34:48 EDT 
The socket timeout used by Mylyn is 30 seconds. If no data is received for that amount of time the connection is closed and the error you have described occurs. Mylyn should probably use a longer timeout for queries since some servers might need some time to compile the results before any data is sent.

Comment 3 Steffen Pingel 2007-07-12 12:02:34 EDT 
My mistake, your measurements are very accurate, Mylyn actualy uses a 30 second timeout for connects and 17 seconds for read timeouts. 

Mylyn uses the XML output of the web interface to perform queries whereas the dash board might use the SOAP service instead. It shouldn't take longer than with a regular web browser though. Could you check how long it takes to display the XML output in the browser (perform a query and select XML in the Issue Navigator above the results).

The timeout can only be changed by modifying the code. It is set in WebClientUtil.SOCKET_TIMEOUT.

Comment 19 Steffen Pingel 2007-09-17 02:10:11 EDT 
From a quick test run against mylyn.eclipse.org it appears that in the case of not using view=rss the whole response is assembled in memory first (leading to server errors when too many results are requested) and then transfered quickly (~300 kb/s). When view=rss is passed results are streamed right away but the transfer rate is much lower (~30 kb/s).
