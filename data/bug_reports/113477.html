Description Navid Mehregani 2005-10-23 13:03:02 EDT 
There are problems with executing a Java application in controlled/enabled mode
from the command prompt and attaching to them using IAC.  In both cases the
agent does actually show up in the agent list, but there are various problems
caused after attaching to the agent.

When the application is executed in 'enabled' mode, no data seems to be
collected by IAC.  When the application is executed in 'controlled' mode and the
agent is attached to, the application doesn't seem to detect the attachment and
doesn't continue its execution.
Comment 1 Navid Mehregani 2005-10-27 13:06:56 EDT 
Actually, the main problem is that the agent doesn't even show up in the agent
list because IAC has not started.  We don't start the IAC when the workbench
starts and when the application is executed under enabled/controlled mode it
never gets registered with IAC. 

If an application is profiled in launch mode with IAC and before that
application terminates, the user starts another application in
controlled/enabled mode, then the agent DOES get registered with IAC so this is
obviously a problem with the way we start/stop IAC.

We should revisit the usability guides for IAC and consider providing the user
with a Start/Stop button in the workbench for both IAC and RAC.  I'm not sure if
this can be contained in 4.1i3 because it will require significant UI changes.
Comment 2 Hendra Suwanda 2005-10-28 10:46:22 EDT 
Revisit in 4.2 (with usability review)
Comment 3 Hendra Suwanda 2005-11-14 11:08:19 EST 
Bug 114831 fixed the "Java - Attach" for profiling under "controlled" and
"enabled" mode.  Sometime you need to hit "Refresh" button twice to get the
updated agent list (See README in bug 115527)

It does not fix the Logging Agent problem.  This is still caused by the way IAC
starts up.
