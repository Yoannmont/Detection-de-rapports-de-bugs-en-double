Description Kaloyan Raev 2007-09-20 12:35:28 EDT 
According to the Servlet 2.5 specification, chapter SRV.11, the only meaningful URL mappings are the ones that begin with "/" or "*.". Any other string cannot be resolved by a web container as a url mapping. 

At the moment, there is no validation of URL mappings in the web.xml descriptor. The web.xml validator (if there is one at the moment) should also check whether the mappings begin with "/" or "*.". This also should be reflected in the validator of the Project Navigator meta data tree. 

The severity of the message is WARNING, but not ERROR, because meaningless mappings do not prevent deployment of the application, but only can cause unexpected (for the user) behavior during runtime.

Comment 3 Kaloyan Raev 2007-10-17 09:48:05 EDT 
I want to generalize this enhancement bug to cover the validation of the "servlet" and "servlet-mapping" tags. 

The validation includes, but not cover completely, things like: 
  - validation of the servlet class (if exists)
  - validation of the servlet name (if is empty, or in case of mapping - non-existing servlet)
  - validation of the url pattern of the mapping
  - etc (consult the Servlet spec and Glassfish reference implementation)
