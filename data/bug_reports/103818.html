Description Ernest Pasour 2005-07-14 10:30:25 EDT 
I'm using 3.1final.  I've been seeing this bug at least since 3.0 final.  I
*think* I only started seeing this bug after I installed WindowsXP SP2, but the
timing may have been coincidence.

I have 3 computers, all with windows xp
A. home
B. work-main (this is where my eclipse runs)
C. work-secondary

I started noticing the problem using Remote Desktop over VPN from A to B.  When
I would do ctrl+C and then do Ctrl+V elsewhere in the file, sometimes nothing
would get pasted.  Note that I did not get the previous clipboard contents, I
got nothing.  After a little investigation, I found that when the failure
occurs, the clipboard is not empty.  Instead it has the appropriate text, but
ONLY in RTF format.  There is no Text format data.  This error is intermittent.
 If I keep hitting ctrl+c, eventually the text will be copied, so I feel it is a
timing problem.  

Interesting corollary.  Occasionally, I copy some text and try to paste it to
another file in eclipse.  When the failure occurs, I still seem to get imports
added to the target file, if new ones were needed (I have "add imports on paste"
turned on).  So some code seems to be looking at the RTF format.

Another case I discovered recently was that I created a Remote Desktop session
from B to C, unrelated to any eclipse work.  I started seeing the same copy
problem on computer B.  So it seems like Remote Desktop is treating a timing
problem or getting in the way somehow.  

No coworkers have encountered these problems that I know of.  I have not seen
this problem with other applications, but Eclipse is where I do most of my
cut/paste work.  I'm finally filing this in the hope that there is some simple
fix, because this is really annoying.
