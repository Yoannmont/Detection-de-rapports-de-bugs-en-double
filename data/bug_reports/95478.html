Comment 34 Veronika Irvine 2005-06-09 12:41:40 EDT 
When a second UI thread is used for the Help browser, the Browser hangs on 
setting the property "RegisterAsBrowser" to true.  This is done to "indicate 
whether the object is registered as a top-level browser for target name 
resolution".  This was added for Help under bug report 46751 - I believe it 
was to support OpenWindowListener.

We need to investigate what kind of work is being done as a result 
of "RegisterAsBrowser".
