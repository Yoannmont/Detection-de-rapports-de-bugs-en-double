Comment 18 Jerome Lanneluc 2004-08-25 03:33:26 EDT 
From the spec, an AbstractMethodError is 
 * Thrown when an application tries to call an abstract method. 
 * Normally, this error is caught by the compiler; this error can 
 * only occur at run time if the definition of some class has 
 * incompatibly changed since the currently executing method was last
 * compiled.
The code in JDT Core was compiled without any problem. However the VM throws 
this error. This is obviously a problem with the J9 VM. You should report this 
problem to the VM provider.

Sorry guys, but I don't see what JDT Core can do.
