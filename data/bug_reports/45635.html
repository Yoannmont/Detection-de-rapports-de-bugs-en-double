Description Kai-Uwe Maetzel 2003-10-27 18:06:20 EST 
I20031023 + editorWork stream

- change AnnotationBarHoverManager to use rectangular hover windows
- start eclipse
- in target create a line with multiple hovers
- hover over the icon shown in the vertical ruler for that line
-> hover appears

1) when going to the right with the cursor the hover stays active, going
   to the left let it disappear
2) the appearing hover should behave like a toolbar in respect to the
   tooltips that are shown for the individual icons:
   a) tooltip appears below the icon
   b) tooltip is aligned with the cursor
   b) tooltip is not closed when going from one icon to the other with the
      cursor
   c) we need a general preference for showing affordances (still an open
      issue). When preference is set to false, the affordance tooltip for 
      double clicking is avoided.
3) Hover window does not close when cursor leaves the window to the left.
4) The hover must listen to marker changes. Example: The breakpoint manager
   can move breakpoints to a valid position if the requested one is not valid. 
   This operation can be successful or fail. Both must reflect in the hover
   without having it to reopen.

Comment 2 Tom Hofmann 2003-10-29 11:21:17 EST

Comment 3 Tom Hofmann 2003-11-03 05:22:04 EST 
The following items have to be looked at before merging this into HEAD:

a) Dynamic update of the rollover popup (see previous comments, point 4:
breakpoints may be changed from a background thread while the popup is showing)

b) Accessibility: invoke the rollover popup using a keyboard command, provide
keyboard navigation inside the popup.

c) API the current implementation adds the IAnnotationListener interface which
lets clients listen to Annotation selection and default selection events.
Discuss wether and if yes how we integrate this interface into the editor
infrastructure.
