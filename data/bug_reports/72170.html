unknown

1) As root I went to Help->Software Updates->Find and Install->Search for New
Features.
2) I added the C/C++ IDE  "http://update.eclipse.org/tools/cdt/releases/new" as
a new remote site.  It wouldn't connect because I hadn't set my proxy.  
3) I set my proxy to "blah-blah.net" in Window->Preferences
4) I went back to the Update page and now the C/C++ IDE "+" box expanded and
showed "CDT 2.0".
5) Because the little checkbox next to "CDT 2.0" wasn't obvious at first (yes, I
was dumb), I quit eclipse.
6) I erased the workspace/ directory to start over.  (It seemed reasonable at
the time...)
7) I went back to Help->Software Updates->Find and Install->Search for New
Features.  The C/C++ IDE entry was still there, so I tried to expand it and
Eclipse crashed!  (see below for trace)
8) I went to Window->Preferences and set my proxy to "http://blah-blah.net".
9) I went back to Software Updates and tried to expand the C/C++ tab and eclipse
crashed again.
10) I went to Window->Preferences and set my proxy to "blah-blah.net" (no
"http://" this time)
11) I went back to Software Updates and expanded the C/C++ tab with no problems.

Everything works now!  So - Eclipse is really picky about getting the proxy
setting right.  It'd be nice if it produced a nice error message instead of
completely crashing...
