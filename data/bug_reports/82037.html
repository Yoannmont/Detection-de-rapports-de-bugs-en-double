Description Christian Nelson 2004-12-30 20:25:36 EST 
The xml editor fails to fetch DTDs and XML schemas when it must communicate
through an HTTP proxy, even if preferences / internet / proxy setting is
correctly specified.

While at work, all of my HTTP traffic goes through a proxy.  I must also
authenticate to use the proxy.  When I specify all of this information under
preferences, it the XML editor still fails to validate.

If I turn on the proxy, and connect directly (i.e. no proxy exists), I am still
able to validate.  This makes me think the XML validation simple doesn't honor
those settings.

I would think that *all* HTTP communication should honor those settings, this
way it can be set once and for all for all plugins / features.

Christian

Comment 13 Lawrence Mandel 2005-07-27 16:10:58 EDT 
The WTP will cache resources when they are successfully retrieved. The XML, XSD,
WSDL, and DTD validators use this functionality. There may be other validators
and some editors that currently do not. If so we will have to fix them post 0.7.

As of RC2 the cache is enabled by default. Prior to RC2 you must enable the
cache by going to Window->Preferences->Internet->Cache.
