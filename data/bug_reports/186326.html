Description tqiu 2007-05-10 05:25:38 EDT 
To reproduce:
1. start demo.db which comes with SCS drop6 install;
2. from workspace, create an ASA connection profile, input a wrong value for "database name" field, as long as specify the correct host, port, user name and password, ping succeeds;
3. when connect the profile, will see the wrong database name show in EE.
4. if do not input any value for "database name" in step 2, will see an empty entry name in EE.
Comment 1 tqiu 2007-05-10 05:26:30 EDT 
1.The ASA connection profile with wrong value of "database name" can be ping successfully and will display a wrong database name in EE if connecting the profile; 
2.If not inputing any value for "database name", Empty entry name will be displayed in EE. 
   The first problem has been fixed. For the second issue, I suggest that we should display the default database name in EE if user dones't input any value for "database name". If you agree with my suggestion, the code could be changed in line 47-49 of class org.eclipse.datatools.connectivity.sqm.internal.core.util.DatabaseProviderHelper like this: 

   if (databaseName != null != 0 && databaseName.length()) { 
                        database.setName(databaseName); 
                }

To fix the problem that If not inputing any value for "database name", Empty entry name will be displayed in EE.

We found that the setName() get called twice seperately in To

 public void setDatabase (Connection connection, ConnectionInfo info, String databaseName)
	{
        Database database = getCatalogDatabase (connection, info);   //initialize the SybaseASACatalogDatabase and set its name to default "asaDemo"
        database.setName (databaseName);    //set the databaseName from user's input.
	}

so i think this should be the best place to resolve this CR.
