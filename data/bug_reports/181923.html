Comment 14 Mik Kersten 2007-04-19 18:07:42 EDT 
Paul: I checked our display filter and that's not the problem.  All it does is store the time of the last event to determine if the user is active.

Looking for the wierdest thing that Mylar does on task activation, I tracked this down to the following code:

try {
    PlatformUI.getWorkbench().getActiveWorkbenchWindow().getShell().setEnabled(false);
     ... open editors ...				
} finally {  				
    PlatformUI.getWorkbench().getActiveWorkbenchWindow().getShell().setEnabled(true);
}

I added this along with some other tweaks in order to work-around Platform bug 138666.  I have no work-removed it and just set the workbench to not redraw during this operation.  

So on the Mylar end this is resolved.  But should I file this as a Platform bug?  It seems like the menus should not disappear just because the shell was disabled?
