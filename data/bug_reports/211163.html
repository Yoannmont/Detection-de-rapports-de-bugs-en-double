You should push the join conditions into an actual join condition rather than putting them in the WHERE clause. The optimizer will usually do that for you, but if it doesn't, you'll end up with a cross join (very slow!), which is then filtered down. For example, the above would look something like so:

select 
 products.name,components.name,profiles.login_name,bugs.bug_id,
 length(longdescs.thetext),length(attach_data.thedata) 
from
 bugs join longdescs using(bug_id) join attachments using(bug_id) join attach_data on attachments.attach_id=attach_data.id join profiles on profiles.user_id=attachments.submitter_id and profiles.user_id=longdescs.who join components on bugs.component_id=components.id join products on bugs.product_id=products.id
where 
 and bugs.bug_id=209410

Also, you can get more information about what the optimizer does with the query if you prepend "describe" before the select.

It probably doesn't matter for this specific query so much now, but I notice the same problem in the later queries too.
