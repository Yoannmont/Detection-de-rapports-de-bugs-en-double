Description benson margulies 2006-03-08 19:03:54 EST 
As far as I can determine, there is no clean way for an IDrawingStrategy to operate on the background color of the text. This is a desirable alternative for annotations. Perhaps an IDrawingStrategy2 could have an API that returns a StyleRange, and then the annotation manager could remove it when the annotation is not being displayed.
Comment 1 Tom Hofmann 2006-03-09 03:02:39 EST 
While it is true that an IDrawingStrategy cannot modify the background color, annotations still support this: you can specify the highlighting color in the markerAnnotationSpecification extension point. 

The background color can be set by the user for annotations that choose to be shown in the text editor preferences.

Closing as works for me as the request ("Annotations should be able to work via background color") works. Does this solve your problem? Please reopen if this does not work for you.

Comment 8 Tom Hofmann 2006-03-09 10:46:52 EST 
The presentation is updated anytime that TextViewer receives a changeTextPresentation message, in particular when the presentation reconciler updates the presentation.

-> you need something to trigger the text presentation change, otherwise this is not going to happen. The best way is to have a presentation reconciler; see SourceViewerConfiguration::getPresentationReconciler.

Comment 10 benson margulies 2006-03-09 13:24:35 EST 
I’ve spend another two hours in here, and I think that I have a legitimate subject of defect/feature discussion. If this message doesn’t persuade you, I’ll go away.

For all other aspects of annotation presentation, the presentation maintains itself when annotations are added or removed. Only the background highlight behavior requires there to be a changeTextPresentation to trigger the display.

I respectfully submit that this, by itself, is a legitimate feature request: adding or subtracting an annotation with a highlight should in and of itself result in 'damage' so that the display updates the highlight, just as it updates things drawn via a drawing strategy.

To get around this, I tried manually calling changeTextPresentation. First, I ran into a sequencing problem. There is an asyncExec in AnnotationPainter.modelChanged that put the painter 180% out of sync with the rest of the code. When I asked to display annotations, they didn't show up until the second time that I went through the process. I got around that by pending my call to changeTextPresentation with my own asyncExec.

Well, OK, now I can get my annotations to appear, but never to disappear.

Documentation would be a big help here. The whole background annotation aspect is barely described in the prose documentation.

2) for now, some explanation of how to make background highlights work. Here's what I've ended up with.

The asyncExec in the following might be pointing to a bug by some standards or just a doc opportunity.

I might be able to spend some time and create an example out of this pile of stuff that could be added to the doc.


public void representAnnotations() {
		final SourceViewer sv = (SourceViewer)getSourceViewer();		
		final TextPresentation tp;

		ITypedRegion range;
		try {
			IDocument d = sv.getDocument();
			if(d == null)
				return;
			range = sv.getDocument().getPartition(0);
		} catch (BadLocationException e) {
			return;
		}
		
		tp = new TextPresentation(range, 1000);
		// start by wiping it out. If we ever want to have multiple annotation types offering to annotate
		// this whole business will need some rethinking.
		tp.addStyleRange(new StyleRange(range.getOffset(), range.getLength(), null, null));
		
		// due to annoying code in the core, we have to defer this to ensure that it runs AFTER
		// the code behind model changed, which is already in an asyncExec.
		// 
		Display d = sv.getTextWidget().getDisplay();		
		
		if (d != null) {
			d.asyncExec(new Runnable() {
				public void run() {
                    // we belive that this will cause the viewer to interact with the painter and
					// pick up the annotations.
					sv.changeTextPresentation(tp, true);					
				}
			});
		}		
	}
