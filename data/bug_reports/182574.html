Comment 7 David McKnight 2007-04-18 13:37:28 EDT 
I managed to reproduce this using the Sun JRE although not consistently.  When duplicate queries happen concurrently, the second one causes the first to cancel and that ends up giving us the empty list.  I've changed the code so that the first query is able to continue while the 2nd one becomes a no-op, relying on the first for it's results.  I can't reproduce this anymore but I can't be entirely sure it's gone.  If you see it again, please reopen this.
