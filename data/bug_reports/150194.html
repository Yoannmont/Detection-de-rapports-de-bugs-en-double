Description Tim deBoer 2006-07-10 15:14:39 EDT 
Maybe I've missing something, but I don't understand how a component would go about determining whether a given server would support a project's facets before creating the server's runtime. For instance, a Web Service wizard might want to determine if Tomcat supports a given project before it creates a Tomcat server to run the contents of the project. The only way that I can currently see how to do this is to create the runtime and call IRuntime.supports(), but we need a way to do this before the runtime exists.

Please let me know if API exists (and what it is) or how this should be done. The implementation that I'll need to fix given the result of this bug is FacetUtil.verifyFacets() in jst.server.
Comment 1 Konstantin Komissarchik 2006-07-10 15:33:47 EDT 
That's right, there is currently no way to do the "supports" check without having an instance of IRuntime. While I can certainly add such api as part of the re-work of the runtime api for 2.0, the use cases such as this should really be re-examined. Why? Facet support is determined by exactly which runtime components and versions end up in the runtime. The problem is that you cannot know this before the runtime is created. Right now, based on which runtime wizard is selected, you know the server runtime type and version, but you do not know the java runtime version. That is only determined when the user selects a java runtime inside the wizard. In 2.0 this becomes even more pronouced as runtimes will be able to have any number of runtime components and we need to support the use case of users creating runtime of any version from a single wizard (for example, launch "New WebLogic Runtime" wizard, point at install location, we figure out the rest). With the wizard in full control over what set of runtime components and versions will compose the created runtime, there will be very little you can assume before hand and we will need to revisit the UI that currently relies on being able to make such assumptions.

Comment 4 Tim deBoer 2006-12-11 16:18:38 EST 
FWIW, with the current server tools API you can create a runtime and call the API to check - as long as you don't call save() the runtime never existed. Konstantin's right that this doesn't work in cases where the user might pick the JRE version, etc. At best we could provide a best-guess API or one that would check the minimal runtime components.

At some point we should probably check with the web services team, since they have some 'complex' cases where they try to do project-server mapping in their wizards before either are created... they expect this info to be available beforehand and then simply create everything when the user hits Finish. As you might expect this doesn't entirely work today :), but it actually comes pretty close.
