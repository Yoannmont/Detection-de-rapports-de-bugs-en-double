Description Fabrizio Giustina 2005-09-25 10:27:52 EDT 
Don't know if this should be considered a bug or an enhancement request, anyway
the current eclipse behaviour it's pretty annoying...

when you use the ${pageContext.request.contextPath} expression in jsp files the
editor report an error in the gutter: "The method getContextPath() is undefined
for the type ServletRequest".

That could actually be considered correct, since the getRequest() method in
PageContext returns an instance of ServletRequest, and not HttpServletRequest.

Anyway, in jsp pages or tags the instance is always an HttpServetRequest and
methods in the HttpServetRequest interface are commonly used (and that
expression is perfectly valid).

The getContextPath() method is expecially needed when linking files in the
webapp context, using something like:
<img src="${pageContext.request.contextPath}/image.png" />


In order to fix this the jsp editor should always consider instances of request
and response returned by the pageContext object be HttpServletRequest or
HttpServletResponse instances.

[the same behavior can be found in WTP 0.7 and M8]
