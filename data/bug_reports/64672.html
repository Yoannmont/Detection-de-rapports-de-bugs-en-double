Comment 5 2004-05-30 03:51:13 EDT 
This is what I did to work around the problem:

1. In an older Eclipse build (M9), I exported my code formatter settings to 
file.

2. I then deleted the org.eclipse.jdt.ui\code_formatter_profiles.xml file in 
my workspace.

3. I launched I200405290105 using the old workspace, went to the Code 
Formatter Prefs page (which did not NPE now) and imported that profile.

It seems that I200405290105 is not using the code_formatter_profiles.xml file 
any more, but instead storing directly into the 
org.eclipse.core.runtime\.settings\org.eclipse.jdt.ui.prefs file.  This is a 
backward compatibility issue.

PB
