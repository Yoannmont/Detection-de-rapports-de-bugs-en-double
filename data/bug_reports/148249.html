Description Casey Watson 2006-06-22 12:57:58 EDT 
I have seen this bug numerous times this week. When Stepping through lines of code, the debugger often loses state. The green line still highlights the previous line of execution, however the "Resume" button in the debug view is not enabled.

In the debug view, the paused thread item is closed/compressed so the stack frame is no longer highlighted. Once you expand the thread and click on the stack frame, the debugger continues where it was.

Comment 10 Darin Wright 2006-07-07 12:23:14 EDT 
This program exhibits some issues with multi-thread debugging, although I'm not sure it's the exact same issues reported in this bug.

* place a breakpoint in method SomeOther.m()
* debug the program

Problems

* Sometimes, the first thead (thread 20) suspends and gets selected, but the debug view reveals the second suspended thread (thread 80)
* As thread 20 is resumed, thread 80 is selected (next suspended thread), but when thread 20 is re-suspended, thread 80 scrolls off the screen (appears to scroll down the number of frames that have been inserted in the newly suspended thread, which pushes thread 80 off the bottom of the view)
* As threads terminate, and thread 80 is selected, the number of threads decreases, scrolling the suspended thread up the view, which moves the selection from the suspended frame to another thread, clearing the variables and making resume/step buttons disabled (no longer applicable)

package junit.samples;

public class OneHundredThreads {

	class OneThread implements Runnable {

		int fId = -1;

		public OneThread(int id) {
			fId = id;
		}

		public void run() {
			for (int i = 0; i < fId; i++) {
				try {
					Thread.sleep(1000);
				} catch (InterruptedException e) {
				}
				System.out.println("Thread: " + fId + " " + i + " of " + fId);
				if (fId == 120) {
					new SomeOther().a();
				}
				if (fId == 180) {
					new SomeOther().a();
				}
			}
			System.out.println("Thread: " + fId + " COMPLETE");
		}

	}

	class SomeOther {
		public void a() {
			b();
		}

		public void b() {
			c();
		}

		public void c() {
			d();
		}

		public void d() {
			e();
		}

		public void e() {
			f();
		}

		public void f() {
			g();
		}

		public void g() {
			h();
		}
		
		public void h() {
			i();
		}

		private void i() {
			j();
		}

		private void j() {
			k();
		}

		private void k() {
			l();
		}

		private void l() {
			m();
		}

		private void m() {
			System.out.println("M");
		}
	}

	public static void main(String[] args) {
		new OneHundredThreads().doit();
	}

	public void doit() {
		for (int i = 100; i < 200; i++) {
			Thread thread = new Thread(new OneThread(i));
			thread.start();
		}
	}

}
