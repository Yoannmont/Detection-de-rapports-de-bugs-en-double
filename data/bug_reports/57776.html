Description Cohan Sujay Carlos 2004-04-07 13:31:45 EDT 
If a FileDialog is opened thrice using the FileDialog.open method, the files in 
the directory are shown only the first two times in the dialog; the third time 
open is called, only the subdirectories are shown.

The testcase attached below demonstrates the problem.

When the testcase is compiled and run, a shell with a button appears.  Click 
the button.  A file dialog appears displaying the files in the C: root 
directory.  Press Cancel.  Press the button again to make the FileDialog 
visible a second time.  Cancel.  Press the button a third time.  On the third 
attempt, it will be seen that none of the files in the directory are listed in 
the FileDialog.  Only the directories under C: are displayed in the dialog.

This bug affects SWT version 3035 on the J9 VM (J9 version 2.2).  In SWT 
versions 3040 and thereabouts, the files are not seen even the first time the 
dialog is opened.  The file type drop down list has '*.*0*.*' instead of '*.*' 
and there are some other options that seem to contain garbage.

In SWT version 3009 and earlier, the same testcase works flawlessly.  When run, 
the file dialog lists all the files and subdirectories in the C: root 
directory.  The file type drop down list shows '*.*', which is again correct.

PLATFORM: Win 2000
VM: J9 2.2 build of 20040405
The bug is present in the 304x versions but not in 3009 and previous versions.

// The Testcase:

import org.eclipse.swt.*;
import org.eclipse.swt.events.*;
import org.eclipse.swt.graphics.Font;
import org.eclipse.swt.widgets.*;
public class SWTFileDialog {
	public static void main(String[] args) {
		Display display = new Display();
		final Shell shell = new Shell(display, SWT.CLIP_CHILDREN | 
SWT.CLIP_SIBLINGS | SWT.SHELL_TRIM);
		shell.setText("Das erste Fenster");
		
		final FileDialog f = new FileDialog(shell, SWT.OPEN);
		
		shell.setBounds(0, 120, 160, 100);
		Button button = new Button(shell, SWT.PUSH);
		button.setFont(new Font(display, "fixed_garbage", 10, 0)) ;
		button.setText("Klick mich!");
		button.setBounds(0, 0, 120, 30);
		button.addSelectionListener(new SelectionListener() {
			public void widgetSelected(SelectionEvent e) {
				System.out.println("Danke");
				f.open() ;
			}

			public void widgetDefaultSelected(SelectionEvent e) {
				// TODO Auto-generated method stub
				
			}
		});
		shell.addListener(SWT.Close, new Listener() {
			public void handleEvent(Event event) {
				System.out.println("Close event passed by!") ;
			}
		}) ;
		shell.open();
		while (!shell.isDisposed()) {
			if (!display.readAndDispatch()) {
				display.sleep();
			}
		}
		display.dispose();
	}
}

Comment 2 Cohan Sujay Carlos 2004-04-07 14:05:24 EDT 
We have a workaround for this now.  The file dialog works fine if 
setFilterExtensions is called every time a file dialog is instantiated, as 
shown below:

		final FileDialog f = new FileDialog(shell, SWT.OPEN);
		f.setFilterExtensions(new String[] {"*.*"});
