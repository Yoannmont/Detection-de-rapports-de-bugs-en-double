Comment 5 John Graham 2007-11-13 12:12:52 EST 
I've seen this problem with the MySQL 5.1 driver ("mysql-connector-java-5.1.0-bin.jar"). I changed the code in org.eclipse.datatools.connectivity.internal.DriverUtil to ignore the UnsupportedClassVersionError, and the expected driver classes where returned. Not sure why some classes in the jar throw this exception, but skipping it (or perhaps just logging a warning that we couldn't read the particular class) seems better, since it lets the dialog complete without error.
