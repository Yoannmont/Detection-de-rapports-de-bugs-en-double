Comment 38 Bogdan Gheorghe 2007-04-11 13:14:43 EDT 
It looks as though the gtk_enumerate_printers hang is a GTK implementation problem (for details look here: http://bugzilla.gnome.org/show_bug.cgi?id=428665).

It boils down to this: if the current GTK version you are using makes use of the Lpr backend, you will experience a hang when making a call to any of the Printer methods that calls gtk_enumerate_printers (getDefaultPrinterData(), getPrinterList(), gtkPrinterFromPrinterData()).

We've released a fix for this on our side that tries to make the best of this: enumerate printers will work as expected in all cases except ones which have Lpr. If we encounter the Lpr backend we terminate the enumeration and return what we got up to that point. It turns out that this is not as bad as it sounds, as that for all of the GTK's I've built, it seems that the CUPS backend supersedes the Lpr backend.

Fixed > 20070411
