Comment 1 David Williams 2006-12-15 20:36:01 EST 
Kathy, remember that the "cure" for permgen memory errors is to set 

 -XX:MaxPermSize=128m 

Well, in addition to using one of the more recent Sun JRE's (e.g. 1.5_09) which 
I assume you are doing. 

These errors are likely a sign that 1) there are some (possibly new) class loader leaks in someone's code ... these are hard to track down, as are filtered from most profilers (see http://blogs.sun.com/fkieviet/entry/how_to_fix_the_dreaded )

But ... we can do it if you still see the problem with 
 -XX:MaxPermSize=128m 

It's is also helpful to use a tool such as the one attached to bug 92250. 
With that tool, it's normal to see the permgen space increase the 'first' time you do some action, but if you see it 'each' time you do some action, then there's your leak ... well, you know, helps narrow it down. 

But, it may simply be 2) a sign that more classes are being used with the M4 version of code. Some of the class loading mechanism in the OSGI framework layer have changed this release ... so, that's a possibility. If this is the case, you'll see (using tools such as the one mentioned above) that permgen allocation increases for a while but then levels off (even at some higher level than with M3).
