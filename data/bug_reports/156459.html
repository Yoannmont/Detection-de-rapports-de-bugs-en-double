Description Steve McInerney 2006-09-06 21:06:08 EDT 
Using normal X11 select and paste (via middle button click) is inconsistant. Often enough to be vexing: the previously selected text is pasted instead of the just selected text.
eg. Select abc, MB click, get abc. Select xyz, MB click, get abc. Select xyz (again), MB click, get xyz.

Sometimes it works, sometimes it doesn't. It *appears* that selecting over white space can trigger the failure where you get the previously selected text.

Can "duplicate" the behaviour on both OpenSuSE 10.1 and Kubuntu 6.06.
Both are using build: M20060629-1905
FWIW, was getting the same behaviour with the previous Eclipse release (3.1?).

This is under CDT. Unsure if this info is useful or relevant.

The selection is usually grabbed via LB select&drag, vs LB double+ -click.

Have not observed the incorrect behaviour within the rest of the desktop environment. Probably do more select n paste in Konsole than eclipse FWIW.

Comment 12 Julian Morrison 2007-05-23 10:54:18 EDT 
I have this bug on my single-processor Linux/x86 laptop with Eclipse 3.2.2 (iteratively upgraded from 3.1.something via the built-in upgrader).

The bug also manifests with mouse-select, control-C, control-V. It behaves identically whether the selection is created by double clicking or by mousedown-drag-mouseup. It's as if some selections are being ignored entirely. However, it does *not* happen using shift-cursor.
