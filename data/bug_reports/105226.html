the:   debug="on" 
for the project that has the problem, but not the other projects, see below: 
(in case anyone else has this problem)

The difference is now, that I can set breakpoints in the project, and compile 
and when I run debug mode, it opens an extra tab pane of my class that has the 
breakpoint, only this extra pane is a "compiled" view with a little compiled 
icon on top of its tab. The breakpoint itself is absent in this compiled view,

source file tabbed pane.

which is why I call it a work-around rather than a fix.

<target name="compile" depends="init" description="compile the source ">
    <!-- Compile the java code from ${src} into ${build} -->
    <javac srcdir="${src}" destdir="${build}" debug="on">
      <classpath>
	<!--fileset dir="/usr/share/java/">
	  <include name="**/*.jar"/>
	</fileset-->
	<fileset dir="external/lib">
	  <include name="**/*.jar"/>
	</fileset>
      </classpath>
    </javac>
  </target>

Comment 10 Darin Wright 2005-07-29 11:10:23 EDT 
Sorry, it's hard to follow these descriptions without a concrete example... 
However, it sounds like this is a source lookup configuration issue. By 
default, the runtime classpath and source lookup path used by a Java 
application launch config is based on the build path of the project being 
run/debugged. However, if you are using Ant to build, I'm not sure that you 
have a biuld path properly configured for your project.

Ensure that each project you are debugging appears as a top level entry under 
the 'Default' item. If not, add the project as a top level item (you have to 
select a top level item to add, or have no selection - as the subelements of 
the default source lookup path cannot be modified).
