Description Maha Gamal 2006-03-21 12:36:38 EST 
IES: M5a
OS: Windows 2000, Windows XP, Windows 2003

Scenario:
1. Create a new Report Project with an Arabic name.
2. From the project's context menu click "New->Report". Give the new report an Arabic name.
3. In the "Datae Explorer" view, right-click "Report Parameters" and select
"New Parameter".
4. Give the new parameter an Arabic name.
5. Run the report (from the report's context menu click "Report -> Run Report".
6. Type a value for the parameter and click "Ok".

Expected results:
The report should run successfully on BIRT web viewer.

Actual results:
The report is not displayed and Internet Explorer shows the following error message: 
"Action canceled 
Internet Explorer was unable to link to the Web page you requested. The page might be temporarily unavailable."

Remarks:
1. The IE version is IE 6.0 with SP1
2. This is not a BiDi defect (i.e. If the project name, report name and parameter name are all in English, the same problem will occure); However, it blocks a BiDi test scenario execution.

Comment 7 Wenbin He 2006-04-11 16:51:58 EDT 
This issue may be caused by the encoding of non-ascii parameter.
Comment 8 Xuelan Li 2006-04-13 06:51:17 EDT 
The reason that caused this bug is the current strategy to encode all input string by UTF-8 no matter whether the input string has already been ecoded by UTF-8. So, the double time ecoded caused many garbage characters. 

The correct strategy should just encode the url or string that created by Viewer itself. And read the encoded value from the request header that user given to parse the input string or url, but not just encode anything by UTF-8. User should take the responsibility to make sure the input url is encoded. Viewer will not do the second time encoding for any input string. 

But this change will effect a lot, we are not sure whether this can be done by RC0. This is a long time bug.
