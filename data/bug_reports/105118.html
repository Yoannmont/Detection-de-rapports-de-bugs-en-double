Comment 2 Richard Kulp 2005-07-26 10:22:42 EDT 
There is a method:

public void setCity(City city) {
   this.city = city;

The this.city = city line caused a recurssion because there was a reference to
"city" in the "initalize" method, BUT city did not get added into the model. So
during save of cache file we were getting a dangling ref. This tries to create a
message that used toString(), well toString() failed because toString() uses the
parse tree, and the parse tree allocation had a reference to itself through city.

*** This bug has been marked as a duplicate of 104803 ***
