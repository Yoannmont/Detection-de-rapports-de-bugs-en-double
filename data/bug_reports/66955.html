Description Geoff Longman 2004-06-14 10:55:07 EDT 
I have a site build setup (1 feature, many plug-ins) that builds fine in RC1. I 
checked all the stuff in RC1 into cvs and checked it back out in RC2.

Opened the site manifest editor and added the feature in the "Features to 
Build" section of the "Features" tab of the site manifest editor.

Click "build" and the build fails with errors. The errors are all classpath 
related. If I generate the ant build file and run the 
target, "build.update.jar" on each of the plugins in the feature, correct jars 
are built with no errors.

Site still builds ok in RC1. 

The only obvious difference I see is that in RC1 files called "dev.properties" 
would be generated. This doesn't happen in RC2. Since the dev.properties are 
generated, I didn't check them into cvs. Actually I added dev.properties 
to .cvsignore.

When I remove dev.properties from .cvsignore in RC1 and check them in, then 
check them out in RC2, the site build still fails with the same errors.

Two of 4 plugins containing java classes fail. 

The core plugin depends on the old xerces 4.0.13 plugin (which I have installed 
and imported - it does appear in the "Plug-In Dependencies" library). The build 
of the core plugins fails as the classes from xerces are not picked up. (As I 
said, if I build this plugin standalone no errors occur).

The second plugin, the ui plugin, does not build because the core plugin did 
not.

The others are ok.

No, in fact I've been fiddling around with things and I did not have to import 
the xerces plugin. I think the thing that's causing the confusion is that I'm 
trying to build a feature with 5 plugins that are in the workspace and one 
(xerces) that is not.

Here's what I did:

I copied the folder org.apache.xerces_4.0.13 from 'plugins' in M7(? - i forget) 
into the 'plugins' directory in RC1.

Start Eclipse RC1, import the projects from CVS, update the plugin classpath 
for all my plugins and thus the jars, xmlParserAPIs.jar and xercesImpl.jar 
appear in the "Plug-in Dependencies" library. At this point the plugins all 
build fine with the JDT compiler.

Since xerces is no longer delivered with Eclipse, I need to make the plug-in 
available from my update site. My feature.xml looks has this 'plugin' tag along 
side the other ones:

 <plugin
         id="org.apache.xerces"
         download-size="0"
         install-size="0"
         version="4.0.13"/>

In the site I add the feature and try to build, complaints that PDE can't find 
xerces in order to build it. So, I took the folder org.apache.xerces_4.0.13 and 
jared in manually into org.apache.xerces_4.0.13.jar.

This I placed in the /plugins folder in the site project.

Now, in RC1, when I build the site, everything works in that I get no PDE build 
error zip file and the site indeed works ok in a test install of Eclipse. So, I 
commited the whole "kit and kaboodle" into CVS in this configuration with one 
exception - none of the files in the /features or /plugins folders of the site 
project were checked in. (Since the files change on a build anyways). I didn't 
check in the manually constructed xerces plugins - but I'll get to that in a 
second.

Close RC1. Before opening RC2 I copy 
RC1/eclipse/plugins/org.apache.xerces_4.0.13 over to RC2/eclipse/plugins

Start RC2 and import all from CVS, all builds fine with the JDT. Try to build 
the update site and, oops, the org.apache.xerces_4.0.13.jar isn't there to be 
found because I didn't check it into CVS. Copy that jar from RC1 into 
the 'plugins' folder in the site project in RC2 to make the layout and 
configuration identical to RC1.

But, the site does not build in RC2. The problems are as I described in the 
first entry of this bug.

I also tried this with a fresh install of RC1, repeating all the steps I did to 
move over to RC2 (copy xerces to eclipse/plugins, checkout from cvs, copy 
org.apache.xerces_4.0.13.jar into the /plugins folder of the site project) and 
it built without error.
