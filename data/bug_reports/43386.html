Comment 1 John Arthorne 2003-09-22 12:23:57 EDT 
If there are any more interesting details, it would be useful:

- If you had crashed in the previous session
- Was it reproducible
- Was the workspace in an inconsistent state afterwards, i.e., did you have to
shutdown and restart?

- It looks like there was some initial error, and then while the ErrorDialog was
up, another asyncExec was processed that caused a second error.  I'm wondering
if the real reason for the error is being lost.  Rafael, what was the original
ErrorDialog saying?

My guess is that it's a threading problem in the tree:

- Operation finishes in ModalContext
- resource change listeners fire asyncExecs
- meanwhile the ModalContext is manipluating the tree (for example it could be
in the middle of ElementTree.immutable())
- Now the UI thread tries to traverse the tree while in an unsafe state.
