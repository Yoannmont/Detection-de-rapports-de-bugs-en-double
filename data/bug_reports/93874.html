Description Darrell Kundel 2005-05-06 01:42:27 EDT 
During development I really appreciate eclipse's ability to give me the javadoc
if I hover over a type and if I right-click it then I can look at the source
code of it's implementation.

Using the 'auto-magic' "Web App Libraries" feature of the webtools in M4 I am
unable to attach either the source or the javadoc to these libraries.  This
disables the native features of the eclipse development experience.

I have tried to "attach source" when right-clicking "open declaration" on a type
as well as manually adding the source path and javadoc path within the project
settings library configuration.  Immediately after settings these - they are
lost and not persisted.

Furthermore, a workaround also fails.  My manually creating the library that has
valid javadoc and source paths - the java editor refuses to resolve to these
libraries - but instead resolves to the libraries in the "Web App Libraries"
library.  I suppose I could take out all of the jar files in the
webapp/WEB-INF/lib directory - but that would defeat being able to run the server.

In summary, this I feel is a major usability bug.  If not fixed, static
(manually set) libraries would be preferable.
