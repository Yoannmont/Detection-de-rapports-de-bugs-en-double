Description Kendric Wang 2007-12-18 12:39:19 EST 
Build ID: I20071101-2000

Steps To Reproduce:
1. Import an existing archived Java Project to the workspace.
- can use sample archive, SimpleStringTest.zip, found in test-results\tests\org.eclipse.tptp.test.tools.api.tests\resources

2. Export the API Recording Probe
File -> Export -> Test -> API Recording Probe
*make sure to export the probe into <java project>/src

3. Instrument the a.b.c.SimpleStringTest Java app statically with API Recording Probe.
Right-click on exported probe -> Instrument -> Static instrumentation -> Select SimpleStringTest.java

4. Add a Thread.sleep(1000) at the start of the main(String[] args) method in SimpleStringTest.java
*also add import java.util.concurrent.ThreadFactory;

5. Create a Profile launch configuration to launch the instrumented a.b.c.SimpleStringTest Java app, with only the 'Agent Discoverer' data collector selected.

6. Click Profile.

7. In the Profiling Monitor, you should see 'Agent Discoverer' and 'Profiling', which will collect data for a few seconds. 

8. After profiling terminates, open 'Execution Time analysis' and you will see that there is no data available to display.
Comment 1 Paul Slauenwhite 2007-12-18 15:18:17 EST 
This defect is blocking the API Recorder, so increasing the severity to critical.

In effect, the problem is that the Agent Discoverer will discover a Logging Agent (or in this case, the API Recording Agent that is a specialized type of Logging Agent) but the data (in this case, trace data emitted from the Logging Agent) is not captured when monitoring the Logging Agent.
