Comment 15 Silenio Quarti 2004-11-08 12:08:25 EST 
Yes, I believe this is a bug in the JNI functions MonitorEnter() and 
MonitorExit() for this VM. We are using these functions to temporarily leave 
the OS monitor when we go to sleep in a stacked event loop so that other 
threads can call OS functions.

To work around this bug, we are going to implement our own locking mechanism 
instead of just synchronize every method in OS. So that we can leave the lock 
properly while in Display.sleep().
Comment 16 Douglas Pollock 2004-11-08 12:12:28 EST

Comment 17 Silenio Quarti 2004-11-08 12:35:45 EST 
We are using this mechanism for a month (maybe two). We used to implement 
Display.sleep() by calling OS_LOCK.wait(50), which is the only way to leave a 
synchronized block properly in java.  The problem with wait() is that the UI 
thread would not wake up right away when an event or a sync runnable was 
available.

So the code that uses MonitorEnter() and MonitorExit() is in place since we 
implement Display.sleep() using the GUI sleep way and not Object.wait().
