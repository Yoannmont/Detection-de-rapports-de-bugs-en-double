Description Jeroen Lapre' 2005-04-25 22:01:24 EDT 
I have been developing a Java3D application with eclipse and for the most part
it works great!
Except, when I try to attach the Java3D source to one of the J3D library jar
files, e.g. j3dcore.jar, java.library.path breaks.
This is the error I get:

Exception: in thread "main" java.lang.UnsatisfiedLinkError: no J3D in
java.library.path

When I tried removing the source from the jar files via the project Properties
Java Build Path Libraries editor, I still get the error.

It appears that attaching the source introduces a bug that cannot be reversed.

Only when I remove the .metadata directory from the workspace, then rebuild the
project, does the bug go away.

Please help! If this bug can be fixed then Java3D development via Eclipse would
be a beautiful thing!
thanks
Comment 1 Jeroen Lapre' 2005-04-25 22:17:52 EDT 
Just to clarify; When I say "attach Java3D source to the j3dcore.jar file",
I ment the Java3D API source code from:
https://j3d-core.dev.java.net/
So that I can debug my application, and learn how the j3d api works.
thanks
Comment 2 Jeroen Lapre' 2005-04-29 18:06:14 EDT 
Have further narrowed this down to:
org.eclipse.jdt.core.prefs file
In the directory:
.metadata/.plugins/org.eclipse.core.runtime/.settings directory
in the workspace directory.
Comment 3 Jeroen Lapre' 2005-05-02 15:39:54 EDT 
I have still further narrowed this down to a relationship between the
org.eclipse.jdt.core.prefs file and the org.eclipse.jdt.launching.prefs file.
When setting the source code for the j3dcore.jar file, the
org.eclipse.jdt.launching.prefs file seems to be overwriting the
org.eclipse.jdt.core.prefs file.

I tried adding -Djava.path.library=/System/Library/Java/Extensions as a VM
runtime argument, but it seems to be ignored. Even though the java3d jnilib
files are in the above path, I still get the can't find J3D in java.library.path
error.

Can someone please respond to this bug report, as I posted the orginal a week ago?

Thanks
