Description Joe Holloway 2005-08-16 10:31:23 EDT 
Sporadically, the "Confirm Delete" dialog box will remain open when deleting a
resource from Package Explorer.  It seems to happen about 50% of the time and I
haven't pinned down exact steps to recreate consistently.  If I click "Yes", the
button becomes dimmed out, but the dialog box is still there.  After that, I can
still click "No" and it becomes dimmed out, but the dialog box is still there. 
The only recourse is to kill the process and start over.

The resources that I'm deleting are associated with a CVS repository, not sure
if that makes a difference.

When this happens, Package Explorer is in Flat mode and there are no Working
Sets selected.
Comment 1 Dirk Baeumer 2005-08-16 10:40:56 EDT 
Joe, can you create a VM dump when this happens. That will help us tracking down
the problem. To create a VM dump do the following:

- start Eclipse using the following parameters:
  eclipse -vm 'Path to your JDK'\bin\java.exe 

- in the command prompt window that opens set the Screen Buffer size large 
  enough to capture the trace. For example width: 120 height: 2048

- press Ctrl+Break in the command window when Eclipse hangs. This prints the
  VM dump to the console
Comment 2 Joe Holloway 2005-08-16 13:26:49 EDT 
In trying to capture the VM Dump, I think I may have actually figured out the 
steps to recreate.  As I mentioned in the initial bug report, the resources 
I'm deleting are associated with a CVS repository.  The issue seems to happen 
only when I delete resources that have been modified and saved locally (as 
denoted by the prepended ">").  If I delete clean files, it works as expected.
