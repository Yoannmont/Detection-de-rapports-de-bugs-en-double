Description Nick Boldt 2005-06-23 10:45:40 EDT 
In org.eclipse.emf_2.1.0.I200506020944.jar, feature.xml contains: 
 
   <plugin 
         id="org.eclipse.emf" 
         download-size="0" 
         install-size="0" 
         version="2.1.0.200506020944"/> 
 
Now, after jarring plugins, org.eclipse.emf_2.1.0.I200506160200.jar's  
feature.xml contains: 
 
   <plugin 
         id="org.eclipse.emf" 
         download-size="0" 
         install-size="0" 
         version="2.1.0" 
         unpack="false"/> 
 
And therefore the features are looking for plugins that don't exist in the 
filesystem.

Comment 3 Nick Boldt 2005-06-30 14:34:32 EDT 
I should mention that the problem here isn't so much that the features aren't
updated properly (though that's the first obvious manifestation of the problem). 

The real issue is that the code we have for creating UM jars assumes that
Manifest.mf and plugin.xml/fragment.xml exist in folder paths, not jar paths...
so we either need to unpack the jars from org.eclipse.foo.jar to
org.eclipse.foo/ then run the existing code, or revise the code to be able to
find and update files within jars rather than folders.

The first attempt at revising the code to read within and repair within jars
proved to be rather onerous, so I've instead opted for the unpack approach.
Doing so has caused the aforementioned semi-success with 45 of 48 plugins being
built correctly and all the features as well. I still need to figure out the
issues with the remaining three plugins.

The revised code can be see in our internal CVS (until it's complete, when it
will be propagated to dev.eclipse.org), here:

http://emf.torolab.ibm.com/viewcvs/indextools.cgi/*checkout*/emf-home/update-site/ant-build/org/eclipse/releng/generators/
Comment 4 Nick Boldt 2005-07-06 18:17:39 EDT 
Reverted code to previous level, but removed dependency on emf.map for naming.
To solve the jarred plugin problem, the shell script that calls the ant task
that runs the update builder now unpacks the jars into folders before running
ant, so that as far as the builder is concerned, we never jarred the plugins.
Perhaps not the most elegant solution, but it works.

Updated files:

emf-home/emf-build/scripts/buildUpdate.sh (which calls ant to run...)
emf-home/update-site/ant-build/buildEMFUpdateJars.xml (which calls java to run...)
emf-home/update-site/ant-build/UpdateBuilderEMF.jar (which contains...)
emf-home/update-site/ant-build/org/eclipse/releng/generators/UpdateBuilderEMF.java

Tested to properly install using EMF 2.1.0RC1 S200506300200.
