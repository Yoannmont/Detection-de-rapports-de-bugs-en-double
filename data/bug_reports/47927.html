There are 3 possible cases for the NPE at this location:
- IExtensionPoint.getExtensions() returned a null element, or
- IExtensionPoint.getDeclaringPluginDescriptor() returned null, or
- IPluginDescriptor.getUniqueIdentifier() returned null.

Moving to Core since this is most likely due to the OSGi runtime changes.

Comment 10 Pascal Rapicault 2003-12-16 11:28:41 EST 
We tracked down the problem and it comes from the fact that the registry is 
not updated when a plugin is being removed from the disk.

There is two folds to the fix. One consists in modifying the state so that 
when the fwk is restarting, the plugin that has been deleted is removed from 
the state causing the timestamp to be updated.

Second, the update.configurator could detect that some plugins changed (see 
bug #48839 which is a friend of this one) / removed and inform the fwk.
Comment 11 Pascal Rapicault 2003-12-16 11:59:25 EST 
I checked in HEAD (not for the IBuild) a fix that reset the timestamp when a 
bundle is not found. 
Now the code that recreates the registry needs to be updated to take into 
account the timestamp (see PlatformActivator.startRegistry()).
