Comment 16 David Cummings 2006-11-15 13:25:54 EST 
This is definitely a GTK2 related issue.  GMF does not work correctly on GTK2 2.10.6.

I have tested this with SLES9 upgraded to GTK2 2.10.6 with the same results you are experiencing.

If your application uses a preference page, you can turn off anti-aliasing by going to Preferences->Your Diagram Name.  There is an option to disable anti-aliasing.  If you aren't using a preference page, please use the logic diagram example, turn off anti-aliasing by using its preference page and test whether you are still having the issues described in the bugzilla.

Comment 19 David Cummings 2006-11-15 15:07:53 EST 
All generated diagrams use the default preferences, which include turning on anti-aliasing.  Thus, unless you specifically disable anti-aliasing within your diagram code, you will experience these issues in your generated diagrams.

You can disable anti aliasing in your GMF editor by updating the XXXPreferenceInitializer class and overriding the initializeDefaultPreferences method.

public void initializeDefaultPreferences() {
   super.initializeDefaultPreferences();
   getPreferenceStore().setValue(IPreferenceConstants.PREF_ENABLE_ANTIALIAS, false);
}
