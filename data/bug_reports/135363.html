Comment 29 Lina Kemmel 2006-05-15 03:27:59 EDT 
UTF-8 is the default Linux encoding and should not be changed, to not introduce numerious problems within eclipse and other software running on the desktop / shell.

Encoding from UTF-8 to UTF-8 should do nothing, i.e. return the original string.
However, URLEncoder.encode( report, "utf-8" ) assumes that the "report" argument is a UTF-16 (not UTF-8) encoded string. Why do you pass it UTF-8 encoded? Is this source string argument also escaped?

> In the current implementation of viewer, We think all the report name or

The problem is most likely caused by improperly determined native language encoding.
If you use presume that the Chinese encoding is GBK, then a user will very expectedly see the reported problems in locales associated with the UTF-8 encoding.
So if you are using predefined hardcoded native encoding values indeed, please change this to query the System property "file.encoding" at runtime instead. On Linux, as a rule, this would be UTF-8.
