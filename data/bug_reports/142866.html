Description David Vukosa 2006-05-19 17:53:30 EDT 
When displaying line or bar chart with a linear Y axis I would expect the option to only step by integer values.  Which would mean only displaying integer values on the axis.

In charts with a range of 20 or whole number are show on the axis.  If the range is smaller like 0-8 decimal numbers are displayed.  

By changing the number formatting to not show any decimal values, duplicate numbers are shown on the axis.  Ex: instead of 0.0, 1.2, 1.8, 2.3, 3.2, 3.7 the following numbers are shown: 0, 1, 1, 2, 3, 3

If I change the step value in the Scale section of the Y-series portion of the chart design, that step value is used regardless of the range of the number.  If I set the step to 1, then 0, 1, 2, 3, 4 would be shown in the example above, which is good.  But if the range is larger 0-100 then there are 100 labels on the axis the overlap each other making the axis unreadable.

It would be nice for the linear scale to have an option to only display whole numbers.
Comment 1 David Michonneau 2006-08-25 10:34:50 EDT 
Needs an API change in the Scale to only display integer values. The engine 
needs to handle that case and recompute the number of ticks to show.
Comment 2 David Michonneau 2006-11-08 13:43:38 EST 
*** Bug 163635 has been marked as a duplicate of this bug. ***
Comment 3 David Michonneau 2006-11-22 14:03:31 EST 
Fixed in 2.1.2 and 2.2 branches.

The auto-scale now automatically detects the precision of data to ensure the 
step is never smaller than the data precision. If your data only consists of 
integer values, there won't be any decimal values on the Y axis.
Comment 4 David Michonneau 2006-12-01 04:14:45 EST 
Some additional fix might be needed if the formatting of the Y axis is stricter 
than the data precision. For instance if the data has value like 2.5, 3.2,... 
and the formatting of Y axis is an integer formatting, it will still show 
duplicate values. Probably the auto-scale also need to check the number 
formatting.

Comment 11 David Michonneau 2007-01-08 10:50:37 EST 
That doesn't solve the problem the user had. We need the autoscale to consider 
the formatting of the y axis when computing the steps.

For instance if the formatting is to show one digit precision, then the scale 
should show two digits. The multiplier can also influence the autoscale: if the 
multiplier is 10 and the precision is two digits then the autoscaling can use 3 
digits of the real values.
Comment 12 David Michonneau 2007-04-27 04:58:27 EDT

formatting precision and multiplier to adjust the automatic step.
