Description Alex Blewitt 2003-02-19 08:26:19 EST 
Mac OS X applications usually have a "Preferences" menu underneath the application name in the menu bar. Eclipse has a Preferences menu item (probably created automatically by the Mac build tool) which is greyed out, although a Preferences option does occur under Window.

Can the Eclipse->Preferences menu item just call up the same preferences as Window->Preferences? It would make the Mac OS X version more Mac-like.

Note that it should be an addition to, rather than a replacement of, the Window->Preferences menu.
Comment 1 Andre Weinand 2003-03-23 12:24:12 EST 
Yes, this feature would make sense.

However the implementation would be tricky:

SWT (where the standard Apple pref menu is known) knows nothing about Eclipse's
preference menu. So it would have to second guess about a menu item with that label.
And this must work under NLS too.

On the other hand, Eclipse knows nothing about Apple's special preference menu (and 
there is not even an API for it). So it cannot connect it up to the Window / Preferences 
action.
Comment 2 Alex Blewitt 2003-03-24 15:07:21 EST 
Note that the two menu options don't need to call each other. They just both
need to call the 'displayPreferences' action (or whatever) to launch the 
preferences window. Surely there is some way in which these commands are
externalised within the interface?

If that isn't possible, then a quick cut'n'paste to have the same code launching
the preferences window would be just as good.
Comment 3 Andre Weinand 2003-04-16 04:40:36 EDT 
Sorry, but its just not as easy as that:

There is no API in Eclipse for accessing the "Preferences..." menu item in the application 
menu, because neither the application menu nor its items are manifest as SWT objects.
So there is no way to register the hypothetical 'displayPreferences' action for the 
"Preferences..." menu item.

Your other suggestion of a "quick cut'n'paste to have the same code launching the 
preferences window" cannot work because you would have to paste the code launching 
the pref window into SWT. However, that would be a gross hack since it would make the 
SWT plugin dependent on the Platform UI plugin (and would introduce a circular 
dependency).
In addition we never do "quick cut'n'pastes" in Eclipse :-)


I think the only way to make this work is to have SWT understand what the (Eclipse) 
"Preference" menu item is. With this knowledge it could internally connect the standard 
(Mac) Preference menu item with the Eclipse one.
Comment 4 Alex Blewitt 2003-04-16 06:22:12 EDT 
Good points. I wanted to try and think of other ways than the menu item depending
on another menu item could work to achieve the same results. I didn't know if
there was some kind of generic action that the Preference menu item could fire
to bring up the menu, or whether it was built into the listener code for the menu
item itself.
