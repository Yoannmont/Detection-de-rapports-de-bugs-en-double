Comment 11 Wendy Wu 2007-03-14 22:12:06 EDT 
I have looked into this issue, and noticed that there was a previous attempt to fix the problem (BUG 113543). However, it didn't fix the problem under ubuntu linux (I don't have redhat, so I didn't test it under redhat). However, I came up with a fix that works for ubuntu linux, just by setting (in file AutoGUITestControllerDialog.java, openDialog function) :
sShell = new Shell(getParent(), SWT.MODELESS | SWT.ON_TOP);  

I don't think it's the greatest to set the gui recording dialog on top, but it's seems like that's the most evasive fix.
