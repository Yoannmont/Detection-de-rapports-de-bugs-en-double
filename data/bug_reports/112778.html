Description Martin Klaus 2005-10-17 08:40:30 EDT 
It should be possible to control all functions of the UI via scripts from the
command line. Plug-ins should export their capabilities such that they can be
accessed easily from a scripting language, e.g. Python, Tcl, Perl, or even a
simple shell script. This would make it much easier to automate tasks that can
run in an unattended manner.

Comment 3 Aaron Digulla 2005-11-07 03:06:51 EST 
Scripting Use Case: Automate installation of Eclipse.

Example: When you have a team of N developers, you'll probably need to setup
every version of Eclipse in the same or similar way: Install plugins, configure
them, etc.

A simple scripting interface would allow to manage patches, updates and the like
in a controlled way.

And it would allow to quickly get an overview which version/patches/plugins of
Eclipse are installed where.
Comment 4 Joshua Benuck 2005-11-30 14:56:56 EST 
I'd love to be able to script additional keybindings. For example, when working in eclipse I often miss the VIM keybinding that goes between panes (Ctrl-W <arrow>). I'd love to be able to script something that makes moving between the package explorer to an editor to the console view as simple as Ctrl-W right, Ctrl-W down. I've often thought about writing a plugin to give me this functionality, but I haven't learned the eclipse API well enough to do this. Scripting has the potential to make it much easier and less painful than writing a full blown plugin.

I'd also like to write a script that runs at startup that allows me to set various preferences so I no longer have to go reconfigure my eclipse installation when I install eclipse on a new box. I just drop in my preferences, start up eclipse and go. The additional keybindings could also be installed at startup.

All of this is much like what you can do in emacs or vim.
Comment 5 Steve Peterson 2006-01-12 18:25:12 EST 
Scripting use case: record a key sequence to automate repetitive typing.

Potential keystrokes:

There's a list of 10 database column names you need to incorporate into 10 calls to an API.  The 10 names are in some text document somewhere.  

The list looks like this:

salutation
fname
lname
address1
address2
city
state
postCode
country
phone

1.  You cut them from the text document.

2.  Paste them into the Java file.

3.  You position the cursor before the word 'salutation' in the list above

4.  You enter the "record macro" keyboard shortcut.

5.  An indicator appears in the bottom of the Eclipse workspace indicating youre recording

6.  You type the following keys:

shift-ctrl-end (select to end of line)
ctrl-c (cut)
home
tab
literal text: Object (with trailing space)
ctrl-v (paste)
literal text: resultSet.get(
end
literal text: );
down arrow (next line)
home

7.  You enter the "end recording macro" key

8.  You press the "play macro key" 9 more times

Options include being able to save the macro with a name, or edit the macro in an editor.
