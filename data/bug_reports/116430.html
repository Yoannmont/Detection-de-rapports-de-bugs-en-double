Comment 5 Alexander Pavlov 2005-11-16 03:35:58 EST 
Just to make sure we are talking about the same version: my build id is
M20050929-0840. In a clean workspace, I've been able to reproduce this bug only 
in the following case:

1. Create a new project "target"
2. Import | Existing projects into workspace - import any plugin (let's call 
it "plugin") containing a nested folder (let's call it "nf") from a different 
workspace
3. Close the imported "plugin" that has appeared in your new workspace
4. For the "target" project choose: New | Folder | Link to folder in the file 
system - select a nested folder "nf" in the original "plugin" 
location, "Finish", and see the message.

However each time the error occurs, a warning appears at the top of the dialog:
"Location <plugin>\<nf> may overlap another resource. This can cause unexpected 
side-effects."
