Comment 18 Jared Burns 2004-08-19 16:34:17 EDT 
The problem of finding the wrong variable reveals a bug in the IJavaStackFrame interface. 
IJavaStackFrame provides a method findVariable(String variableName) that returns a single 
IJavaVariable. But there can actually be any number of variables (ok, N+1 where N is the hierarchy 
depth) with the same name in a given frame.

To really fix this problem, I think we need to add the method IJavaStackFrame#findVariables(String 
variableName) which returns an array of IJavaVariables. The JavaDebugHover can then look through 
these variables and try to find the correct one.

Test cases we need to handle:

public class A {
  protected String var="A";
}
public class B extends A {
  protected String var="B";
  public static void main(String[] args) {
    new B().foo();
  }
  public void foo() {
    String var= "foo";
    System.out.println("Breakpoint here");
    var= "foo";
    this.var= "B";
    super.var= "A";
  }
}

1. User hovers "var". Look for a local variable first, then look for a variable in this type, then look at 
variables in the hierarchy.
2. User hovers "var" with "this." prefix. Look for variable in this type.
3. User hovers "var" with "super." prefix. Look for variable in the supertype.

DW, what are your thoughts on adding this API?
