Comment 4 Billy Biggs 2005-10-01 01:16:41 EDT 
This bug is rather old and I think the relevant SWT code has likely changed,
especially since we did fix some issues with ssh forwarding a while back. Please
re-open or file a new bug if you are still having problems.

Since we work in many remote X situations without crashing other X servers, I am
tempted to assume that any crash from X-Win 32 is a bug in their server.  Also,
servers that don't support RENDER will likely not be well supported by GTK+.

The specific X request which was crashing here with a BadDrawable was an
XPutImage() request.  One possible problem with ssh forwarding is using their
secure mode, which will deny certain requests based on policy.  Try "ssh -Y"
instead of "ssh -X" if you are having trouble.
