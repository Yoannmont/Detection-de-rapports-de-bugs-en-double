Comment 3 Marcelo Paternostro 2005-05-30 16:13:28 EDT 
When you change the ecore model that is referred by a genmodel (in other words,
the ecore located in the same directory of the genmodel), you don't need to
reload the genmodel (the changes are reconciled automatically when you open the
GenModel editor) **and** you are supposed to do some extra changes, such as
setting the prefix attribute of the genPackage created for your new EPackage.

This is different than, for example, creating a genmodel for a given ecore and
saving the genmodel file in a different directory.  Actually, if you look
closely, you will notice that a copy of the ecore file is created in the
genmodel's directory.

That said, the problem you are seeing is happening because the new GenPackage
doesn't have a 'Prefix' (you can set it through the GenModel editor).  To make
the tree look a little nicer, I've changed the genPackage content provider to
present the name of the EPackage between '[' and ']' when genPackage.getPrefix()
returns null or an empty string.

Comment 5 Marcelo Paternostro 2005-05-30 17:14:41 EDT 
We don't have anything that would do that for you.

That said, maybe your tool could reconcile the ecore and gen models (look at 
the GenModelEditor.initialize(GenModel) method) and then adjust the 
genPackages setting their prefixes (which are usually the 'non qualified' name 
of the ePackage, uppercasing the first letter) and basePackages.
