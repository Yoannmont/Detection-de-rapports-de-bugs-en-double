Within the 2.0 development stream, we did not guarantee that existing launch 
configurations would remain valid from one build to the next.

If you create the same setup in a new workspace and create a new launch 
configuration, does it work?
Comment 2 Philippe Krief 2002-07-02 09:04:45 EDT 
Hi,
I didn't use any existing launch!
I went farther in my investigations.
Actually, if you have a New project (built the latest drop) and you select your 
main and create a launch and you don't touch any default setup, including the 
check box "Use default class path" [which is checked] in the "Classpath" tab): 
It works fine!
Now, if you change the classpath in your New Project and try to run your 
previous launch, you get the same error: "Exception Starting process"!

Now, if you delete this launch and recreate a new one, this one will work 
fine...
I hope it will help.
