Comment 12 Nick Edgar 2003-03-13 10:30:12 EST 
The default setting is for color icons to be off (enabled variant shown on 
toolbars).  People who use this preference already have extra noise since the 
icons were designed for highlights, not to be all in colour at the same time.
So I think there's a fair bit of room to play with here.  It's just that we 
would probably get screams if the highlight was -really- strong.

At this late stage I think we should keep the enabled look the same, and just 
give a stronger highlight in the colour variants.  We can revisit this for 2.2 
if needed.

The pin/fast view button is really confused.  I was trying to make it 
consistent with the Fast View toggle item in the view's system menu (right 
click on the view title bar).  But the end result of only showing this button 
when it's a fast view is that you get a pushed in button that says "Fast View" 
and when you click it, the view becomes a regular view.  The pin metaphor is 
confusing since it's unclear whether a pin should pin it to the left as a fast 
view, or pin it as regular view.  I don't think we should change this now, but 
should revisit this in 2.2.

Comment 21 Nick Edgar 2003-03-17 11:34:22 EST 
Linda,
We tried out the Option C cuts and we feel that this is too strong a departure 
from the look of the colour variants for the other toolbar icons.
It feels "wrong" to give the view controls so much emphasis.
I now think that we should not draw as much attention to the view controls as 
we do to the other tool items.  The tool items are more unique than the view 
controls, which are exactly the same on each view.  The emphasis should be on 
the tool items, and the items within the view drop-down (which are always in 
colour), not the view controls.

Could you please attach cuts for Option B and the new enabled variants (they 
look darker than our current ones).  We'll try them out, and if they play well 
we will go with them.

If not, then for 2.1 we will stick with always showing the old colour variants 
(solid black) and we will live with the limitation that the close box is not 
sufficiently highlighted on the Mac.  There are other problems with the Mac 
look anyway, for example Mac users expect such controls to be on the left 
rather than the right.

Thanks,
Nick

Comment 25 Nick Edgar 2003-03-18 16:41:52 EST 
We tried out Option 2 and found that the enabled variants looked a little too 
gray (showing Option 2 as the colour variants required changing the code to 
show the old enabled variants, which we weren't doing previously).  
The enabled variants appeared a little too close to disabled, particularly for 
min_view.gif.  Changing the enabled variants would also affect screen shots.

Also tried making the old (solid black) colour variants the new enabled 
variants, along with Option 2 variants for the colour variants.  This looked 
pretty good, and made the close box on the Mac look quite a bit better.
Note that on the Mac, it always uses the colour variant, and does its own 
darkening treatment when the button is armed.  The change was noticeable but 
still subtle.

Unfortunately, on Motif, the pin_view icon shifted vertically noticeably when 
the button was armed.  This was due to there being a 1-pixel vertical shift in 
the icon between the enabled and colour variants.

This just emphasized the fact that it's risky to be making any such changes at 
this point in time, so I'm chickening out on this for 2.1 (mainly due to having 
to make code changes, not the shift in the icon file which can be easily fixed).
We can revisit this in 2.2.

Thanks Linda for giving us the chance to consider the various options though.
