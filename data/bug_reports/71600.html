Description Kevin Klinemeier 2004-08-06 16:41:21 EDT 
1 Tear off the package explorer and give it the focus 
2 Refresh a project via the package explorer, causing the progress box to appear.
3 Wait for the progress box to close on its own.

At this point click on the title, menu, or shortcut bars has no effect, and does
not change the focus.  I have to click on one of the main window views first to
gain focus, then on the menu or shortcut item I'm interested in.
Comment 1 Daniel Schneller 2005-04-06 08:42:33 EDT 
Also nice if you have *all* views detached to get the maximum free screen space
for the java editor (all the other views on a second monitor).
In this case you hit Ctrl-Shift-F4 to close everything. You have no chance of
any action in the main window unless you drag one of the remaining detached
views back so that it gets focus.
Comment 2 Eric Moffatt 2005-08-04 16:21:19 EDT 
You can repro this by bring up any modal dialog when a Detached Window's shell 
is active...I used the 'Delete' in the Package Explorer, cancelled it and still 
get the problem.

This looks like a confusion where the system is left in a state where 
it's 'partly' modal (i.e. no trim handling). The most likely cause is that 
somebody thinks that the dialog is being parented by another modal dialog 
(because the Detached Window's shell has no parent??) and acts accordingly when 
the dialog is dismissed and the system resets the focus.
Comment 3 Eric Moffatt 2005-08-05 13:41:26 EDT 
I lied...it's not ANY modal dialog. Bringing up the "Proerties" dialog and 
cancelling doesn't cause the problem. I'm starting to suspect progress-related 
interactions now since both the Refresh and any refactoring related commands 
use progress monitors...
Comment 4 Eric Moffatt 2005-08-05 15:19:12 EDT 
This has something to do with the ProgressManager's 'setUserInterfaceActive' 
method. If I wire off the disabling (just returning if 'active' == false) then 
I don't see the issue...but who knows what I've broken...;-0.
Comment 5 Eric Moffatt 2005-08-08 11:05:12 EDT

re-induce the defect...
