Comment 4 Chris Recoskie 2006-12-04 17:19:51 EST 
That version of Eclipse is compiled with GCJ, which is not a supported configuration.  Not that I want to discourage you from filing bugs but any GCJ compiled versions of Eclipse have been notoriously buggy, and I seem to recall that we've seen similiar bugs filed previously that turned out to be just due to the flakiness of GCJ.

Try downloading the official builds of Eclipse + CDT from Eclipse.org and see if that solves your problem.

Comment 7 Harmon Nine 2007-02-05 16:36:09 EST 
I'm using eclipse on Debian Linux.  It was installed via a .deb, and I was having the same problem with search not working.

The "eclipse" executable in Debian is a bash script front-end to the actual executable that, amongst other things, sets a value for the JAVA_HOME environment variable when it is otherwise unspecified.  When executing "eclipse", I could see that the script was setting JAVA_HOME to the gcj installation, i.e. the following message was displayed:

searching for compatible vm...
  testing /usr/lib/jvm/java-gcj...found

Once I read the comments on this bug, I added a JAVA_HOME variable definition to my .eclipse/eclipserc file, assigning it the path of a local sun-java installation, i.e. "/usr/local/java/jdk1.5.0_09".

Search is now working.  Looks like gcj is indeed "flaky."
