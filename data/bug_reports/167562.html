Comment 23 Darin Wright 2007-04-23 23:17:38 EDT 
In both traces, the crash occurrs after we attempt to filter a static or static final variable (not sure which filters the user has on here)... The child count has been updated with the total number of children, and when we retrieve the first variable, we find out it has been filtered, so we remove that item from the tree (before it even gets mapped to a model element).
Comment 24 Darin Wright 2007-04-23 23:20:40 EDT 
(So... perhaps try stepping into a static method that has statics and static finals in the class, and make sure "Show Statics" and "Show Constants" are turned off in the variables view - i.e. filtered).
Comment 25 Darin Wright 2007-04-23 23:27:28 EDT 
other observations:

* in the first case there were 13 children for the root of the tree, and only 8 were visible.
* in the second case, there were 8 children, and 8 visible.

In both cases, the first element ends up being filtered (removed)
