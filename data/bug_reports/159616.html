Comment 17 Helmut J. Haigermoser 2006-10-09 05:08:15 EDT 
I've was finally able to track this problem down: It's mozilla.
My system, Red Hat Workstation 3, Update 4 has mozilla version 1.4.3
(a supported version http://www.eclipse.org/swt/faq.php#browserlinux),
this version seems to be the problem, I've updated to 1.7.10 to verify
this and voila, the crash is gone.

So, the real question is: Is Mozilla <  1.7.10 no longer supported by Eclipse 3.2.1 on linux?
Comment 18 Grant Gayed 2006-11-08 11:59:33 EST 
Mozilla >= 1.4 is supported in 3.2.1.  However I've never seen or tested with mozilla 1.4.3 (it's not available for download from mozilla.org).

The swt profile14 library does some static linking from mozilla 1.4, so it's likely that a change was made between 1.4 and 1.4.3 that was incompatible with the statically-linked content.  These profile libraries will likely be going away, so this should not be a problem once that happens.  Marking report as a duplicate of the bug report that discusses doing this.
