Description Jim des Rivieres 2002-05-08 20:56:11 EDT 
Integration build 20020508 (1411)

0. Fresh install; no workspace
1. Start eclipse.exe -debug
2. Exit eclipse

3. Insert a new feature version into <install>/eclipse/features/ directory.
Insert a new plug-in version into <install>/eclipse/plugins/ directory.
I created a new version of feature org.eclipse.sdk version 1.9.0.001
I created a new version of plug-in org.eclipse.sdk version 1.9.0.001
See attached zip.

5. Start eclipse.exe -debug

Observe: The workbench opens normally.

Expectation: This is a case where startup should have discovered that
there were new feature and plug-in versions in the product install.
The workbench should be called with the -hasUpdates command line option,
and the workbench should say to the "You have updates." and ask them
whether they want to visit the Update Manager.

6. open the update manager

Observe: The Configuration View shows that both the current configuration 
contains the feature "Eclipse SDK 1.9.0.001".

Expectation: The new feature version should not be configured automatically.
However, it should be very convenient for the user to do just that if
they choose.

Observe: The Configuration View shows that the configuration in the history 
contains the feature "Eclipse SDK 1.9.0.001", and there are no preserved
configurations.

Expectation: The original configuration with feature "Eclipse SDK 1.9.0"
should be somewhere, shouldn't it?


This whole usecase is one that needs to work smoothly.
