Description Robert Elves 2007-02-12 21:08:05 EST 
The team editor annotation with history reveals changes so we should be able to right-click and open the task that corresponds with the change.
Comment 1 Mik Kersten 2007-02-20 20:34:18 EST

Comment 2 Dani Megert 2007-04-11 06:47:54 EDT

Comment 3 Mik Kersten 2007-04-11 14:52:22 EDT 
 (In reply to comment #2)
> What exactly is the team editor? Do you mean the Team > Show Annotation action?

Yes.  I clarified the summary to be about the use case.  I can see three ways of implementing this, (1) is probably best?

1) Make the annotation popup accept hyperlink detectors.  Makes sense since the History view does it (in I-builds post M6) and shows the same information.  Challenge here is that users would have to know to hit F2 for the popup to get focus.  If the popup was flush with the annotation and didn't disappear this might not require focus.

2) Add a ruler action for Open Corresponding Task that would somehow do the right thing.  It would be confusing unless it disabled itself if annotations were not visible.

3) Support a menu/contributions on the actual annotation, since it is possible to select it.
Comment 4 Dani Megert 2007-04-12 02:43:04 EDT

2) agree

3) this should already be doable by getting the ruler via editor.getAdapter(IRevisionRulerColumn.class) and then adding a IVerticalRulerListener

Comment 23 Eugene Kuleshov 2007-04-17 11:17:17 EDT 
Created attachment 64047 [details]
proposed patch for team.cvs.ui plugin

Here is patch for team.cvs.ui plugin. It adds custom annotation hover for cvs that uses registered hyperlink detectors (including one from Mylar). There are few lines changed in ShowAnnotationOperation class and added copy of the SourceViewerInformationControl with enabled hyperlink detectors and minor additions to the text decoration.

Note that all the new code has very little to do with cvs. So, I am still convinced that would be more elegant and much more beneficial for the entire Platform if annotated editor would use control with enabled hyperlink detectors instead of browser (which is only needed for one proprietary plugin, and that plugin could use browser if it wanted to). Without that every team provider have to duplicate code from this patch in order to support these hyperlinks.
