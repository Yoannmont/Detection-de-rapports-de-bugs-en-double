Description Cameron Bateman 2006-03-28 14:19:27 EST 
Copy the html below into a JSP editor.  Note that the editor should immediately flag a syntax error related to the </body> tagging missing a ">".  

However, saving, building or validating the file or project will never cause the error to translate into markers on the navigator tree or in the Problems view.

Behaviour seen in I20060323.


<%@ page language="java" contentType="text/html; charset=ISO-8859-1"
    pageEncoding="ISO-8859-1"%>
<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01 Transitional//EN">
<html>
<head>
<meta http-equiv="Content-Type" content="text/html; charset=ISO-8859-1">
<title>Insert title here</title>
</head>
<body>
</body
</html>

Comment 3 David Williams 2006-05-12 15:41:00 EDT 
It seems to me, the immediate problem is just the way HTML Validator is defined in the plugin.xml. Perhaps we haven't "kept up" with other changes in validator default assumptions? (Either that, or its been broken for a long long long time!). 

Changing the current 
           <run
                enabled="true"
                class="org.eclipse.wst.html.internal.validation.HTMLValidator">
           </run>
to
        <run
             async="true"
             fullBuild="true"
             incremental="true"
             enabled="true"
             class="org.eclipse.wst.html.internal.validation.HTMLValidator">
        </run>

seems to fix the problem.

Comment 18 Amy Wu 2007-02-14 01:03:46 EST 
Created attachment 58949 [details]
org.eclipse.wst.xml.core.patch

As Nitin stated in bug 143213, the validation adapters can only report errors to 1 reporter at a time, and they were only being reported to the source validation reporter when batch validation was being ran.

The current validation code actually tries to set the validation reporter to the appropriate one before calling validate.  The problem was that the setting of the reporter wasn't being propogated.  The attached patch makes sure to set the reporter to propogating validators.
