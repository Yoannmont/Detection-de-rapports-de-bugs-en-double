Comment 4 markusle 2004-01-24 22:47:59 EST 
BTW -- My theory for this theme is:
In the native GTK-code of SWT  was a mutex introduced, somewhere
between M5 and M6, and on slow machines like mine (1400Mhz Athlon) this
leads (in not very rar cases) to a dead lock between Threads invoking
OS.g_object_unref(int) and "OS.new_image_gtk(int)" (perhaps others).

new_image_gtk: I can't tell you the real name of this method because
I got only 2 debug session with stalled thread and in one of this
sessions an "Animation thread" (perhaps for the search progress bar)
stalls the same way as the cited thread above -- in a method
named like "new_image_gtk" or so.

I believe you Hardware is much faster then my, so you probably have 
no problems with such race conditions.

Again: That is only a theory, but the problem is really extrem for me.
