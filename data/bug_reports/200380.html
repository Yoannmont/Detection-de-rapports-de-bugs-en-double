Comment 19 Pascal Rapicault 2008-05-07 21:59:58 EDT 
It is important to note that the actual message returned is scarier than it needs to be and it has been fixed for the next I-build. 
More importantly, this message does not represent the reality of what the resolver is capable of doing. Indeed the resolver can handle an unlimited number of versions of the same bundle (being singleton or not) and will keep on finding a solution if there exist one.

As for the particular case of ECF described in the previous comment, the problem results from the fact that the p2 feature requires a precise version of the ECF filetransfer bundle (v20080428-0800) and that the version of ECF you are trying to install (or something else you are trying to install) precisely requires another version of the ECF filetransfer bundle (v20080406-1615). Since both requirements are strict on a particular version then there is no solution to be found, since the filetransfer bundle is singleton and rightly so.
In short, some dependencies are not properly aligned and the error you are getting is normal.
(Note that I have been using the term requirement generically, not in the sense of feature.xml).

To avoid this kind of situation in the future and give more flexibility to ECF to evolve without being blocked by the version of ECF shipped by the platform we want to change our feature (see bug 225783)
