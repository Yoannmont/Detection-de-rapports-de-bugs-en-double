Comment 9 Gary Johnston 2003-10-07 14:41:50 EDT 
Here's a clue I discovered.  I (and Tom, too, I believe) use the JET support in
org.eclipse.emf.codegen to generate Java classes for various things.  Lately,
we've been having a problem with it such that JET regenerates some of the classes
at every opportunity.  And, in fact, during a "Rebuild all" I could see JET
recompilation messages flashing past in the dialog.  Getting suspicious, I
exited my workbench, disabled plugin org.eclipse.emf.codegen (renamed its

voila, no
problem.  So, perhaps the JET builder/compiler re-writing a few .java files during
the build is a trigger of this problem.

Comment 18 John Arthorne 2004-02-20 19:07:27 EST 
The JET builder is attempting a re-entrant build:

JETBuilder.build calls JETCompileTemplateOperation.run, which calls IProject.build.

We have guard code in BuildManager to prevent this, but one of the build methods
puts this guard inside the try/finally that cleans up the build manager's
temporary state, so this ends up killing the outer build that is going on. I
have fixed up the code in BuildManager, but the JETBuilder really shouldn't be
trying to build from within a builder.

Moving to EMF for further action.  The NPE should no longer happen as of 3.0 M8,
but the JETBuilder should also be fixed.
