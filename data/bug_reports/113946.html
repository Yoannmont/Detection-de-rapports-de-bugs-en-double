Description Eoin 2005-10-27 06:01:22 EDT 
[formatting] CtrlShiftF does not reformat JavaDoc above package keyword in 
package-info.java. If you have a lot of Javadoc text, some going past the 80 
column mark and try to reformat it it does not work. Moving the Javadoc below 
the package keyword allows you to reformat the Javadoc comments.

The comments need to be above the package keyword, however, if they are to 
appear in the html output of the javadoc tool.

Comment 12 Markus Keller 2012-03-12 12:12:30 EDT 
For package-info.java, the situation regarding header comments is a bit special. The last Javadoc-style comment (in /** ... */) before the

package xxx;

declaration should be considered as the package's Javadoc comment. If the file starts with a comment that is not the package's Javadoc comment, then that first comment should be considered as header comment.

AST nodes and IPackageDeclaration's source range look correct.

Example:

/************************************************
 * Copyright (c) 2011 Stephan Herrmann and others.
 * All rights reserved. This program and the accompanying materials
 * are made available under the terms of the Eclipse Public License v1.0
 * which accompanies this distribution, and is available at
 * http://www.eclipse.org/legal/epl-v10.html
 *
 * Contributors:
 *     Stephan Herrmann - initial API and implementation
 *     IBM Corporation - bug fixes
 ************************************************/

/**
 * This package contains annotations that can trigger special behavior
 * when annotated types are compiled by the Eclipse Compiler for Java.
 * <p>
 * Currently, the package contains annotations that specify nullness contracts
 * for annotated elements.
 * </p>
 */
package org.eclipse.jdt.annotation;
