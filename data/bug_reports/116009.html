Comment 1 Billy Biggs 2005-11-16 14:41:52 EST 
SWT 3.1's cairo integration only works with cairo 0.4. It looks like whomever
built the FreeBSD packages of eclipse somehow built this code against cairo 1.0,
which is not compatible.  I believe this is a bug in the FreeBSD package.

You should indicate to the FreeBSD packager that they should either apply the
patches from bug 106527, stop building the cairo library and allow it to fail
with an exception which the client code may handle, or update to an Eclipse 3.2
stream build.
Comment 2 Billy Biggs 2005-11-16 14:43:51 EST 
I'm going to close this as INVALID since I don't think there's any bug in our
code, but please let me know here what you find out and how I can help.
