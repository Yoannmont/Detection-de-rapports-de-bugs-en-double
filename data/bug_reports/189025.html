Comment 14 Michael Valenta 2007-10-18 16:50:50 EDT 
I have seen an instance of this at a colleagues desk and I believe I know why this is happening. I don't have time to test my theory but here are the set of steps I believe would reproduce the problem

1) Make some local changes to a file from workspace A
2) Commit some conflicting changes to the file from workspace B
3) Perform a Team/update in workspace A. This should result in conflict markers in the file
4) Fix the conflict markers
5) restart Eclipse
6) Perform a Synchronize

The problem is that the update in step 3 places a "result of merge" indicator in the meta data for a file. This marker is then transfer to the metadata used when trying to fetch the file contents which results in a failure to fetch the contents. The fix would be to ensure that the merge indicator is not included when trying to fetch the contents of the remote revision.
