Description Leo Dos Santos 2007-05-11 22:48:25 EDT 
The arrow to expand the sub-task renders on top of the activation widget. This makes it almost impossible to expand the sub-task without also activating the task.
Comment 1 Mik Kersten 2007-11-01 15:26:01 EDT 
*** Bug 207843 has been marked as a duplicate of this bug. ***
Comment 2 Steffen Pingel 2007-11-01 16:21:27 EDT 
Mik, I have a patch for disabling the activation column. Let me know if you want me to attach it. I think this should be considered for 2.2. I would prefer to disable that feature on Linux until we find a better solution instead of distributing the current implentation which is severely broken.
Comment 3 Mik Kersten 2007-11-01 19:24:39 EDT 
I *really* don't like the idea of having this significant a difference in the Task List UI across platforms.  I already feel bad enough about the fact that strikethroughs are windows only ;)  So let's discuss alternatives before removing entirely.
Comment 4 Steffen Pingel 2007-11-01 22:50:47 EDT 
I agree. Here is a compromise: Do not draw the task activation button (or at least disable mouse events) for nodes on the 2nd hierarchy level that have children (if technically possible). That should fix the usability issues while still maintaing most of the functionality and UI accross platforms.
Comment 5 Mik Kersten 2007-11-09 22:14:25 EST 
Steffen: Disabling mouse events is a great suggestions and quite easy.  I can walk you through how the mouse click computations are done and you could make a special rule for Linux.
Comment 6 Steffen Pingel 2007-11-10 01:49:49 EST 
Created attachment 82598 [details]
screenshot

I have committed a tweak that draws the activation and incoming image when a task is selected on Linux.

I am not convinced it is a good idea to draw the activation images on Linux. I tried to move it in between the expansion controls but there is not enough space. With the buttons enabled my task list looks very crowded and I like it much better without the activation buttons after working with it for a week. I have to admit though that I mostly activate tasks from the context menu and never use these buttons.

Comment 9 Mik Kersten 2007-11-14 23:36:26 EST 
Steffen: I think that the only real reason to keep the overlaid Activation button and expansion control is for us to make sure that we fix it before too long.  I don't like the idea of keeping it either, but these activation buttons are an important part of the UI.  

I think that the best fix for this might be to simulate what's happening on Windows for Mac and Linux, in other words, to provide the additional padding needed in order to get the spacing.  I experimented with this a bit by having platform-specific widths set up in CompositeTaskImageDescriptor.  Note that the underlying problem is bug 185004.
