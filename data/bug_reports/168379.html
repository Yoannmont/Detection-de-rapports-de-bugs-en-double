Comment 4 Boris Bokowski 2006-12-22 13:00:20 EST 
Andrée, thanks for your comments!  For now, all I did was experiment with a new scheme for managing the tabs.  Time permitting, I would like to see if moving the editor toolbar items to the editor area is doable, I like this idea too.

About what I did so far - it's not perfect, but I already like it a lot because it is much more predictable than before. Here is how it works:

- No MRU visibility for tabs - you see a "sliding window" of all your tabs.
- Opening an editor either replaces the editor in the current tab, or opens a new tab.  A new tab is opened when the current editor is pinned.
- Editors are pinned automatically when they first become dirty.  Saving does not make them un-pinned.
- You can open a new empty tab (File->New Editor Tab).

What you get is very similar to tabbed browsing in e.g. Firefox as long as you don't start editing files. Any opening of editors, or navigating between editors (e.g. F3, clicking on links) will take place in the same tab, and you only get a new tab when you create one using File->New Editor Tab. (I have bound this to Ctrl-T...)

It differs from Firefox behaviour when you start editing a file.  From now on, this file will stay in that tab until you close it, or un-pin it.  The intuition is that files you are editing (or have edited) are more important than the others where you were just browsing and should stay around.

Overall, this scheme leads to a smaller number of tabs being opened, depending on how much you browse vs. how much you edit.  It also makes it predictable where editors will open because it's either in the current tab or in a new tab. You can leave a tab open as a reminder to do something in that file, or because you want to continue browsing in there - if it is not the current tab, it will stay around.

If anyone would like to experiment with this, you need the last N-build from this year (N20061220-0010 - watch out for bug 168700 if you are on Linux) and change two preferences:
 - General->Search, uncheck "reuse editors"
 - General->Appearance, pick the "3.3 presentation (experimental)"
If you were using the 3.3 presentation already, you need to switch to the default and then back to 3.3 again.

Here are the next steps for this:
 - The navigation history (back and forward buttons in the toolbar) should be tab-specific, not global, just like in Firefox
 - Window->New Editor should clone the current tab including its navigation history and needs to be renamed accordingly
 - Navigating to a file that is already showing in another tab should not switch  to that tab, what you want is this file in the current tab.  Browsers don't switch tabs just because you navigated to a page you already have open in another tab.

I think you've misinterpreted why MRU has been around as long as it has. Essentially, there's only so many times we can keep changing around how people work.
