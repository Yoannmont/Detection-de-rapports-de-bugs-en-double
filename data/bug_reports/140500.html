Description Heitzso 2006-05-07 10:02:34 EDT 
SWT/OS X Label right justify jams the text right up on the right edge of the label versus some space when left justified.  Works fine on Windows.

Comment 4 Heitzso 2006-05-10 22:15:07 EDT 
My app. code is more complex than originally noted.  I use composites in order to force the text of the label to be vertically centered within a large label.  

I created a test program that display 4 labels: plain label, plain label in composite, clabel, clabel in composite.  A side thread cycles through center, left and right alignments.  The test program behaves differently on Linux/OSX/Win.

The program is (with a few extraneous lines):

package test;
import org.eclipse.swt.*;
import org.eclipse.swt.custom.*;
import org.eclipse.swt.events.*;
import org.eclipse.swt.graphics.*;
import org.eclipse.swt.layout.*;
import org.eclipse.swt.widgets.*;
public class LabelTest {
	public LabelTest(String[] args) {
		int height, width;
		final Display display = new Display();
		final Point screenDPI = display.getDPI();
		final Shell shell = new Shell(display, SWT.SHELL_TRIM);
		final Rectangle appRect = display.getBounds();
		shell.setText("Label Test");
		shell.setSize(500, 400);
		shell.setLocation(50, 50);
		Font font = new Font(display, "Arial", 18, SWT.NORMAL);
		Color white = display.getSystemColor(SWT.COLOR_WHITE);
		shell.setBackground(white);
		final Label l1 = new Label(shell, SWT.BORDER|SWT.CENTER);
		l1.setFont(font);
		l1.setText("TEXT FOR L TEST");
		l1.setLocation(50, 5);
		l1.setSize(400, 40);
		
		final Composite c2 = new Composite(shell, SWT.BORDER);
		c2.setSize(400, 40);
		c2.setLocation(50, 110);
		final Label l2 = new Label(c2, SWT.SHADOW_NONE|SWT.CENTER);
		l2.setFont(font);
		l2.setText("TEXT FOR C+L TEST");
		l2.setLocation(0, 0);
		l2.setSize(400, 40);

		final CLabel l3 = new CLabel(shell, SWT.BORDER|SWT.CENTER);
		l3.setFont(font);
		l3.setText("TEXT FOR CL TEST");
		l3.setLocation(50, 210);
		l3.setSize(400, 40);
		
		final Composite c4 = new Composite(shell, SWT.BORDER);
		c4.setSize(400, 40);
		c4.setLocation(50, 310);
		final CLabel l4 = new CLabel(c4, SWT.SHADOW_NONE|SWT.CENTER);
		l4.setFont(font);
		l4.setText("TEXT FOR C+CL TEST");
		l4.setLocation(0, 0);
		l4.setSize(400, 40);

		shell.open();
		display.asyncExec(new Runnable() {
			public void run() {
				for (int i = 0; i < 6; i++) {
					try { Thread.sleep(4000); } catch (Throwable e) {}
					if (i == 0 || i == 3) {
						System.out.println("align right");
						l1.setAlignment(SWT.RIGHT);
						l2.setAlignment(SWT.RIGHT);
						l3.setAlignment(SWT.RIGHT);
						l4.setAlignment(SWT.RIGHT);
					} else if (i == 1 || i == 4) {
						System.out.println("align left");
						l1.setAlignment(SWT.LEFT);
						l2.setAlignment(SWT.LEFT);
						l3.setAlignment(SWT.LEFT);
						l4.setAlignment(SWT.LEFT);
					} else if (i == 2 || i == 5) {
						System.out.println("align center");
						l1.setAlignment(SWT.CENTER);
						l2.setAlignment(SWT.CENTER);
						l3.setAlignment(SWT.CENTER);
						l4.setAlignment(SWT.CENTER);
					}
					l1.update();
					l2.update();
					l3.update();
					l4.update();
				}
				shell.dispose();
			}
		});
		while (! shell.isDisposed())
			if (! display.readAndDispatch())
				display.sleep();
		white.dispose();
		font.dispose();
		shell.dispose();
		display.dispose();
		System.exit(0);
	}	
	public static void main(String[] args) {
		new LabelTest(args);
	}
}

The OSX compile and run code (because of oddity of OSX JVM I'm including):

cd ..

java -XstartOnFirstThread  \
 -classpath  .:../../tools/lib/osx/swt.jar  \
 -Djava.library.path=../../tools/lib/osx/swt.ppc \
 test.LabelTest

With the jnilibs in the java.library.path directory.

On Windows code acts as one might expect with the exception that there
are variations in horizontal spacing between Label and CLabel, with CLabel having slightly more space.  

An exagerated version of this on OSX is what triggered my original bug report on this issue because the OSX implementation of Label within a composite pushes over further or strips trailing spaces (my app code brackets text with a space on either side to provide some visual space).  While the CLabel either does not strip trailing spaces and/or it provides a little more room on the right.

On Linux the bottom label gets out of sync from the above labels (same code as running on Windows) re horizontal alignment.  It appears to not be updated on the display at the same time as the others.  This may be related to other bug reports I have filed re problematic SWT on Linux behavior.  
For example: https://bugs.eclipse.org/bugs/show_bug.cgi?id=140615 
coule be explained by an out of sync display that is picking up out of date widgets.
But the code consistently gets out of sync re what is being displayed when run on Linux.

On OSX the labels are never updated.  Instead a whirly-gig-circle is displayed. The test program is structurally different from my app code, my app code updates the displayed CLabels but not from a display.asyncExec() thread.  I wasn't ready to fight that further at the moment.  You may wish to pause and consider whether there's another bug in SWT on OSX that this code is tripping on.  Anyway, that is why I'm including the script to compile/run on OSX.  OSX java is current 1.4.2.  If you want me to test w/ OSX 1.5.x I can.  In any case, if you alter the code in some way to get OSX JVM to update the display and show the altered alignment you should see behavior which differs further from Windows behavior in this program.

Linux env. ubuntu with IBM java 1.4.2
Windows env. win2k pro w/ sun jvm 1.4.2_04

Comment 6 Steve Northover 2006-05-11 10:46:44 EDT 
There is at least one SWT bug here (update() was broken on the Mac due to recent HIView changes) and a misunderstanding of the API.  I'm looking into the update() problem right now.

The coordinate system of the child does not include the border.  So when you use setSize() to set the size of the child to be the same size as the parent, you cause the area to the right to be clipped.

Try this instead:
    c2.setSize(400, 40);
    c2.setLocation(50, 110);
    final Label l2 = new Label(c2, SWT.SHADOW_NONE|SWT.CENTER);
    l2.setFont(font);
    l2.setText("TEXT FOR C+L TEST");
    l2.setBounds(c2.getClientArea());
