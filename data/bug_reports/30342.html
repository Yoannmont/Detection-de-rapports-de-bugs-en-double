Comment 6 John Arthorne 2003-01-31 17:02:15 EST 
More findings...

It appears that Jerome deleted about 60,000 files from the workspace before
zipping it up.  The workspace on disk contained 73,712 resources, but after
startup the tree only contained 12,333 resources.  These deleted files appear to
have been JCK test files.

When computing the resource delta, we use a simple lookup table to store the
paths when doing the calculation to track moves. (see NodeIDMap). This map uses
the node ID as the hash value.  The resulting hash distribution is *terrible*,
and since on collision we iterate until an empty slot is found, the performance
of table get and put approaches O(n).  This makes delta calculation approach
O(n^2) (don't forget n=73,712, n^2=5.3 billion).

We should investigate finding a better hash algorithm for hashing large numbers
of sequential longs.  The current hash algorithm is:

long id = ...;
int hash = Math.abs((int)id) % table.length;

The table load factor is 0.75, the growth function is x2, and the initial size
is 10.  In this example the table had to grow and rehash 14 times.
Comment 7 John Arthorne 2003-02-03 17:11:41 EST 
One more finding:

The methods NodeIDMap.getNewPath and getOldPath did an exhaustive search of the
table in the case where the search item was missing.  In this case, 60,000
deletions times a delta table of size roughly 160,000 = 9.6 billion probes.  Ouch.

I have applied the following fixes:

- changed hash function to use Knuth's multiplicative hash function (see Knuth
section 6.4).
- use prime numbers as table sizes to further improve hashing behaviour.
- changed lookup method to stop linear probing on encountering first empty slot.

Together these roughly divided the startup time in half for this case.  Roughly
half of the speedup is due to the change of hash function, and the other half is
due to the lookup fix.

I don't think these fixes will give us much performance increase in more general
cases.  This was a pathological case (run several days, crash, delete 60,000
resources, restart).  However, it should help a bit in the large delta cases.
