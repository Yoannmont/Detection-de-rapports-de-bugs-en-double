Description Petr Lindovsky 2007-01-15 08:01:26 EST 
Build ID: M20060921-0945

The attached sample shows a window with one SWT text area and an AWT text are. If you run it with Java 5, the focus listener on the embedded AWT component properly reports when the component gains and looses focus. In Java 6, you have to click twice to transfer focus to the AWT component.

Steps to reproduce:
- Run the sample and click the AWT text area. In Java 5, the component gets focus (you see a notification on the console). In Java 6 (the final version), the component doesn't get focus; click for the second time to make that happen.
- Click the SWT text area. Focus is lost both in Java 5 an 6.
- Click the AWT text area. Again, focus is gained in Java 5 but not Java 6.

Comment 4 Oleg Sukhodolsky 2007-02-09 04:15:50 EST

Comment 5 Oleg Sukhodolsky 2007-02-15 08:25:36 EST 
Ok, it looks like we need to change both AWT and SWT to resolve this problem.

SWT need to call sun.awt.EmbeddedFrame.synthesizeWindowActivation(boolean) instead
of posting window events on SWT.Activate and SWT.Deactivate (you need to pass true
for activation and false for deactivation)

This change should fix problem with heavyweight.

The problem with lightweights is a bug in AWT (#6516675) I hope it will be fixed 
soon :)
Comment 6 Oleg Sukhodolsky 2007-02-15 08:26:40 EST 
BTW as a workaround for problem with lightweights user may use JApplet insteaf of
JPanel and this will work (with double click :) since JApplet is a heavyweight.
Comment 7 Dirk Lemmermann 2007-06-01 05:23:45 EDT 
I am surprised that this bug is not considered a blocker. We have an application that was migrated from Swing to SWT but because of its size we still have tons of SWT dialogs that wrap Swing components. Because of this bug we can not upgrade to Java 5 update 10 or 11 and also not Java 6 (which we really want to). I am assuming that there are many more projects out there that are now stuck with older Java versions. This bug blocks us from adding new features and 3rd party libs that require Java 6.

Dirk
