Comment 37 Andre Weinand 2005-05-12 06:52:29 EDT 
Ok, I think I understand what's going on:

In Display.runEnterExit the branch that contains OS.SetEventLoopTimerNextFireTime is
probably called more often than expected (it looks like it is polling?)
As a consequence the hover timer is reset so fast that it is no longer able to fire.

If I comment out OS.SetEventLoopTimerNextFireTime, the hover show up again.
Comment 38 Silenio Quarti 2005-05-12 13:06:51 EDT
