Comment 9 Timothy Mowlem 2008-11-21 04:27:52 EST 
It would be best not to assume that there must always be two scroll arrows. Apple have always shipped MacOSX in that configuration by default to the best of my knowledge but I vaguely remember the option being available for a short time (maybe in the DP rather than released version). So it is possible in the underlying code to have FOUR scroll arrows, two at each end.

This is not available through the Mac GUI but the option can be turned on using TinkerTool, a well known configuration tool from Marcel Bresink (http://www.bresink.com/products.html). The key point here is that Marcel displays a dialog to explain that TinkerTool only changes preferences and doesn't patch code, which confirms the built in nature of this option.

Actually TinkerTool is very popular so its quite possible that a significant number of users will be using it anyway.

So if possible should take into account both number of scroll arrows and their location.

Comment 15 Silenio Quarti 2009-10-15 11:12:09 EDT

Comment 16 Silenio Quarti 2009-10-15 13:09:13 EDT 
On Windows, GetScrollBarInfo() returns the thumb top and bottom coordinates (left and right for horizontal scroll). So it would be possible to figure out the thumb location for the current selection of the scroll bar. The problem is that the app code (or SWT) would have to change to selection to the appropriate value and them call the new API to get the thumb rect/position. I am not sure this would perform well and it could cause flashing.
Comment 17 Silenio Quarti 2009-10-15 13:22:40 EDT 
With the proposed API (getRangeBounds()), application code still have to do some math to figure out the thumb position for a given selection. I suppose it is something like:

Rectangle rect = getRangeArea();
int y = rect.y + (int)(rect.height * ((selection - minimun) / (maximum - thumb - minimum));

but I am not sure this is correct for all platforms.

For the API proposed in comment#16 (getThumbBounds()/getThumbLocation()),  this math would not be necessary.
