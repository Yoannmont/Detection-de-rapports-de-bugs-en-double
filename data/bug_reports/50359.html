Description Søren Sandmann 2004-01-21 16:41:57 EST 
This GTK+ bug:

    http://bugzilla.gnome.org/show_bug.cgi?id=128678

describes a problem with the vertical toolbar in Eclipse and the unstable
development version og GTK+. I can't get gdb to work on Eclipse, so I have
been using printf statements in the gtk+ code to debug it. These print
statements indicate that something in Eclipse is calling
 
       gtk_widget_set_size_request (toolbar, 1, 26);

and the expected result of that is a 1 pixel wide toolbar, so as far as I can
see there is no GTK+ bug. However, it works with GTK+ 2.2, so something has
changed, and I would really appreciate any help from someone who know the
Eclipse system better than I.

To debug this, you'll need to use version 2.3.2 of GTK+, which will be released
this week, because previous versions still have bug 46225.

Comment 3 Felipe Heidrich 2004-03-19 17:40:05 EST 
I've downloaded and compiled GTK 2.4.0 just to test this bug and it worked fine
on my machine.
I've tested eclipse and the example bellow, both working fine on gtk 2.2.1 and
2.4.0.

import org.eclipse.swt.*;
import org.eclipse.swt.graphics.*;
import org.eclipse.swt.internal.gtk.OS;
import org.eclipse.swt.widgets.*;

public class PR50359 {	  
public static void main(String[] args) {
	Display display = new Display();
	System.out.println(OS.gtk_major_version()+"."+OS.gtk_minor_version()+"."+OS.gtk_micro_version());
	final Shell shell = new Shell(display);
	ToolBar toolBar = new ToolBar (shell, SWT.BORDER | SWT.VERTICAL);
	for (int i = 0; i<4; i++) {
		ToolItem toolItem = new ToolItem(toolBar,0);
		toolItem.setText(""+i);
	}
	Point pt = toolBar.computeSize(SWT.DEFAULT, SWT.DEFAULT); 
	toolBar.setBounds(10, 10, pt.x, pt.y); 
	shell.setBounds(200, 200, 300, 300);
	shell.open();
	while (!shell.isDisposed()) {
		if (!display.readAndDispatch())
			display.sleep();
		}
	display.dispose();
}
}
