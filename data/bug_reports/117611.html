Description Stephen Smith 2005-11-22 20:22:30 EST 
After setting up a WTP project, I was unable to open a Tomcat server.  I'm running Tomcat 5.0.28.  When I try to create a new server, or open Tomcat v5.0 Server, I get the message "Could not load the Tomcat server configuration at /Servers/Tomcat v5.0 Server @ localhost-config.  The configuration may be corrupt or incomplete."  When I look in [eclipse]/Servers/Tomcat v5.0 Server @ localhost-config, there are indeed no files in it.  I tried selecting Tomcat v5.5 and it did create the files in the appropriate [eclipse]/Server directory, but it would not launch.  As an aside, I was not able to get Tomcat 5.5 to run on my machine anyway.  I have tried this with Tomcat already started (under Apache), and without it started.  My Tomcat is configured to use port 8080.  Below is the appropriate log entry

Comment 1 Tim deBoer 2005-12-19 09:56:04 EST 
I have been unable to reproduce this, but another bug raised one possibility - do you have access control working, and does the current user have the correct rights to load from the external Tomcat directory (i.e. the one that you typed in when creating the runtime)? If the user can't load from this directory it would cause this problem, and there appears to be a missing error message.
Comment 2 Dan Bradley 2006-01-07 11:39:43 EST 
I ran into this same problem and it turned out that I would get this kind of error if Tomcat was someplace I didn't have access to (/usr/local/tomcat) when I was running as a non-root user. Moving Tomcat to ~/Applications/Tomcat fixed the problem. It seems that making the error message give you information that you don't have the right permissions would be helpful.

Comment 8 Tim deBoer 2006-06-06 11:18:32 EDT 
This is a different problem. In the case above, the New wizard was not giving an error message and allowing the user to create it anyway. In your case, you are deliberately deleting the server configuration, and that is (correctly) causing the error.

The easiest way around it is to just delete the server and recreate it. If you think the error message could be better please open a new bug with your comments.
