"Extract" is available instead of "Open". (refer to 28001160_1.jpg)
BTW, select "Extract", an exception will be thrown. (refer to 28001160_2.jpg)

Step :
1. Start Eclipse
2. Activate the Database Explorer and Data Output views by selecting the Window | Show View | Other? menu.
3. Expand the Data category
4. Select both the Database Explorer and Data Output items in the Data category and click OK.
5. Right-click on Connections in the Database Explorer and select the New Connection? menu item.
Expand Derby. Select the Derby 10.1 database manager in the tree view.
6. Select Derby Embedded JDBC Driver from the JDBC Driver combo box. Enter the valid connection information from the setup testcase
Database Location:  <derby_install_dir>\Test (ex./root/Desktop/ve)
Class Location:  <derby_install_dir>\derby.jar   (ex. /root/Desktop/ve/derby.jar)
User ID:  can be any value			(ex. root)
Password:  can be blank			(leave blank)
Connection URL:				(jdbc:db2://host:50000/database)

7. Click the Next button to view the Filter page.  Click OK.

8. In the Database Explorer, right-click on the SYSCONGLOMERATES table and select the Data | Open menu item to open the Data editor.

Comment 17 Quy V. On 2006-05-16 19:08:24 EDT 
The patch files add more user friendly error messages when an unknown data type is encountered.  The proper fix to support new data type should be done in the model when the table and columns are first loaded.  Defect 142106 has been opened for this purpose.

The menu item Data->Open does not Exists.  The following is a description of the menu items and their purpose:
Data->Edit opens the table editor and populate it with the data from the database to allow users to modify and insert rows in to the database
Data->Extract saves the table data to a file which can be used as input to load into another table
Data->Load Reads the data from a file (from Data->Extract) and load that into a table.

The test cases will be modified to reflect the new wordings.
