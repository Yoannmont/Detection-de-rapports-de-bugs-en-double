Comment 7 Kyle Secrist 2003-09-03 16:22:24 EDT 
Ahh, no, it's _not_ a program I'm developing.  As in the first discription of
this issue - when eclipse opens, it displays an "Operation in progress..." modal
which you can't escape from, or cancel. Once this begins to happen in eclipse,
there is no way to stop it from happening each time eclipse is run. I moved my
entire instillation to a backup folder and installed eclipse anew. The  freshly
installed version does not exibit the problem, while the backup version does.

I learned a bit more about eclipse, and I was able to see that it creates a
.lock file in the workspace/.metadata directory for this operation.

WORKAROUND:
I discovered a workaround while experimenting with my 'bad' version, trying to
gather more useful debugging information. If you run the application with the
below command, the modal box will not show, and the application will run,
however the CPU will still be heavily used by one of the java processes running
under eclipse:
./eclipse -refresh
After I got this working, I noticed that the Package Explorer was not behaving
correctly, the slider bar wold not move, and there was a lot of 'flicker' in the
  UI.
I closed the Package Explorer, and the CPU Utilization dropped to normal levels.
Upon closing eclipse, and re-running it normally. All was well!

Looking at my workspace settings, I see that I have Refresh Workspace on Startup
checked, which may or may not have something to do with it.

I hope helps.
-Kyle
