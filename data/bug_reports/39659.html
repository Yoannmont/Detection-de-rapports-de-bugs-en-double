Description Bob 2003-07-04 16:46:40 EDT 
Select Window->Preferences. Double click one of the installed JRE's to bring up the Edit JRE dialog.  
The bottom of the window is cut off making it impossible to OK or Cancel the dialog. In addition, 
the various Up, Down, Remove, etc. do not appear to be functioning since they never become 
enabled. This is actually a very critical failure due the way the third party jars are added to the JRE 
when copied to the Library/Java/Extensions folder. For xample, I develop a JDBC driver, which 
normally would reside in the Extensions folders. However, moving the driver to that folder and 
subsequently removing it for any reason creates a circular build path problem where the driver jar 
file is now dependent upon itself. The only way I could resolve this issue was to remove Eclipse 
and re-install;-(
Comment 1 Andre Weinand 2003-07-04 18:38:02 EDT 
Since I don't experience the Window being cut off could you please attach a screenshot?

The Up, Down, etc. buttons become enabled after the checkbox "Use default system 
libraries" (see my screenshot) has been checked off and a list selection has been made.

I would not add third party jars to a JRE. Instead, select your project, open the properties 
dialog on it, select the "Java Build Path" page, go to the "Libraries" tab and press the 
"Add External Jar" button. Here you can select your jar.

Comment 5 Andre Weinand 2003-07-05 17:59:36 EDT 
Just a warning: the problem with Library/Java/Extensions is that it affects all Java 
programs (because its part of the boot classpath). So for example if you put a wrong 
version of xerces.jar there, Eclipse will crash, because it will find your version before 
finding its builtin version.

No, I don't think that the "Add External Jar" button is chopped off (because its not at the 
bottom of the dialog, but at its right side). However, its in the Libraries tab of the "Java 
Build Path" page of the project properties dialog.

Anyway, what version of Eclipse are you using?
