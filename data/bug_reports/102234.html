Comment 3 Jonathan Stinton 2005-06-30 10:05:49 EDT 
Hi James,

One further question for you.  Did you expect to see any particular properties
for a GridBagLayout?  Java does not expose any properties directly on it for us
to add to our customize dialog, that I am aware of at least.

What you may be thinking of is the Constraints on the children.  If you select a
child of the container set to GridBayLayout you can then use the "Component" tab
of the "Customize Layout" window to edit its constraints.  The 1.1M2 page
provides access to some of the properties (fill, anchor, inset) and soon there
will be a release with a few extra properties (weight, span, padding).  The
property sheet for the child already has all of these available, however.

If you have any further questions or requests for enhancements to GridBag please
let me know as I'm working on it right now.

Thanks
-Jon

Comment 5 Jonathan Stinton 2005-06-30 13:31:32 EDT 
James,

You are correct in that you would need to use the GridBagLayout to do this.  The
option you are probably looking for is the Weight setting.

Set the parent to GridBag, select a child in the left column, then in the
property sheet go to Constraints->WeightX and change this to 0.25.  Then select
a child in the right column and change its weight to 0.75.  This will give you
the effect of a right side 3x the size of the left side.  As I mentioned in my
above comment this feature is not currently in the customize layout window for
M2.  However, it will be available there in 1.1.

As for the "Show Grid" option in the context menu, that is an out of date help
file.  We recently added a preference to Window->Preferences->Java->Visual
Editor called "Show grid when container or children are selected".  This is on
by default (which is why you see the grid now).  If you were to turn it off the
grid would hide from you.

Hope all that helps, let me know if you see anything else weird.  We will fix
those help docs.

-Jon

Comment 7 Jonathan Stinton 2005-07-01 10:02:20 EDT 
James,

I brought it up in the VE and played around with it a little.  Currently you
have the ipad_x set on all of your controls.  I believe when you do this it will
force the controls to be that size despite their weight.  The weight field
simple says "if there is extra space, grab this much of it."  What I did was to
set the fill on the empty labels to fill horizontal & vertical, then set the
ipad_x on all 4 controls to zero, this seemed to do the trick for me.

You may not have intentionally set the ipad values at all.  This is something we
are looking at right now on the VE.  It has to do with when you convert from one
layout to another we try to keep the components looking exactly as they did
before.  When switching to GridBag this often means setting the ipad values for
you.  

I think we will probably leave this in for the 1.1 release because the enhanced
layout customizer will show the user the ipad values without having to search
for them in the property sheet.

Hope that helps,
-Jon

Glad that worked, a different version I came up with would be this, but you 
know better what you want so feel free to play around with all this.

1) Set all 4 controls' padding (x & y) to zero.
2) Set all 4 controls to fill horiztonal and vertical.
3) Set 2 left controls weight to .10, left inset to 20, right inset to 10.
4) Set 2 left controls text alignment to left align.
5) Set 2 right controls weight to .90, right inset to 20.

This should give you a good feel for what everything means in gridbag.  Weight
is relative to free space, padding is a definite size for the control.  Inset is
how much space is between the next control on the given side.  Fill means take
up the entire cell, whatever size that cell may be.

The reason you are getting the parent selected is probably because you have to
click directly on the control to select it.  Often there is some space around
the control within the cell, and clicking on that is actually a click on the parent.

Good luck with the class.  I'm going to close this bug, but feel free to email
me directly with any other questions, or post them to the visual editor
newsgroup which can be found from our main page... www.eclipse.org/vep
