Comment 6 Henry Huang 2005-04-20 02:48:15 EDT 
I am re-opening this bug to allow the developers to make sure that there is 
indeed no problem here.

When using no command-line arguments, the problem happens as stated before in 
the problem description.  The Linux terminal dumps 

JVMDG217:  Dump Handler is Processing Signal 11 - Please Wait.

and the Eclipse window indefinitely hangs.

However, when using the -clean -vmargs -Xj9 arguments in the command line, 
the test case executes with no problem.  A similar situation can be seen when 
performing a "Search" in the External Browser, Web Browser Preferences.  If no 
command-line arguments are used, then Eclipse hangs.  However, using the 
command-line arguments results in proper execution.
