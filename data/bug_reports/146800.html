Comment 36 John Arthorne 2006-07-20 10:25:19 EDT 
This seems very similar to bug 144823 and bug 149948. To summarize the common factors (please correct me if wrong):

 - Occurs on Windows only
 - Occurs on Sun 1.5 VM only
 - Happens after a long period of use
 - Eclipse SDK + other plugins such as WTP or other Callisto features installed
 - Stack traces are hung within class loader natives.  No apparent application-level deadlock.
 - CPU usage remains high indefinitely (97%, or 50% on a dual processor machine)

One theory that fits this information is that we are seeing a bug in the class loader implementation that is occurring when the VM is low or out of permanent generation space. For example, it may be trying to load a class, but cannot load it because there is not enough space in the perm gen area.  The Sun VM has a fixed limit on the size of this space that is used to store classes and other VM structures. This limit can be increased using the -XX:MaxPermSize argument.  

Does everyone see this problem disappear or drastically reduce when setting a larger perm gen size (such as -XX:MaxPermSize=200M)? Are the people seeing this problem also occasionally seeing OutOfMemoryErrors in their log file?
