Description Ted Kubaska 2007-04-25 05:35:04 EDT 
I installed eclipse-SDK-3.3M6-linux-gtk.tar.gz on Suse 10.2. Build id: l20070323-1616. When I make a helloworld Java project, eclipse hangs on opening the Java Editor on my Hello.java class. The hang does not occur with the same eclipse download installed on Ubuntu Dapper. Ubuntu Dapper and Suse 10.2 are the only two Linux distros I tried.

Hello.java is created ... I can see it in my workspace from a terminal window. If I kill eclipse (I cannot exit normally) and restart eclipse, I can display Hello.java with the System Editor. It will not display with the Text Editor or the Default Editor.

I tried various JDKs, but the bug does not appear to be dependent on the JDK. My last test was with Sun's JDK 1.5.0.0_07.

Comment 7 Bogdan Gheorghe 2007-04-30 10:58:20 EDT 
This is a duplicate of Bug 164611. The root cause is a problem in GTK 2.10 (which is used on Suse 10.2 but not on Dapper which uses 2.8).

We have released a fix for this from our side (you can try any Integration build >20070411 or wait for the M7 build).
