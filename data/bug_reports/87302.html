Description Paul Slauenwhite 2005-03-07 14:04:21 EST 
Clean-up TPTP Logging samples.

In preparation for a Hyades/TPTP presentation at EclipseCon 2005, several of 
the TPTP Logging samples require clean-up and/or improvements to better 
showcase TPTP functionality.  

The enclosed ZIP contains demo Analysis Engine, Correlation Engine and Log 
Parser (local and remote rules-based parsers) plug-ins that may be used to 
improve the TPTP Logging samples.  The Log Parser sample should be improved to 
showcase local and remote deployment of both rules and static log file parsers.

NOTE:  The enclosed samples will need to be ported to TPTP 4.0 since they where 
written for Hyades 3.2.

Comment 4 Paul Slauenwhite 2005-03-08 12:12:33 EST 
I had to modify the following to make the example work in TPTP 4.0:

1. MyStaticParserExtension.java:
(Changed the parser wrapper)
import org.eclipse.hyades.logging.parsers.importer.ParserWrapper;

/**
 * This is just a simple extension of the StaticParserWrapper in Hyades
 * so that we can refer to it in our extension point.
 */
public class MyStaticParserExtension extends ParserWrapper {

	public MyStaticParserExtension() {
		super();
		currentPlugin = "custom.myparser";
	}
}

2. Dependancy in plugin.xml
Removed:
      <import plugin="org.eclipse.hyades.logging.adapter.config"/>
Added:
      <import plugin="org.eclipse.tptp.platform.log.views"/>         


3. Extension points:
Replaced:
        <extension
                point="org.eclipse.hyades.logging.parsers.logParser">
with:
   <extension
         point="org.eclipse.tptp.platform.log.views.logParser">

Replaced:
   <extension
         point="org.eclipse.hyades.logging.adapter.config.staticParser">

with:
   <extension
         point="org.eclipse.hyades.logging.parsers.staticParser">
