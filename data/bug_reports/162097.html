Description Michael Konopka 2006-10-24 11:26:23 EDT 
Eclipse 3.2.1 has changed the org.eclipse.osgi bunlde name such that WebStartMain cannot parse correctly.

WebStartMain throws a null pointer exception on line: 58 "System.getProperties().put(PROP_FRAMEWORK, fwkURL);"  because fwkURL is null, causing webstart to shutdown.

I walked the code in the debugger and found that the parsing methods "extractBundleIdFromManifest(String)" & "extractBundleIdFromURL(String)" return the string value "org.eclipse.osgi_3.2.1.R32x_v20060919".  Then method WebStartMain.mapURLsToBundleList() tries to get the bundle structure from the map "bundleList" two times (once on line 185, then on line 190) unsuccessfully because the key in the map bundleList is "org.eclipse.osgi".

The problem is the inclusion of the extra underscore in the bundle version part after the "R32x" that throws mapURLsToBundleList() off.

Comment 6 Pascal Rapicault 2006-12-13 08:41:25 EST 
Created attachment 55578 [details]
Patch webstart main to fix the problem

This patch fixes the problem mentioned here. The changes in the patch look impressive but they are not really.
Here is the idea behind the changes:
1) there used to be a method who was returning a string representing the concatenation of the bsn and the version (bsn_version), either extracted from the URL or from the manifest.
2) based on the return value of this method a lookup was done to see if the bundle was in the osgi.bundles list.

The changes are in 1) return an object keeping bsn and version separate. In 2) react to the changes. 
The other big change is in the method responsible for extracting bsn / version from a URL, since it now validates the version number.

I will attach a built version of the startup.jar.
