Description Xuan Chen 2006-06-13 14:10:59 EDT 
Our product includes a debugger which is built on top of Eclipse debug
support.  When we moved to Eclipse 3.2, M4, we noticed that when the
application is running, the Halt and Terminate buttons are not available
any more, since there is nothing selected in the debug view.   But in
previous release of Eclipse, when an application is running, the Halt and
Terminate button were available, since one of the threads will always been
selected.  This cause some usability issue of our debug support.

We need the debug target be selected while an application is running.
Comment 1 Darin Wright 2006-06-13 14:18:54 EDT 
By default, we should select a debug target or process in a launch, when added to the launch (unless something else in the view is selected).

Targetting for 3.3
Comment 2 Samantha Chan 2006-06-15 00:16:01 EDT 
Hi Darin, 

Is it possible to target this for 3.2.1?

I think we need something selected after a resume has happened too, not just after the launch.  In 3.1, when a thread is resumed, the stackframe is collapsed, and the thread is selected.  In 3.2, when a thread is resumed, nothing is selected.

This is also affecting the Memory View with its flashing and not being able to show changes after resume and hitting a breakpoint.  (Bug 144221)

I have already put in a fix to ignore the empty selection in order to avoid the flashing.  However, now that I think about this, the Memory View should have emptied the view upon an empty selection.  It works on the current debug context, and if there is no debug context, the view should be emptied.  As long as one of the debug elements is selected after a resume, the Memory View will not flash.

So, this is the behavior I would expect, after looking at 3.1 again:
1.  When a debug session is launched and running, the debug target should be selected.
2.  When a debug session is resumed, the thread that the resume was initiated from should be selected.  If the user has selected the target, and resumed the entire debug session, then the debug target should remain selected.

Thanks...
Samantha
