8) press Ctrl+Space again

Comment 14 Steve Northover 2006-09-08 16:51:42 EDT 
The summary is that Windows draws right away when setting a column width rather than damaging an area and waiting for a repaint.  Here is the comment from the work around:

/*
* Feature in Windows.  When LVM_SETCOLUMNWIDTH is sent,
* Windows draws right away instead of queuing a WM_PAINT.
* This can cause recursive calls when called from paint
* or from messages that are retrieving the item data,
* such as WM_NOTIFY, causing a stack overflow.  The fix
* is to turn off redraw and queue a repaint, collapsing
* the recursive calls.
*/
Comment 15 Steve Northover 2006-09-08 16:58:38 EDT 
NOTE: I only fixed the recursive crash.  I didn't fix every set width for every column.  Do you have some example code that shows the problem you are seeing?  If so, please enter a bug report for it.  Thanks.
