Description Richard Vaughan 2005-11-23 07:28:01 EST 
When doing [Help]>[Software updates]>[Find and Install...] Eclipse locks up, and the IDE is unresponsive (hour glass when the mouse moves across it, etc).

There is no java core dump (that I can find), and no entries are written to the log file - verified by clearing the log first.

The only way to 'recover' is to use TaskManager to kill the Eclipse process.

This is with Eclipse build I20051102-1600 (3.2 M3 I think)

Comment 5 Martin H. 2006-12-14 16:29:06 EST 
I'm experiencing the same behaviour.

I'm running:
* Eclipse 3.1.2 (Build id: M20060921-0945)
* on Windows XP 
* over my company's VPN 
* and through a Proxy Server.
* I let it run for 5 minutes and experienced none of the "timeout/only first time's a problem" like others.

I've used the [Software updates] > [Find and Install...] before in the office through my company's proxy server but never from home through both VPN and Proxy Server.  After reading the comments I dropped off VPN and and removed the proxy server settings (now going with just my ISP) I was able to access the Find and install.

I'll have to try again from the office (with Proxy but not VPN) to see if something is permanently broken in my install or if it's the combination of VPN and Proxy.

So, if you have the luxury, the workaround might be to get a hold of a an internet connection that does not need a proxy server to perform the updates.

BTW: I found that the [Sofware Updates] > [Manage Configuration] still works over proxy so it's only a matter of getting the plugin the first time and then using the "Scan for Updates".
