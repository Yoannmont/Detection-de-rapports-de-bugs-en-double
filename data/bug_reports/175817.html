Description Markus Keller 2007-02-28 06:50:00 EST 
I20070227-0800

FilteredItemsSelectionDialog should show "n items selected" in status line as soon as 2 or more items are selected. This would be in line with what views show in the workbench window status line on multiselection.

The ItemsListSeparator should not be counted as a selected item.
For the empty selection, the status line should stay empty.

Comment 9 Krzysztof Michalski 2007-05-11 07:48:22 EDT 
Right we shouldn't change the strategy of hitting the label provider because we will resign from some profits of label providers mechanism. Eg. decorating it by image or new text. Moreover as Simon said we should back to use String than DetailsMessage because it will be used in extended label provider. In conclusion if we change strategy of hitting and if we use DetailsMessage we prevent developer who wants to extend our dialog from decorating these other messages(like number of selected items).
Comment 10 Markus Keller 2007-05-11 08:27:36 EDT 
What meaningful decorations can you imagine for a string like "7 items selected"? If you really want to give clients an opportunity to change this message, then you should probably pass them the full list of selected items, rather than passing a pre-composed string they would have to parse.

I see two possible solutions:

a) pass an array or a List of elements if multiple are selected
  => would require most current clients to adapt their details label provider

b) pass the ready-made string as suggested, but tell clients that they can replace the String and should use getSelectedItems() to find out how many and which items are selected (getSelectedItems() should be fixed to filter out the ItemsListSeparator).

In any case, setDetailsLabelProvider(..) would have to specify how clients must handle the additional element types.

getSelectedItems modifies a list while iterating over it. This could totally break your iterator. As the list is immutable anyways it won't get that far before the UnsupportedOperationException occurs.
