Description Tammo van Lessen 2007-10-17 14:10:51 EDT 
Hi folks,

I've got a strange error message when I was synchronizing a Mylyn Task Query for a saved filter on Apache's JIRA.

I've selected the query from the list, the query URL looks also fine when opening it in the browser, however when synchronizing I get the following error message:
"Error parsing server response: No project with key 'SLING' found".

That's pretty strange as the project key is 'ODE' and neither the filter name nor the XML representation of the query page contain the term 'SLING'.

Any ideas?

Thanks,

Comment 1 Eugene Kuleshov 2007-10-17 15:32:19 EDT 
First of all which version of JIRA connector you are running? Can you please attach your Eclipse configuration details from Help / About Eclipse SDK / Configuration Details.

This most likely happens because some issues from your query results been moved from other project. I thought we fixed this issue at some point, so I recommend to update to the latest build. If that don't resolve this issue you can also try to update repository attributes and if that won't help, please enable debug tracing as described in [1], it should dump query error in the eclipse log or the Error Log view, so please attach that log to this issue. Thanks.
