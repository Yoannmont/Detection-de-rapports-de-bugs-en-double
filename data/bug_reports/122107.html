Description Noel Bush 2005-12-27 01:59:26 EST 
After upgrading WTP from 0.7 to 1.0 (following a fresh install of Eclipse 3.1.1), I found that previously working WTP projects ("Dynamic Web Project"s) no longer worked:  I got complaints about missing context.xml files, complaints about missing libraries, and the "Web App Libraries" special object disappeared.

I tried to fix one of the projects by creating a new "Dynamic Web Project" using the wizard, then carefully examining the .project file and files in .settings, copying over their contents to my "broken" project.  But it only made the situation worse.  The project said that it could not be published because of missing libraries.  But I could not find any place where these "missing" libraries were specified.

Then, when I created a new "Web App Libraries" object, the view of its contents was inconsistent between the Project Explorer and the Project Properties tab.  In the Explorer, it contained only items that I expected it to (those jars which were in WEB-INF/lib), but in the Project Properties, it listed a whole bunch of libraries that were not in the lib directory, or even anywhere.  Moreover, I could not find *any* file in the project where these files were listed.  Somewhere there is apparently a "phantom" part of the configuration, whose location I can't discern.

I tried removing the server configuration (Tomcat 5.5) to which this project was assigned, and creating a new server configuration.  (Just guessing that maybe some of these "mystery settings" were in the server config.)  But now, I cannot even add the project to the server!  When I try, I get a useless error message (see attached screenshot), which says:

Server Error
null
Reason:
null

I have no idea what to do.  My work is suddenly at a standstill.

Strongly held beliefs as a result of this experience:

1.  There *has* to be some way to "reset" or otherwise "clean up" an existing WTP Dynamic Web Project.

2.  *All* of the settings for a WTP project must be in a file somewhere inside the project!  Otherwise, how can anyone be sure of being able to share the project?  It appears that, at the very least, the specification for the mysterious "Web App Libraries" special object is not in a file that is contained within the project itself.  Where is it?  In the .metadata folder?  I tried searching, but couldn't figure it out.  This is really scary.

Comment 5 John Lanuti 2006-02-06 20:46:16 EST 
Hi Noel,

The configuration settings for flexible projects is stored in the .component file.  This is a direct replacement for the .wtpsetting file from 0.7.  This should be automatically migrated for you in WTP 1.0.1 the first time the component is loaded. There are also facets in wtp 1.0.1 which are stored in the org.eclipse.wst.common.project.facet.core.xml file.  These should also be added for you.  These can also be edited using the project property pages.  This is the preferred way to "manage" projects.  The .component file is also no secret and is well known.

Is the web app libraries you're main concern?  Is this really the main thing what wasn't migrated properly?  (Not including the server part) The original defect is pretty old, this is the first I've seen it and 1.0.1 is basically shut down.  Can you list here the blocking issues you currently can reproduce and can you provide a test project we can use?  And if you do have a reproducible scenario, would you consider blocking, as in should we hold up wtp 1.0.1 release?  Or can we lower the severity to critical?  Have you tried on windows or do you think it is a linux only issue?

Thanks for your candid feedback Noel, I assure you we don't want things to be a "great mystery"!
