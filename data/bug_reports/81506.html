Comment 4 Jerome Lanneluc 2005-01-13 12:48:01 EST 
Unfortunately the indexing thread cannot take a lock on the resource as this
would cause a deadlock in the scenario where the user renames a type:
1. The main thread takes the lock on /P/p (so as to modify A.java)
2. The indexing thread attempts to lock /P/p/A.java (so as to read its contents)
3. The main thread proceeds and search for references to A.
4. The search needs to wait for the indexing to finish to get accurate results
(WAIT_UNTIL_READY option of the SearchEngine)
This is deadlock situation.

There is no solution to this problem at the JDT Core level as JDT Core doesn't
own the lock taken in step 1.
