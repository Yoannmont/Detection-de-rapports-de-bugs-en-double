- class names
- php built-in functions (str_replace, ...)

- properties ($this->any ...)

And additionally:

- static method calls
- static properties

Comment 5 drm / Gerard 2007-06-05 10:04:44 EDT 
A vote from me as well. I have composed a wish list for all separate code coloring types ordered by preference.

Parenthesis
Brackets
Braces
Single quoted strings separately
Double quoted strings separately
Arithmetic operators
String concatenation operator
PHP Built-in functions
PHP Built-in functions (deprecated)
Separate coloring for function names when defining functions and calling functions
Ditto for classes
User function calls (recognized by the builder; i.e. can be ctrl-clicked)
User function calls (Not recognized by the builder; i.e. can not be ctrl-clicked)
PHP Built-in constants
User constants (recognized by the builder; i.e. can be ctrl-clicked)
User constants (not recognized by the builder; i.e. can not be ctrl-clicked)
Type-cast operators separate from keywords
Loop-construct (for, foreach, while ...) keywords separate from conditional (if, switch ...) construct keywords

It may seem overdone, but I have some experience with the IDEA java ide, which has about everything mentioned above, and it is really improves readability of code and the ease-of-use of code navigation using ctrl+click and/or F3 (open declaration).

Additionally it would be nice to be able to select the color of the underline.
