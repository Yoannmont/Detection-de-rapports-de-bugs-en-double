Comment 32 Sonia Dimitrov 2004-10-20 12:30:31 EDT 
There are a couple of problems leading to the compile errors in the source 
builds when using the IBM SDK (therefore, failure to compile the IdReplacer 
class used in the taskdef)

1.  The java libraries are not contained in an rt.jar (the current default 
bootclasspath setting in the scripts), but broken into a number of jars.  It 
is awkward to specify the bootclasspath with the -bc option in this case.
2.  There are some file encoding mismatches (ie. system default may be UTF-8, 
but some files require ISO-8859-1, org.eclipse.search for example)

I will attach a build shell script and build.xml which should be placed in 
the "buildScripts" directory or top level directory extracted.  These should 
address both issues by setting a proper default bootclasspath and setting the -
encoding compiler arg to ISO-8859-1 as a workaround.

To execute, "build -os linux -ws gtk -arch ppc" should be all that is required.
