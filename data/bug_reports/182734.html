Comment 11 qingwu lin 2007-07-05 21:11:52 EDT 
This defect's root reason that eclipse don't load and  instantiate my PropertyTest class which was used to determine whether make visibility of our BtM Add/Remove context menu item in the popup menu of eclipse.  

So, if eclipse don't instantiate my PropertyTest class where all control logic codes are placed, Our BtM Add/Remove context menu item will always be visible in popup menu by default. Meanwhile, our BtM context menu items are dependent on the context menu org.eclipse.jdt.ui.source.menu defined in the plug-in org.eclipse.jdt.ui. In another word,  the context menu org.eclipse.jdt.ui.source.menu which is displayed as "Source" string in the popup menu is the parent menu item of our BtM context menu item. In such case, if user right-click a Resource Element(such as IJavaProject, ICompilationUnit, IPackageFragment, IFolder, and so on) which don't be existed context menu org.eclipse.jdt.ui.source.menu("Source" menu item), for example, user can right click a tree item in Test navagator view ; eclipse will reported that org.eclipse.jdt.ui.source.menu is invalid in .log file because our BtM context menu item is always shown in popup menu but the context menu org.eclipse.jdt.ui.source.menu("Source" menu item)  isn't existed yet(a context menu is shown in popup menu only if its parent menu item must be exist).

If eclipse instantiate my PropertyTest class, we can determine that which case will make visibility our BtM context menu item in the popup menu. 
According to the description of org.eclipse.ui.popupMenus extension point, PropertyTest extension point is used to determine whether make visibility for the context menu item before eclipse show the popup menu contributed by other plugins.  But in fact, eclipse don't load and instantiate our PropertyTest class before eclipse show the popup menu, it indicated that  PropertyTest extension point wasn't taken effect yet.
