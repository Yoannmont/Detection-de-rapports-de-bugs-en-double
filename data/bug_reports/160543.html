Description Carolyn MacLeod 2006-10-11 15:36:43 EDT 
I20060922-0010

Get inspect32.exe from Microsoft:
http://www.microsoft.com/downloads/details.aspx?familyid=3755582A-A707-460A-BF21-1373316E13F0&displaylang=en
Click the Download button to the right of inspect32.exe

Start eclipse.

Start inspect32.exe, and under Options, check "Show Highlight Rectangle".
This shows you which widget has focus as you tab around in eclipse.

Using a combination of "Ctrl+Tab" (forward tab traversal) and "Shift+Tab" (backward tab traversal), change the focus to the "Show View as a Fast View" button. (Note that only "Ctrl+Tab" will get you out of an editor). Now look at the "Name:" that is showing in the inspect32.exe window: "none [false]". This should be: "Show View as a Fast View".

The way to achieve this is to add an AccessibleListener to the toolbar, and make the listener implement the getName method, which needs to set the value of event.result to the desired string for the current toolbutton.

So the following code (approximately) needs to go in the org.eclipse.ui.internal.presentations.defaultpresentation.DefaultTabFolder constructor (maybe other places as well):

actualToolBar.getAccessible().addAccessibleListener(new AccessibleAdapter() {
	public void getName(AccessibleEvent e) {
	        if (e.childID == actualToolBar.indexOf(pullDownButton)) {
	        	e.result = WorkbenchMessages.Menu;
	        }
	}
});

(also need to make actualToolBar and pullDownButton final and add the import for org.eclipse.swt.accessibility.*)

Comment 3 Carolyn MacLeod 2006-10-11 16:08:44 EDT 
Here is the fix for the "Show View as a Fast View" button accessible name problem:

add the following code after showItem.setToolTipText(menuTip); in org.eclipse.ui.internal.FastViewBar.createChildControls()

menuTB.getAccessible().addAccessibleListener(new AccessibleAdapter() {
	public void getName(AccessibleEvent e) {
	        if (e.childID == menuTB.indexOf(showItem)) {
	        	e.result = menuTip;
	        }
	}
});


(also need to make menuTip final and add the accessibility import).
Comment 4 Carolyn MacLeod 2006-10-11 16:52:00 EDT 
Actually, for the view menu accessible name, I think I would personally go one step further and use the view title in the accessible name string as well. In English, this would simply be prepending the view title, but you probably should localize it properly using MessageFormat.

Here's an example. Note that this is horrible code - it reaches all over the place, and I had to make tabFolder public in PaneFolder class, and it just slams all of the strings together instead of using MessageFormat... I'll leave the refactoring up to you to make it pretty <g>. The bottom line is that a screen reader will say, for example "ToolBar Button: Package Explorer View Menu" when a blind user shift+tabs to the view menu. Currently, JAWS just says "ToolBar Button".

In org.eclipse.ui.internal.presentations.defaultpresentation.DefaultTabFolder()

actualToolBar.getAccessible().addAccessibleListener(new AccessibleAdapter() {
	public void getName(AccessibleEvent e) {
	        if (e.childID == actualToolBar.indexOf(pullDownButton)) {
	        	e.result = paneFolder.tabFolder.getSelection().getText() + " View " + WorkbenchMessages.Menu;
	        }
	}
});
