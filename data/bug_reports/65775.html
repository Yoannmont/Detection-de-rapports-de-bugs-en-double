Comment 24 Douglas Pollock 2005-06-15 11:29:21 EDT 
The problem is that update on the menu manager is only being called when shell
activation or deactivation occurs.  It is not being called when the binding
manager changes.
Comment 25 Douglas Pollock 2005-06-15 11:36:03 EDT 
Created attachment 23206 [details]
Patch to "org.eclipse.ui.workbench"

This makes it so that any binding change will trigger an update of the text on
the menu managers of the all the workbench windows.  This is sub-optimal in
performance.  A better solution would be for the menu manager to register as a
binding manager listener, and only update itself if its corresponding key
sequence becomes (or ceases to be) a binding.
