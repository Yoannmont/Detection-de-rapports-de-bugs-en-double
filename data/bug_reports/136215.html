Description rice 2006-04-11 18:33:49 EDT 
I am an experienced java developer, and have been using eclipse for over 3 years. I have just installed Eclipse 3.1.0 onto an acer twin core laptop, running Windows XP service pack 2.

When I try and debug anything (even HelloWorld) I get an instant dialog box "Cannot connect to VM". 

In the log file I get the dump that I display at the bottom of this message.

A search of the Internet reveals that there are two likely candidates: either a firewall, or some clash in the path between different JREs using to compile and run. I have ensured that the only installed JVM is 1.5.0_06, and the path to it is C:\Program Files\Java\jre1.5.0_06. I have disabled the firewall, and still have the issue.

A third theory is that the issue might be caused by an existing piece of software that is bound to the debug port. I performed tests on a working eclipse and believe that the required ports are 2681..2684. I have attached a netstat -a dump, and I think this shows that the ports 2681 to 2684 are free (I am not very experienced in the use of netstat, so I might have misunderstood this)

A fourth theory is that this might be a result of the debug time out, so I have set this to 5000.

I have run out of things to try, so any advice is appreciated.

Thanks

Phil

Comment 1 Kevin Barnes 2006-05-02 12:58:23 EDT 
We're essentially opening 2 ServerSocket's. One to find a free port, and one to listen for the connection from the target vm. The second server socket uses the port that was discovered by the first.

Does the following code run on you machine w/o errors?

import java.io.IOException;
import java.net.ServerSocket;

public class SocketTest {
	public static int findFreePort() {
		ServerSocket socket = null;
		try {
			socket = new ServerSocket(0);
			return socket.getLocalPort();
		} catch (IOException e) {
		} finally {
			if (socket != null) {
				try {
					socket.close();
				} catch (IOException e) {
				}
			}
		}
		return -1;
	}
	
	public static void startListening(int port) throws IOException {
		ServerSocket socket = new ServerSocket(port);
		socket.close();
	}
	
	public static void main(String[] args) {
		try {
			int port = findFreePort();
			if (port > 0)
				startListening(port);
		} catch (IOException e) {
			e.printStackTrace();
		}
	}
 }
