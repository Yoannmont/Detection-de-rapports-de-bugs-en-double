Description U. Melendez 2005-11-25 02:29:53 EST 
Overview Description:
When I create a new Visual Class using the Wizard, I get an error stating: "Creation of element failed."  The Wizard window does not close, and the class IS created.

It may also be relevant that I configured the default JDK.  I had eclipse search my system for JDKs, and I switched the default from 1.4 (the version eclipse had found automatically) to 1.5.

I also created an Ant build.xml before attempting the first Visual Class.


Steps to Reproduce:
I performed a fresh download and installation of eclipse 3.1.1.  I searched for updates.  None were found.  I installed the Visual Editor, EMF, and GEF.  Quit and restarted eclipse.  Performed an update, and installed updates for the new group of features (VE, EMF, GEF).  Quit and restarted eclipse.

1) Create a new project via context-menu in Package Explorer.  Used separate src/bin folders and Java 5.0.  Finish.
                          
2) Right-click src folder, New | Other... | Java | Visual Class.  Next.
   Java Class Name: MyGui
   Style: Swing-Dialog

4) Click Finish.


Actual Results:
Observe error window titled "New" with the message "Creation of element fialed."  Error log info is attached below.

Expected Results:
The New Java Visual Class should disappear, and the workspace should show the new visual class.

Build Date & Platform:
eclipse build id: M20050929-0840

Additional Information:
Below the text "Creation of element failed," the following exception message is shown: "org/eclipse/ve/internal/jfc/codegen/jjet/wizards/contributors/JContentPaneGenerator1 (Unsupported major.minor version 49.0)"

Comment 7 Olivier Thomann 2006-01-31 11:02:26 EST 
If the visual editor is opening the class using reflection, then you need to run your workspace using a 1.5 VM if you want to set up your project compiler settings to use 1.5 settings (-source 1.5, -target platform 1.5 and -compliance 1.5).
According to the data you provided, you are running using a 1.4 VM. A 1.4 VM cannot open a .class file compiled with 1.5 settings.

Comment 12 Olivier Thomann 2006-02-24 20:36:23 EST 
(In reply to comment #11)
> I think I understand now.  I originally thought you were talking about the VM
> used *within* eclipse, but you were referring to the VM running eclipse.  I
> hope that's correct. 
Correct.

> A suggested improvement to avoid this trouble would be to alert the user about
> the version conflict as soon as he tries to create a 1.5 project while eclipse
> is running atop an earlier VM.
Unfortunately this is not possible since it is perfectly fine to run Eclipse using a 1.4 VM and then install a new JRE (1.5 VM) to develop 1.5 code.

The problem that you have in your case is that the VM running Eclipse is used with reflection to open some classes compiled by Eclipse compiler using the VM on the classpath of the project. In your case a 1.5 VM.
This is why you ended up with this error.

I think this should be documented in the doc of the visual editor. This is not a problem with the Platform or JDT.
