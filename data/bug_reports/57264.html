Comment 7 Dominique Guilbaud 2004-04-05 05:48:35 EDT 
It was due to a missing widget update call in setFrame for the sdwidget class.
Also, now, to avoid sequence diagram blinking please call the new API 
setFrameAndEnsureVisible in the SDView class, when you want to move the diagram 
right after setting it in the View. This will move it to the desired position 
without moving it to 0,0 first.
Comment 8 Eugene Chan 2004-04-05 09:07:26 EDT 
Tested with the latest code in sd.ui and found the following problems.
Rerun test scenario, expand last node, see peropeties value, then select the 
node above the last selected node, and make a selection anywhere in the view, 
see 2 selections(highlighted) exist in the view. and no property shown in the 
second selection. Seems like the call in view.setFrameAndEnsureVisible
(frame,node); forgot to update the selection and notify the listener, as well 
updating the selection.

The code changed on sd.logc is basically 
replacing view.getSDWidget().ensureVisible(selNode);
to	view.setFrameAndEnsureVisible(this.frame,selNode);
