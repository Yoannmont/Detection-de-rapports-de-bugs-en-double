Comment 44 Darin Wright 2006-07-27 12:23:57 EDT 
Debug has fixed the following (in 3.3) - we no longer report/log an error when source cannot be found *and* an arhcive goes missing. Instead, we simply offer an "warning" dialog saying that source could not be found, and there is a missing archive. 

NOTE: We do not fail/abort source lookup when a source path contains one missing archive - we search the entire path for source even if one or more archives are missing. I've tested various scenarios to confirm this. We only report an error if the entire path is searched *and* source cannot be found *and* at least one archive is missing.

I do not think this fix will address the problem that most people are encountering. It looks like people are really encountering the problem of a broken build path when migrating from 3.1 to 3.2, and the location of JUnit changing.

See bug 150961.

For this reason, I don't think releasing the "fix" for this problem will really be a help in 3.2.1.

Comment 47 Rüdiger Lincke 2006-11-02 09:36:15 EST 
If you update eclipse, it does not update your project settings (.classpath, ...). If you got there some links, e.g.,

<classpathentry kind="var" path="JUNIT_HOME/junit.jar" sourcepath="ECLIPSE_HOME/plugins/org.eclipse.jdt.source_3.1.1/src/org.junit_3.8.1/junitsrc.zip"/>

So check your project settings, and update the path.
