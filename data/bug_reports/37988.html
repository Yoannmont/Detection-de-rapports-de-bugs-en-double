Comment 6 Konrad Kolosowski 2003-06-18 18:41:31 EDT 
If navigation does not expand that is a bigger problem, and synchronization 
cannot work since it relies on navigation working correctly and being able to 
expand it.

You might have a large number of topics in a navigation book (>2000), that 
Eclipse 2.1 and browser's have trouble handling.  If this is the case, thing 
might take very long and time out.  This problem has been cured in 3.0 stream, 
see bugs 30973, 35873, 37685.
Do you have large books?  If yes, than try waiting a little longer (a minute 
or so) after selecting a book, expanding or synchronizing and see if it helps.

Starting stand-alone should not take 7 minutes.  If you do not call stand-
alone shutdown, the next startup will take longer to execute as it will try to 
reuse the previously run stand-alone, and later recovers to start it again.  
If you had to shutdown improperly, just delete left over 
workspace/.metadata/.lock and workspace/.metadata/.connection files.  The next 
startup should be fast.  We plan on improving things in 3.0.
On slow machine, you are probably passing -servertimeout parameter to the 
standalone, experiment a bit and try decreasing the value a bit.

When you have a time, try the above suggestions and let us know what works and 
what is still broken.  You can try running latest Eclipse 3.0 (it has fixes 
for large books).
Thanks.
Comment 7 Konrad Kolosowski 2003-06-30 16:56:34 EDT 
Another possibility why synchronziation might not work is if you use document 
links in TOC that specify document fragment, like /plugin/path#anchor.  If 
such links exist without /plugin/path href also being present in the TOC, 
synchronization will not work as search will only find full 
document /plugin/path (we have no ability to index anchors only), but TOCs 
contain /plugin/path#anchor instead.
Comment 8 Carol Christensen 2003-07-01 16:47:32 EDT 
Ahhh, yes, you are so right about this!  I've removed all the anchor points 
(EVERY link had one and there were literally thousands of links).  And it seems 
to work.  I should have thought of this myself - I remember reading a warning 
about this but for some reason didn't apply it to my situation.  I guess 
because the warning seemed to be about the Search function not working properly 
(search has always worked ok for me).  I didn't equate this to SYNC.  Anyway, 
I'm closing this out.  Much thanks to you and my apologies for being so dense.
