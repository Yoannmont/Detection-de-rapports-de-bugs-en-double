Comment 7 Jeff McAffer 2006-04-06 17:23:36 EDT 
Currently we do not use the OSGi Bundle-NativeCode header at all as there are various problems with the way it works.  The simple statement is that it does not support variables and so requires the host to enumerate/know about all possible supported platforms.  As I recall (could be old info) putting the header in the fragments was also problematic.

The other problem was that the bundle native code elements were agressively extracted and installed at bundle install time.  We have several scenarios where the bundles fro several platforms are present (installed) in a system but may or may not resolve depending on the current running environment (e.g., whether or not the WS is GTK or Motif).  Note here that the WS is not know until (and change at) runtime.  That is, well after installing the bundle

Having said that, we should move to use a standard mechanism as soon as is pratical.  That does not unfortunately help in this case.
