Comment 3 Julien Canches 2006-07-31 10:27:45 EDT 
Thanks,
It looks like TPTP is not correctly recognized on your remote eclipse. The test fails because eclipse can't find the plugin "org.eclipse.hyades.test.tools.core". This is either because this plugin is not installed, or one of its pre-requisites is missing (for instance, EMF).
Another explanation could be that you installed TPTP after having launched eclipse once. In this case you need to run the remote eclipse with the -clean option on the command line, so eclipse may rescan the plugins directory, and find TPTP plugins.

The better way to fix the problem is to make sure that TPTP is working on the remote eclipse first, launched in standalone mode. Can you launch the remote eclipse instance, and check that TPTP plugins are available? (One simple verification is to see if the "Test" category is available in the "New..." wizard). If TPTP is not available, could you 1) check that all pre-requisistes are installed (EMF) and 2) try launching eclipse with -clean on the command line.
If TPTP is working on the remote eclipse launched standalone, and you still have problems with launching remote tests, then let me know.

Comment 7 Julien Canches 2006-08-07 10:11:10 EDT 
(In reply to comment #6)

> Now that may be the problem. On the remote platform I do not have all the
> plugins that are enabled on the Target Platform page of the local eclipse.
> Though I have all the required plugins for my plugin on the remote platform(I
> can run the test locally there). What could be the possible way to select only
> the plugins required by my plugin from the Target Platform sheet.

That's likely to be the problem. In the current version, TPTP does not check that the remote platform has all the plugins available. The list of plugins to activate can be edited in the workbench location editor (in the "Eclipse Workbench Properties" tab > "Choose plug-ins to launch from the list").
You need to uncheck in this list all the plug-ins that are not installed on your remote eclipse. Also, you need to make sure that the versions of the plug-ins shown in this list match the versions of the plug-ins on the remote machine (the versions need to be *completely* identical).
> 
> Also, the Location text box on the local eclipse is not the same as the remote
> one.
If you mean the "Eclipse location" text box, this field must contain the path of the eclipse directory on the remote machine.
