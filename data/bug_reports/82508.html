Comment 2 Konrad Kolosowski 2005-01-11 11:43:17 EST 
Thanks Rad.  Next time please open a bug report and add a comment (or create 
attachements) here.

Eclipse relies on web browser to view help.  Help system provides several 
browser adapters for viewing help.  The browser adapters are enabled if the 
prerequsite are met.

In 3.0 they are following browser adapter:
1) embedded mozilla - If you have Mozilla 1.4 GTK2 - or above 1.4 you will be 
able to view help in a help window which is SWT window with an embedded 
mozilla widgetg.  You will also see the option "Always use external browser" 
that will be deselected by default.
2) Mozilla - If you have Mozilla browser on they system path you will be see 
Mozilla in the external browser list.  It will be selected by default.
3) Netscape - If you have Netscape browser on they system path you will be see 
Netscape in the external browser list.  It will be selected by default, only 
if Mozilla is not installed.
4) Custom browser - This is the one you saw in the prefernce page.  It is 
selected if no other browser adaters have prerequisites met.

The best option for you is to install supported version of Mozilla.  See 
http://dev.eclipse.org/viewcvs/index.cgi/~checkout~/platform-swt-
home/faq.html#browserplatforms

In 3.1 we have added browse adapter for Firefox, and have changed the default 
path of the custom browser adapter to be "konqueror %1" which you confirmed 
works on your system.  The problem is thus fixed, even if your system does not 
have required mozilla installed.  I will mark this bug a duplicate of bug 
74935 - the one that changed default path to konqueror.
