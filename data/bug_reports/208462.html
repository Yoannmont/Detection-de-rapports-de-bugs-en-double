2. Close the Java perspective
3. Open the Java EE perspective
4. Create a new Web project.

Comment 9 Michael D. Elder 2007-11-25 18:47:05 EST 
This is the result of a very old bug. I couldn't locate the exact bug number. 

When you close the Java perspective and re-open the Java EE perspective, the IWorkbenchPage from the IWorkbenchSite supplied to the CommonNavigator view returns null: 

	/**
	 * @return The initial input for the viewer. Defaults to
	 *         getSite().getPage().getInput()
	 */
	protected IAdaptable getInitialInput() {
		return getSite().getPage().getInput();
	}

As a result, the input to the NavigatorContentService is null, and none of the extensions are activated. 

I traced the issue back to 
ShowPerspectiveHandler.openPerspective(String, IWorkbenchWindow):

...
IAdaptable input = null;

final IWorkbenchPage activePage = activeWorkbenchWindow.getActivePage();
if (activePage != null) {
	input = activePage.getInput();
}

try {
	workbench.showPerspective(perspectiveId, activeWorkbenchWindow,
				input);
...


.. where the active page is null since there is no open perspective.

Tod -- can you shed some light on this issue?
