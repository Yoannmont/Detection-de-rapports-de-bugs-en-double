Comment 9 Ed Merks 2005-04-03 11:38:20 EDT 
Created attachment 19493 [details]
Simplified handling of @ for terminating tag content.

I've made the following changes.
It's not possible to know all valid tags, so TAGS is removed.
A tag should not be ended by a '.' character.
The next '@' character should terminate a tag's content, regardless of what
comes after.
This way, there's no need for a PushBackReader, so that's removed.
And methods that are unused as a result of the simplification are removed.
When subsituting qualification, a #getXyz() should turn into getXyz() not
.getXyz().

Comment 21 Dani Megert 2005-05-25 07:00:37 EDT 
I reviewed tag coloring patch and can't apply it because it is too aggressive
since it colors every word that starts with a '@' as tag. For example e-mail
addresses in Javadoc would get the tag color.

We should only do this for the first word of a tagged paragraph (see JLS 18.4)
i.e. for tags that start a line. In addition there needs to be fixed list of
tags that are allowed outside such paragraphs, e.g. {@inheritdoc}.

Removing target milestone until we get a revised patch.

Comment 25 Tom Hofmann 2005-05-27 05:19:32 EDT 
Reopening. Mail address handling does not work at all. We should only accept
something as tag (custom or not) if it either:

- starts a line (less any indent conforming to this regex: '\w*\*?\w*')
- follows an opening brace '{'

Currently, the mail address in the followin snippet is recognized as tag:

	/**
	 * Comments with <a href="mailto:mail@address.name">mail</a>
	 * 
	 * @custom persistence:none
	 * @param raw foo
	 */

The hover is:

 Comments with
Parameters:
    raw foo
@address.name
    >mail
@custom
    persistence:none
