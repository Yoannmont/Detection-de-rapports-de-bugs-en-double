Description Phil Herold 2004-10-25 10:31:22 EDT 
The code at http://java.sun.com/features/2002/09/pword_mask.html
(which is useful for getting and masking a password from a shell), sends the 
Eclipse console into an infinite loop. It doesn't seem to recognize/handle \b.
Comment 1 Kevin Barnes 2004-10-25 13:27:28 EDT 
It's true that the console does not handle \b, but the code from the sun site doesn't use \b. The loop is 
intentional in this code. MaskingThread tries to write \r + prompt to the console every millisecond to 
prevent the user's input from being displayed. 
The problem here is that the Console does not properly handle \r because its document considers \r to 
be a legal line delimiter. Handling \r (and \b) properly would require changes to ConsoleDocument and 
to the Viewer which would need to update it's caret position.
Comment 2 Kevin Barnes 2004-10-25 15:02:34 EDT

Comment 78 Paul Pazderski 2019-10-10 16:25:51 EDT 
Soon we can celebrate the bugs 15 years anniversary. But a fix is in sight. In fact I started working on this 9 month ago because I don't want to wait another 5 years just to see it's still unresolved.

One of the main reasons I need so long was the state of existing Eclipse console. Console worked quite well most of the time but had some nasty bugs for some less common situations. Two examples:
  Before bug 544970 some input could totally break Console View with fixed console width enabled and it was more likely to trigger those problems if \b is interpreted. Also the fix required a problematic workaround for some input and I just recently found a better solution for this workaround.
  Before bug 548356 some user input could totally muddle console partitioning. Not a good condition to start overwriting existing partitions.
In general many parts of console are written on the assumption that console output is always appended on the end. And finally the existing test coverage for console was... incomplete.

But I'm optimistic to finally bring handling of \b and \r for 4.14 release.
