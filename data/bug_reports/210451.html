The SWT code below will reproduce the same behaviour where the second shell's button cannot be clicked on. The APPLICATION_MODAL style bit was copied from JFace's Dialog class. The snippet was tested on I20071212-1300.

import org.eclipse.swt.SWT;
import org.eclipse.swt.events.SelectionAdapter;
import org.eclipse.swt.events.SelectionEvent;
import org.eclipse.swt.layout.FillLayout;
import org.eclipse.swt.widgets.Button;
import org.eclipse.swt.widgets.Display;
import org.eclipse.swt.widgets.Shell;

public class Test {

	public static void main(String[] args) {
		Display display = new Display();
		Shell shell = new Shell(display, SWT.APPLICATION_MODAL);
		shell.setLayout(new FillLayout());

		Button button = new Button(shell, SWT.PUSH);
		button.setText("Open Modeless"); //$NON-NLS-1$
		button.addSelectionListener(new SelectionListenerImpl());

		shell.pack();
		shell.open();

		while (!shell.isDisposed()) {
			if (!display.readAndDispatch())
				display.sleep();
		}
		display.dispose();
	}

	public static class SelectionListenerImpl extends SelectionAdapter {
		public void widgetSelected(SelectionEvent e) {
			Button b = (Button) e.widget;
			Shell modelless = new Shell(b.getShell());
			modelless.setLayout(new FillLayout());

			Button clickThis = new Button(modelless, SWT.PUSH);
			clickThis.setText("Can't click this!"); //$NON-NLS-1$

			modelless.pack();
			modelless.open();
		}
	}

}

Comment 17 Praveen 2009-06-11 06:31:17 EDT 
Created attachment 138912 [details]
Patch

I have investigated the reported problem (Modal Shell (Parent) --> Modelless Shell (Child)). 
When a modal shell doesn't have any parent (or having another modal shell as it's parent), then the modal shell is incorrectly added to the default group and so it's children(modelless shell). Thus, since both the shells fall in the same window group, modal shell prevents the interaction of modelless shell. In order to avoid this, modelless shell should always fall in different group from it's parent modal shell.
So, the fix is to ensure that whenever there is a modal shell (having no parent, or another modal shell as it's parent) in the hierarchy, then we always add modal shell's group into it's own group, or parent's modal shell group. Thus, it's children's (modelless shell) group is always segregated as they are added into other group (could be default group).
