Thanks for all you comments.

Big files can require bigger chunks of memory when they are loaded in memory. As
mentioned, we are using the nio buffers from the JDK libraries so no file would
be fully loaded in memory (only partly memory mapped)

I'm surprised that you can work nicely with 64m, my avarage required size of my
workspace is around 150 M. I'm also surpised that full build can du much better
than a full search, but of course a Java full build only has to touch Java files
with are usualy not that big.

I had a look at the code again. It is not much code and I can't really see where
the problems can be. 

How big are the files you are looking at? Can you specify a better file name
pattern to filter out binary files?

I would like to find out if the nio is the reason for your problems. I
