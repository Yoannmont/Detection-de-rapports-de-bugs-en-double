Description Rafael Chaves 2003-09-10 12:24:16 EDT 
i20030910

Eclipse deadlocks when organizing imports. This is happening with M3 as well.
Not sure it is related to bug 41925. 

My test case involves two projects. One class in one project refers to another
class in another project, but the import is incorrect/missing. When organizing
imports on project that has a missing import, Eclipse deadlocks.

The projects used in the test case will be attached. Just select package
packageA in project A and press Ctrl-Shift-O.

It does not happen if the referred class is in the same project, or only one
class is selected in the operation.
