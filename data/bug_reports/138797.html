Description Jesper Moller 2006-04-26 20:42:26 EDT 
XSD Graph editor:

When refactoring a local element containing form, minOccurs or maxOccurs attribute into a global element, these elements are copied to the global element, and minOccurs and maxOccurs are no longer in effect for the element reference.

Try this XSD file:

<?xml version="1.0" encoding="UTF-8"?>
<xsd:schema targetNamespace="urn:whatever" xmlns:whatever="urn:whatever" xmlns:elementFormDefault="unqualified" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
    
	<xsd:complexType name="myComplexType">
		<xsd:sequence>
			<xsd:element name="movingElement"
				id="abce" nillable="false" 
				type="xsd:string" default="fine" form="unqualified"
				maxOccurs="5" minOccurs="3"/>
		</xsd:sequence>
	</xsd:complexType>

</xsd:schema>



Now, go to graph mode, doubleclick on the complex type, right click on the "movingElement" and refactor that into a global element.

It will look like this: 

<?xml version="1.0" encoding="UTF-8"?>
<xsd:schema targetNamespace="urn:whatever" xmlns:whatever="urn:whatever" xmlns:elementFormDefault="unqualified" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
    
	<xsd:complexType name="myComplexType">
		<xsd:sequence>
			<xsd:element ref="whatever:movingElement"></xsd:element>
		</xsd:sequence>
	</xsd:complexType>

	<xsd:element name="movingElement" id="abce" nillable="false"
		type="xsd:string" default="fine" form="unqualified" maxOccurs="5"
		minOccurs="3" />
</xsd:schema>

And this is wrong because of the form and maxMinOccurs which are on the new global element, but no longer on the local reference.
Also, "form" shouldn't be used anymore, since we have eliminated the local usage.
Comment 1 Jesper Moller 2006-04-26 20:55:41 EDT 
Created attachment 39616 [details]
Patch to fix this bug (against 1.0.1)

Patch to fix this bug - by removing form and preserve mix/maxOcccurs on local particles when refactoring into global elements. The result of the example in the description is now like this (no red wavy lines!!!):

<?xml version="1.0" encoding="UTF-8"?>
<xsd:schema targetNamespace="urn:whatever" xmlns:whatever="urn:whatever" xmlns:elementFormDefault="unqualified" xmlns:xsd="http://www.w3.org/2001/XMLSchema">
    
	<xsd:complexType name="myComplexType">
		<xsd:sequence>
			<xsd:element minOccurs="3" maxOccurs="5"
				ref="whatever:movingElement">
			</xsd:element>
		</xsd:sequence>
	</xsd:complexType>

	<xsd:element name="movingElement" id="abce" nillable="false"
		type="xsd:string" default="fine" />
</xsd:schema>

P.S: It would be prudent to warn the user if his element used form="unqualified" (or inheirited this setting from the schema level) since when the element is refactored into a global one, it will always have to be qualified.
