Comment 4 David Williams 2007-06-03 16:33:10 EDT 
> Is it planned to make the separate feature suitable for use in an RCP
> application?

No, not literally an RCP Application. But, I've found many times many people say "RCP Application", when they really mean "Eclipse Platform Only" requirement. 

Our goal is that latter case. We deliberately rely too much on "Eclipse Projects" and "Eclipse Resources" to be RCP only, and we deliberately _want_ be be well integrated into the platform's IDE. 

None of us on the core teams have sized what it would take to be literally RCP only ... and, we are not clear on what the need is (that is, what the end-user user case is for RCP only Editors).
Comment 5 David Williams 2007-06-06 00:45:26 EDT 
I have confirmed we can easily "expose" just the XML editor and tools (and DTD, and Schema Editor) by simply listing the XML UI feature in the site.xml. 

It does require one complication of having the Common feature components in an "Enabling Features" part of the update site (which is unfortunately, the only solution other than using 'includes', but then that makes maintenance etc., more difficult). 

Also, while it is a bit late to be changing UI labels and words, I really think the name and description for the current "top" XML feature should be changed. 
It is currently sort of short technical names and descriptions, and then we should now spell these out since exposed to end-users. I propose the following. 

WST XML UI  -->  Eclipse XML

Web XML tools --> Eclipse XML Editors and Tools (XML, DTD, Schemas)
Comment 6 David Williams 2007-06-06 00:47:14 EDT 
BTW, I have found we will likely expose some other, minor bugs with the "reduced install" ... in some quick testing, for example, I saw some log messages about certain preference page paths not being found ... but, everything still seemed to work ok. Just wanted to document this "negative" part, to set expectations, that things won't be perfect with this late change.
Comment 7 David Williams 2007-06-06 00:52:26 EDT 
Craig (and Nitin) I'll need your review and approval of this concept. It will make the xml, dtd, and schema editor separately installable from the Europa update site. In particular, it can be installed with just the Eclipse Platform, a minimal part of EMF, and GEF. 

But, since late, there is some risk it will expose new bugs, which, I suspect, will be valid but minor bugs that we can fix in 3.0 stream. 

It's hard to "install" this using zips, because it requires a subset of EMF (if doing without JDT) that is not available in just zips. But, if you approve, we can put on a "test Europa site" for some quick testing, and roll back if something seems damaging.

Comment 10 David Williams 2007-06-07 04:03:56 EDT 
I have talked to Craig and Nitin, and they agree this is ok, so submitting for PMC Review and approval. 

Both Craig and Nitin had suggestions to improve the name (currently "WST XML UI")  and description (currently "Web XML Tools"). Normally these are not very "visible" unless a feature is a "top level" feature in the Update Manager. 

Combining all the suggestions, for this release, sounds like the 
best name would be 

Eclipse XML Editors and Tools

and the description expanded to 

XML, DTD and XML Schema Editors, validators, and XML Catalog support. 

= = = = 

I should also note here explicitly and publicly, that since this is 'going in late' there will undoubtedly be new small bugs found from running a "reduced installation". Out of time and resource constraints, we will have to give most such bugs a little lower priority in some cases than "full install" bugs, but for the most part would appreciate knowing about them, having them well documented, researched, etc., to improve our overall code base for next release. Or ... perhaps this new feature will generate some new interest from the community in joining the effort and relieve some of the time and resource constraints! (hint hint :)
