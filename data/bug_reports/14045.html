Comment 6 Darin Wright 2002-04-22 11:56:15 EDT 
Added these fixes - now unable to get deadlock on close:
* When the streams proxy is killed, it also closes the input stream monitor
* When the streams proxy is closed, it checks to see if already closed before 
closing again
* Output stream monitor checks for an NPE while reading from the underlying 
stream (when a stream is closed while a read is in progress, the underlying 
stream can throw an NPE). The NPE is only considered safe if the output stream 
monitor is killed/closed
* ConsoleDocument now supports a kill mode as well as a close - the active 
documents are killed on shutdown so reading from queued data stops immediately

Fixed
