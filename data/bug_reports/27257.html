Description Adrian Mardare 2002-11-27 12:38:17 EST 
We discovered a problem in the implementation of swt for photon - related to 
graphical primitives like gc.drawLine, gc.drawRectangle etc.
This is impacting the performance of awt based applets or java applications, 
that use the j9 personal configuration.
To really see this the applet/application has to draw a lot of primitives - we 
have seen this using the dinasaur applet example for sun, that is drawing a 3D 
object using more than 1500 lines.
 
The problem is that a call to PgFlush() is made indirectly after each 
line/rectangle etc is drawn. This means that a context switch to the graphic 
driver is made, the line/rectangle is drawn and then a context switch back is 
made etc.
 
I attached a GC.java that is fixing this. We tested this with awt based applets 
and the performance is much better. I also built a swt.jar to be used with the 
eclipse for photon and it doesn't seem to be any problems in the eclipse.
 
I did my changes on the head branch version on GC.java ver. 1.31. The location 
in the repository is
org.eclipse.swt\Eclipse SWT\photon\org\eclipse\swt\graphics\GC.java
 
About my changes:
The PgFlush() is made indirectly from two causes:
 
a) when a PgSetGC is made. The old photon GC is flushed if its different from 
the new one.
A code like this
<<<<<
int prevContext = setGC();
setGCClipping();

OS.PgDrawILine(x1, y1, x2, y2);

unsetGC(prevContext);

>>>>

will be causing a PgFlush() because setGC() contains a PgSetGC( handle) and 
unsetGC contains a PgSetGC( prevContext ).

b) when the OS.PgSetClipping() is made. This call is made from inside the 
setGCClipping().

What I propose is to:

- record the old photon context in "prevGC" in the constructor of GC( 
drawable ).

- move the PgSetRegion() calls from setGCClipping() into the constructor of GC( 
drawable ). This way you don't need to call PgSetRegion() when each drawLine 
etc is made.

- comment out the remaining code of the setGCClipping()

- in the unsetGC() method remove the code that does a PgSetGC( prevContext ) 
and do that only once, in the dispose() method, instead of doing it for each 
primitive ( I mean add a PgSetGC( prevGC ) in the dispose() method ).

	public static void main(String[] args) {
		Display display = new Display();
		Shell shell = new Shell( SWT.CLOSE | SWT.TITLE | SWT.MIN | 
SWT.MAX | SWT.RESIZE );
		shell.setText( "title" );
		shell.setSize( 500, 400 );
		Color bkgd = new Color( display, 255, 255, 255 );
		shell.setBackground( bkgd );
		bkgd.dispose();
	
		Raw raw = new Raw( shell, 0 );
		
		Color yellow = new Color( display, 255, 255, 0 );		
		raw.setBackground( yellow );	
		yellow.dispose( );
		
		raw.setLocation( 10, 10 );
		raw.setSize( 200, 200 );

		Raw raw1 = new Raw( shell, 1 );
		raw1.setLocation( 100, 100 );
		raw1.setSize( 200, 200 );

		Button b = new Button( shell, 0 );
		b.setText( "draw" );
		b.addMouseListener( raw1 );
		b.setLocation( 400, 0 );
		b.setSize( 50, 20 );


		shell.open();

Comment 5 Adrian Mardare 2003-01-02 11:10:33 EST 
I looked at your patch and the reason you are seeing the error is because in 
the init() you have to first set PgSetGC(handle) and then call PgSetRegion().
So instead of:

.....
	int rid = data.rid;
	int widget = data.widget;
	if (rid == OS.Ph_DEV_RID) OS.PgSetRegion(rid);
	else if (widget != 0) OS.PgSetRegion(OS.PtWidgetRid(widget));
	
	if (widget == 0) return;
	data.prevContext = OS.PgSetGC(handle);
.....

it should be:
....
	data.prevContext = OS.PgSetGC(handle);
	int rid = data.rid;
	int widget = data.widget;
	if (rid == OS.Ph_DEV_RID) OS.PgSetRegion(rid);
	else if (widget != 0) OS.PgSetRegion(OS.PtWidgetRid(widget));
	
	if (widget == 0) return;
......

Anyway, in the meantime Misha Nefedov has made some changes in the photon 
library clipping related code ( PgSetClipping is not flushing the gc anymore ) 
and he has the latest changes to the GC.java that has to be made in order that 
no flushing is done when gc.drawLine and the like are called.
The latest changes do the clipping before each drawing primitive ( so that no 
drawing is done outside the widget canvas ) but the performance is much better 
bacuse of the changes in the photon library. Also instead of recording the 
prevContext and switching back to it in the dispose(), we always use NULL as 
the prevContext ( PgSetGC( 0 ) in the dispose()) since this will always set a 
safe photon context ( this avoids potential problems with two threads recording 
prevContext from one to the other and then setting a prevContext that was 
freed ).
I added mnefedov to the cc list of this bug and he will give you more 
details/patches if neccessary.
