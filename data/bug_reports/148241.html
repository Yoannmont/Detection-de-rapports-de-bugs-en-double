Description Vladimir Nicolici 2006-06-22 11:50:39 EDT 
I created a "Dynamic Web project" for a new web application. 

I used "J2EE Module dependencies" and I added several dependencies to other projects. I integrated a Tomcat application server with Eclipse, and started testing the application.

Eclipse automatically generated JAR files for the project dependencies, and placed them in WEB-INF/lib.

Then, I needed to add the JARs from an "User Library" to WEB-INF/lib, and I didn't find any way to do that. 

My "User Library" contains 70 jars from 30 directories, and I don't want to add each of the 30 directories by hand every time I create a new project that uses this library, so this is a big problem, and I will probably file an enhancement request for this.

Apparently, the only other way to add JARs to WEB-INF/lib is to use the "Add External JARs" option from the "J2EE Module dependencies" dialog. 

There is also a "Add JARs" button in that window, but the JAR selection dialog shows a empty list, and I don't know how to populate that list. Pressing F1 doesn't describe what this dialog shoud do or how it should be used.

So I tried using "Add External JARs", and I added one of my 70 JARs (c:\JavaProjects\lib\apache\log4j-1.2.9.jar), but it didn't work:

Publishing failed Error reading file C:\Program Files\eclipse\JavaProjects\lib\apache\log4j-1.2.9.jar C:JavaProjects\lib\apache\log4j-1.2.9.jar (The system cannot find the path specified)

The error message contains two paths, and both are wrong, the first one starts with C:\Program Files\eclipse\ instead of C:\, and the second one starts with C: instead of C:\ .

In the "J2EE Module dependencies" a new entry was created, but the icon has an small yellow warning sign, and the JAR/Module and Project columns are empty. Also, there is no way to remove this entry from the interface.

So I started to dig in the XML files of the project. I found this: 

in .classpath, the path is correct:
<classpathentry kind="var" path="C:/JavaProjects/lib/apache/log4j-1.2.9.jar"/>

in .settings/.component, the path is most likely incorrect (C: instead of C:/):
<dependent-module deploy-path="/WEB-INF/lib" handle="module:/classpath/lib/C:JavaProjects/lib/apache/log4j-1.2.9.jar">
<dependency-type>uses</dependency-type>
</dependent-module>

I closed the project, manually changed C: to C:/ in .settings/.component, opened the project, cleaned&built it, but it still doesn't work. The same publih error occurs, and the project is marked is marked with a small red error icon.

I also checked the problems window, and I found this:
Severity	Description	Resource	In Folder	Location	Creation Time	Id
2	Unbound classpath variable: 'C:JavaProjects/lib/apache/log4j-1.2.9.jar' in project pp	pp			22 iunie 2006 18:34:38	407439

So somehow, eclipse still "eats" that / after C:, no matter what I do. And AFAIK, C: on Windows means the current directory on the C: drive, not the root directory. (On MS-DOS and Windows you can have a different current directories on each drive). This probably explains why one error message referred to C:\Program Files\eclipse\ . Probably that was the current directory on the C: drive.

Ugly. Not uglier than Netbeans, but still ugly :)

Comment 2 Jason Sholl 2006-06-22 12:22:58 EDT 
How old is your WTP driver?  The reason I ask is you should not have a .classpath entry pointing directly to the external jar anymore.  Instead, it should be resolved by the Web App Libs classpath container: 

<classpathentry path="org.eclipse.jst.j2ee.internal.web.container" kind="con"/>
	

Also, on the latest driver, you the .component entry something like this:  

<dependent-module deploy-path="/WEB-INF/lib" handle="module:/classpath/lib/C:/temp/ArchiveUtil.jar">
            <dependency-type>uses</dependency-type>
        </dependent-module>

Note that this entry has the extra slash.
