Comment 19 Olivier Thomann 2006-04-19 00:07:51 EDT 
The problem was a incomplete initialization of a byte array from a data input stream.
This was a bug in ICU. In 6.0b77, the read(byte[]) call was initializing the whole array and therefore no error was raised.
In 6.0b78, the same call initialized only a subpart of the array (the first 8K to be precise) and this is the cause of the failure.
The fix was to replace a d.read(raw); with d.readFully(raw);
Doing this, the raw byte array is fully initialized and it works.
