Comment 7 Andrew Niefer 2007-02-12 11:10:46 EST 
Sebastian, what happens when you put Current back where it was pointing to A.  Then in the Eclipse.App/Contents/Info.plist, uncomment one of the -vm lines?

The eclipse launcher does the following:
1) For the java command given (from the search path: /usr/bin/java, or from the Info.plist file), figure out what version we want and set the environment variable JAVA_JVM_VERSION
2) Try to load /System/Library/Frameworks/JavaVM.framework/Versions/Current/JavaVM which chooses the vm based on the above property.  

On my mac,  Current->A, and /usr/bin/java -> CurrentJDK -> 1.5 -> 1.5.0
This means we load .../Versions/A/JavaVM and set the requested version to 1.5.0

On my Mac, JavaVM only exists under A, and so moving current would result in the 2nd error message you get.

Comment 14 Andrew Niefer 2007-02-15 10:27:52 EST 
Thank you Sebastian, from comment 4 and comment 13, this is what I think is happening:
1) eclipse finds /usr/bin/java on path.
2) resolving symlinks goes
->/System/Library/Frameworks/JavaVM.framework/Versions/Current/Commands/java
->/System/Library/Frameworks/JavaVM.framework/Versions/A/Commands/java
3) eclipse parses this path to choose a version, and gets "A".  

Note, on my mac, A/Commands/java does not even exist.  It looks like installing 6.0 changed something with the A.

I will change the code so that if we get something that we don't understand for the version (maybe as simple as if the first character isnt a number) we just won't specify one, then I think that the JavaVM will pick whichever one it likes.
