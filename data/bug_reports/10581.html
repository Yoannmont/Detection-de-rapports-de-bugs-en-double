   - extra RPMs: Sun's JDk 1.3.1_02

20020228)

2. Description
   If your project contains files whose names end in anything other than .java
or .class (I've tried with .xml and .txt, but not with .jar.  I know that
.classpath  is also guilty of causing this) then expanding the Project folder in
the Resource View (so that you see the files/folders contained therein) causes
the exception below.

3. How I tested
   I'd touch(1) the filename I wanted to test against, start Eclipse and do a
refresh-from-local.  The first thing I did was remove .classpath, but I tested
against it later on.

4. Work-around
    Rename the kde-specific motif .so so that they cannot be found.  (I usually
rename the gnome-specific motif .so as well).
