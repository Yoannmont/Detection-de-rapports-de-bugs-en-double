Description Alan Malloy 2005-04-18 20:41:55 EDT 
"Compare with->Local history" can cause the entire Eclipse platform to crash and
burn. 

Reproducing:
This can be reproduced with any class in the workspace. Simply right-click that
class and select "Compare with->Local history". In the menu which appears,
select the uppermost label in the left pane - it will be labeled something like
"Today (April 18, 2005)". Eclipse will play the system bell, display its usual
"Unexpected exception" error window, and disappear. All unsaved work will be
destroyed without warning.

Note: after writing this, I saved my workspace and reopened it, and the bug had
disappeared. I can no longer reproduce it. I realize that unreproducible bugs
are a pain, but I'm going to post this anyway because it did happen to me about
five times as I was writing this and double-checking the steps to perform, and
it shouldn't have. Perhaps just knowing what I did will give you enough of an
idea to figure out what went wrong. The problem fixed itself only when I let
Eclipse close itself naturally and save its workbench state - perhaps there was
some stale entry in the workbench state causing it to behave this way.
