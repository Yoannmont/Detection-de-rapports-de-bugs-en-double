Comment 3 Wassim Melhem 2004-01-19 06:35:25 EST 
When you use Update Manager to install plug-ins at a location other than the 
main Eclipse install, UM creates a soft link (i.e. it does not explicitly 
create a .link file containing the information where to find the plug-ins in 
this new location).
When searching for plug-ins, PDE searches the main Eclipse install tree and 
goes through all the .link files and search all the locations listed in those 
files as well.  So all the soft links created by UM are (and always have been) 
missed by PDE because it does (never had) access to that information.

Since you're suggesting that, prior to M6, the plug-ins in question were 
indeed picked up, then I'm certain (although you did not say so) that your 
host platform = target platform.  This is because, in this case, prior to M6, 
we use to read the plug-ins from the live registry, and the live registry 
contains all the plug-ins that you see in the About Dialog.  However, with the 
OSGi runtime, we were no longer able to do that, and even when target platform 
= host platform, PDE can only resolve plug-ins as outlined in the previous 
paragraph, so we miss those softly-linked plug-ins.

Jeff and Dorian, any thoughts?
