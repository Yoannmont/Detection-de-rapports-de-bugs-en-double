Description Eric Moffatt 2007-02-06 14:01:22 EST 
Found in I20070206-0010 (M5 candidate).

When editing a plugin.xml file the context menu entries for "Shift Left" and  "Shift Right" don't work. They do work for '.txt' files...
Comment 1 Markus Keller 2007-05-02 14:11:23 EDT 
Same for Shift+Tab and Tab with a selection.
Comment 2 Wassim Melhem 2007-05-02 14:15:10 EDT 
I tried to debug this one the other day and if I recall correctly, it seems that Text does not like one of the XML document partitions we have.

This whole partition thing is a black art to me, and I think we have to bring in the big guns (ie. Dani) to help with this one.
Comment 3 Wassim Melhem 2007-05-02 15:52:48 EDT 
With a short tutorial from Dani, I now understand the problem.

The XML configuration was using the default partitioning, but when asked what its partitioning was, it was returning some bogus value.  So actions like Shift Left and Right were disabled.

Fixed by removing the bogus getConfiguredDocumentPartitioning() method.

I have been shifting left and right non-stop for the past 5 minutes and it works well.

Thanks Dani.

Comment 14 Mike Pawlowski 2007-05-09 16:01:18 EDT 
Content assist works great.  So does 'Shift Left' and 'Shift Right'

I only noticed one minor issue whose correct behaviour I'm not sure about.

If you select all four lines of an extension block - for example:

   <extension-point
      id="cheatSheetItemExtension"
      name="%CHEAT_SHEET_ITEM_EXTENSION"
      schema="schema/cheatSheetItemExtension.exsd"/>

and then shift them left, you get:

   <extension-point
  id="cheatSheetItemExtension"
  name="%CHEAT_SHEET_ITEM_EXTENSION"
  schema="schema/cheatSheetItemExtension.exsd"/>

If you do the same thing in the text editor, nothing will get shifted because the first line has only 3 spaces prepended to it (requires 4 spaces or 1 tab I believe).  Maybe a result of different partitioning used?
